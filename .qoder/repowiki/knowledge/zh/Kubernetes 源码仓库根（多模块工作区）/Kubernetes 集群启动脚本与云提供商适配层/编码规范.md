- 每个 provider 目录遵循 `config-common.sh` + `config-default.sh` + `util.sh` 三层结构，其中 `config-default.sh` 只暴露 `export VAR=${KUBE_VAR:-default}` 形式的变量定义。
- 所有脚本顶部统一声明 `set -o errexit; set -o nounset; set -o pipefail`，并通过 `KUBE_ROOT=$(dirname "${BASH_SOURCE[0]}")/..` 定位仓库根路径。
- Provider 接口通过 `skeleton/util.sh` 中的函数签名约定，新增 provider 只需在该文件中补充同名函数即可接入 `kube-up.sh`。
- Add-on 清单按功能域拆分到 `addons/<component>/` 子目录，每个子目录含 OWNERS 文件，RBAC 与 Deployment 分离存放。
- DNS 类 add-on 使用 `.yaml.base` + `.yaml.in` + `.yaml.sed` + `transforms2sed.sed` 的模板管线，由 Makefile 驱动生成最终清单。
