模块以 `cluster/kube-up.sh`、`cluster/kube-down.sh`、`cluster/validate-cluster.sh` 为统一入口，通过环境变量 `KUBERNETES_PROVIDER` 动态选择 provider：默认加载 `cluster/${KUBERNETES_PROVIDER}/util.sh`，该文件必须实现 `detect-master/detect-node-names/detect-nodes/verify-prereqs/kube-up/kube-down/test-build-release/test-setup/test-teardown/prepare-e2e/detect-project` 等函数；`cluster/skeleton/util.sh` 给出所有必需函数的空实现作为新 provider 的模板。

- 配置分层：每个 provider 目录包含 `config-common.sh`（共享逻辑）、`config-default.sh`（变量默认值，如 GCE 的 ZONE/NODE_SIZE/IMAGE_FAMILY 等）、`upgrade.sh`、`util.sh`；provider 的 `util.sh` 再 source 这些配置文件。
- 运行时依赖：`cluster/common.sh` 提供跨 provider 的通用工具（kubeconfig 读写、版本正则、kubectl wrapper），由入口脚本在调用 provider 前 source。
- Addon 资源：`cluster/addons/` 按组件分目录存放 YAML（coredns/kube-dns、calico、metrics-server、fluentd-gcp、ip-masq-agent、volumesnapshots 等），GCE 特有扩展位于 `cluster/gce/addons/`；DNS 子目录使用 `.base/.in/.sed` + Makefile 的模板替换机制生成最终清单。
- 镜像构建：`cluster/images/` 下的 etcd-version-monitor、kubemark 各自带独立 Dockerfile/Makefile，由对应目录的 OWNERS 维护。
- 辅助工具：`cluster/log-dump/`、`cluster/get-kube.sh`、`cluster/get-kube-binaries.sh`、`cluster/kubectl.sh` 分别负责日志导出、二进制下载和 kubectl 封装。
- 依赖方向：入口 → common.sh → provider util.sh → provider config-default.sh → 外部 gcloud/kubectl；addon YAML 仅被 kube-addon-manager 消费，不反向依赖脚本。
