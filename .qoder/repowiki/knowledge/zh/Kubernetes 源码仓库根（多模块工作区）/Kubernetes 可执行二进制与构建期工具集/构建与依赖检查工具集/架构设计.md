本模块由三个并列的独立 `main` 程序组成，每个子目录对应一个可执行二进制，彼此无共享代码：
- `cmd/clicheck/`：基于 `k8s.io/kubectl/pkg/cmd/util/sanity` 对 kubectl CLI 进行约定性自检（命令命名、帮助文本、flag 风格等），通过 `RunCmdChecks` + `RunGlobalChecks` 收集错误并汇总退出码。
- `cmd/dependencycheck/`：读取 `go list -deps -json` 输出的 JSON 流，用 `-restrict` 正则匹配受限制的 import 路径，用 `-exclude` 排除特定包，输出违规清单后以非零退出码结束。
- `cmd/dependencyverifier/`：调用 `go mod graph` 解析依赖图，结合 `vendor/modules.txt` 与 `dependencies.json`（spec/status 双结构）对比当前状态与期望状态，检测 unwanted 模块的新增引用、被移除的引用、新增 vendoring 以及 pinned 版本漂移；使用 `github.com/google/go-cmp` 做结构化 diff 并提示需要更新的文件位置。

依赖方向单向向外：仅依赖标准库、kubectl sanity 框架和 go-cmp，不反向依赖仓库内任何业务包，因此可直接作为 CI 阶段的独立步骤运行。
