模块按 `cmd/kubelet` 标准 Go 应用结构组织：
- `kubelet.go` 是唯一的 `main` 包，仅注册 JSON 日志、Prometheus clientgo/version 指标等 side-effect import，随后调用 `app.NewKubeletCommand(context.Background())` 并交由 `component-base/cli.Run` 执行。
- `app/` 是核心启动层：`server.go` 通过 cobra 构建命令，采用 `DisableFlagParsing=true` + 手动 `pflag.FlagSet` 的方式实现自定义 flag 优先级（先解析基础 flags → 加载 `--config` 与 `--config-dir` drop-in `.conf` 文件 → 再 re-parse 以覆盖），随后构造 `options.KubeletServer` 并通过 `UnsecuredDependencies` 组装运行时依赖，最终进入 `Run(ctx, ...)` 启动 kubelet 主循环；`auth.go` 处理认证初始化，`plugins.go` 集中声明所有静态 Volume 插件的 `ProbeVolumePlugins()` 调用列表，`GetDynamicPluginProber` 暴露 Flexvolume 动态探测入口。
- `app/options/` 定义 `KubeletFlags`（进程级、不可共享的配置）与 `KubeletConfiguration`（可跨节点共享的 v1beta1 配置对象），并提供 `AddFlags` / `Validate*` / `New*` 工厂方法；`globalflags.go` 与平台变体 `globalflags_linux.go`、`osflags_windows.go` 分离全局 flag 注册。
- 平台差异通过 build tag 拆分：`init_windows.go` 在 Windows 下创建 Job Object 并设置进程优先级类，`server_windows.go`/`server_linux.go`/`server_others.go`/`server_unsupported.go` 分别承载各平台的 server 初始化逻辑，`init_others.go` 为其他平台空实现。
- 依赖方向单向向下：`cmd/kubelet/app` → `pkg/kubelet/*`（core、cm、eviction、server、metrics 等），本模块不反向引用 pkg 以外的 k8s.io 组件，仅作为装配器与 CLI 外壳。
