- 平台相关代码通过 go build tag（如 `//go:build windows`）拆分为同名文件（`*_windows.go` / `*_linux.go` / `*_others.go` / `*_unsupported.go`），由编译器选择编译目标。
- CLI 启动流程遵循 `NewKubeletCommand` 返回 cobra.Command → RunE 中禁用 Cobra 自动解析、手动用独立 `pflag.FlagSet` 解析 → 加载 config/dropin → re-parse 覆盖 → 构造 KubeletServer → UnsecuredDependencies → Run 的标准顺序。
- 配置对象统一提供 `New*` 工厂函数与 `AddFlags` / `Validate*` 配套方法，并在 `options.go` 中以注释说明字段应放入 `KubeletFlags` 还是 `KubeletConfiguration` 的决策规则。
- 插件集合通过显式 import 驱动链接，再由 `ProbeVolumePlugins` 等集中扫描函数收集，避免反射或动态加载，变更需同时修改 import 与探针列表两处。
