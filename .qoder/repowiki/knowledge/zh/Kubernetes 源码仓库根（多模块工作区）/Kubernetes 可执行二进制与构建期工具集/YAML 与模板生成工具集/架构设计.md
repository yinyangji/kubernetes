本模块由三个互不依赖的 Go `main` 包组成，每个子目录对应一个独立的二进制：
- `cmd/genyaml`：通过 `k8s.io/kubectl/pkg/cmd` 构造完整的 kubectl 命令树，递归遍历 `cobra.Command` 及其 flag，将名称、Synopsis、Long、Options/InheritedOptions、Example、SeeAlso 等字段序列化为 `go.yaml.in/yaml/v2` 格式的 YAML 文件，输出到 `docs/yaml/kubectl`（可通过命令行参数覆盖），用于生成 kubectl 文档。
- `cmd/gotemplate`：极简的 `text/template` 包装器，stdin 读入模板、stdout 写出结果，支持 `<key>=<value>` 形式的键值注入，并扩展 `include`、`indent`、`trim` 三个自定义函数，供构建脚本在模板中嵌入片段或缩进。
- `cmd/prune-junit-xml`：CI 后处理工具，使用 `flag` 包解析 `-max-text-size`、`-prune-tests`、`-add-owners` 三个开关，借助 `third_party/forked/gotestsum/junitxml` 结构体读写 JUnit XML；`logparse` 子包实现流式 klog 日志解析器，按行识别 I/W/E/F 级别及多行字符串续行，使主程序能剔除纯 klog 输出、仅保留失败信息；可选地通过 `go list -f '{{.ImportPath}}:{{.Dir}}'` 反向查找包源码路径，向上遍历 `OWNERS` 文件提取 `sig/*` 标签为测试套件名添加 `[sig-...]` 前缀。
依赖方向单向：`prunexml.go` → `logparse` + `third_party/forked/gotestsum/junitxml` + `sigs.k8s.io/yaml`；`genyaml` → `kubectl/pkg/cmd` + `cli-runtime`；`gotemplate` 仅依赖标准库。
