- Options 对象遵循 `New*` 构造 + `ApplyTo(*Config) error` + `Validate() []error` + `Config(ctx) (*Config, error)` 的标准生命周期模式，由 server 层按序调用。
- CLI 标志通过 `cliflag.NamedFlagSets` 按功能分组（如 "secure serving"、"authentication"、"leader election"、"deprecated"），每个子选项通过 `AddFlags(nfs.FlagSet(name))` 注入统一命名空间。
- HTTP 服务通过 `buildHandlerChain` 以装饰器方式叠加 WithAuthentication / WithAuthorization / WithRequestInfo / WithCacheControl / WithHTTPLogging / WithPanicRecovery 过滤器，再交给 `mux.PathRecorderMux` 挂载具体 handler。
- 健康检查分为 healthz、livez、readyz 三类，分别通过 `healthz.InstallHandler`、`InstallLivezHandler`、`InstallReadyzHandler` 注册，并由 `isLeader` 与 `handlerSyncReadyCh` 控制就绪语义。
- LeaderElection 在启用时通过 `leaderelection.NewLeaderElector` 包装 `OnStartedLeading`/`OnStoppedLeading` 回调，非 leader 路径直接 inline 运行，退出码区分正常终止与丢失租约。
