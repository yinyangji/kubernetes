- 使用 Options → Config → CompletedConfig 三层结构：Options 负责 flag 绑定与 Validate，Config 负责资源创建，CompletedConfig 通过私有 embedded pointer 对外暴露只读已完成配置。
- 服务器链采用委托模式（DelegationTarget）串联：notFoundHandler → ApiExtensions → KubeAPIs → Aggregator，每个阶段返回的 GenericAPIServer 作为下一个阶段的 delegate。
- 对可被测试覆盖的全局状态（如 `testServiceResolver`）提供 `SetXxxForTests` 函数并在返回闭包中清理，强制测试串行执行以避免并发污染。
