本包是 `cmd/kube-apiserver/app` 的入口层，围绕三个职责分层：
- 选项定义 (`options.go`)：导出 `ServerRunOptions`（嵌入 `controlplaneapiserver.Options`）和仅本进程使用的 `Extra`（如 `ServiceClusterIPRanges`、`KubeletConfig`、`EndpointReconcilerType`、`MasterCount`），通过 `Flags()` 方法向 `cliflag.NamedFlagSets` 注册所有 `--kubelet-*`、`--service-cluster-ip-range`、`--endpoint-reconciler-type` 等 flag。
- 完成阶段 (`completion.go`)：`ServerRunOptions.Complete(ctx)` 负责把用户字符串形式的 `ServiceClusterIPRanges` 解析为 `PrimaryServiceClusterIPRange`/`SecondaryServiceClusterIPRange`/`APIServerServiceIP`，并委托 `controlplaneapiserver.Options.Complete` 完成通用组件初始化；同时根据 feature gate 合并 watch-cache 默认大小。对外暴露不可实例化的 `CompletedOptions`（内部持有私有 `completedOptions`）以强制调用顺序。
- 校验阶段 (`validation.go`)：`CompletedOptions.Validate()` 串联多项业务规则——CIDR 数量与最大前缀长度、双栈 IP 族一致性、NodePort 范围包含关系、advertise address 与 service IP 族匹配、`--apiserver-count` 正数约束，并复用 `genericoptions.ServerRunOptions.Validate()` 做通用校验。
依赖方向单向向外：本包依赖 `pkg/controlplane/apiserver/options`、`pkg/controlplane/reconcilers`、`pkg/features`、`pkg/kubeapiserver`，被上层 `app/server.go` 消费，不反向依赖任何 server 实现。
