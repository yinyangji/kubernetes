本模块位于 `cmd/kube-controller-manager/app` 包内，遵循 KCM 的“按功能拆分文件 + 统一 ControllerDescriptor 注册”模式：
- `discovery.go` 与 `validatingadmissionpolicystatus.go` 各自实现一对函数：`newXxxControllerDescriptor()` 返回包含 name/aliases/constructor 的 `*ControllerDescriptor`，以及 `newXxxController(ctx, controllerContext, controllerName)` 构造具体控制器并通过 `newControllerLoop` 包装成 `Controller`；这些描述符由上层入口（如 `app.go`）收集后按名称启用。
- `plugins.go` 不注册控制器，而是集中列举所有持久卷插件子包（hostpath、nfs、fc、iscsi、csi、flexvolume），通过 `ProbeAttachableVolumePlugins` / `ProbeExpandableVolumePlugins` / `ProbePersistentVolumePlugins` 等工厂方法按接口过滤（`AttachableVolumePlugin`、`ExpandableVolumePlugin`、`ProvisionableVolumePlugin`/`DeletableVolumePlugin`/`RecyclableVolumePlugin`），并提供 `GetDynamicPluginProber` 暴露 Flexvolume 动态探测能力；`AttemptToLoadRecycler` 从文件加载回收器 Pod 模板注入到 `volume.VolumeConfig`。
- 依赖方向单向向下：本模块仅依赖 `pkg/controller/*`、`pkg/volume/*`、`apiserver/pkg/admission/plugin/policy/validating` 等实现包，自身不含业务逻辑，只承担“组装 + 注册”职责。
