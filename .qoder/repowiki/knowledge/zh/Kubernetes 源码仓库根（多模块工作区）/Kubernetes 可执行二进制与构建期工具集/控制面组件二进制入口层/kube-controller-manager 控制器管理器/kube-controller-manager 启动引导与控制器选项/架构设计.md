本模块由三层组成：
- 顶层入口 `controllermanager.go`：定义 `NewControllerManagerCommand()`（cobra 根命令）和 `Run()`，负责日志/特性门控初始化、HTTP 安全服务、事件广播、configz 注册、健康检查挂载；通过 `CreateControllerContext` 构建共享 InformerFactory、元数据 InformerFactory、DeferredDiscoveryRESTMapper 与 GC GraphBuilder，再调用 `BuildControllers` + `RunControllers` 并发拉起控制器；支持 Leader Election 与 Leader Migration（双锁：主锁运行非迁移控制器，迁移锁运行已迁移控制器），并优先以全权限客户端启动 ServiceAccount Token Controller。
- 控制器注册中心：`NewControllerDescriptors()` 汇总所有控制器描述符，`bootstrap.go` 额外注册 `newBootstrapSignerControllerDescriptor` 与 `newTokenCleanerControllerDescriptor`（均默认禁用），每个控制器提供 `*ControllerDescriptor`（含 name、aliases、constructor、isDisabledByDefault）并通过 `ControllerContext.NewClient` 获取命名 client。
- 选项层 `options/`：`options.go` 中的 `KubeControllerManagerOptions` 聚合 Generic/KubeCloudShared/各控制器 Options/Serving/AuthZ/AuthN/Metrics/Logs 等子选项，统一实现 `Flags`（按 controller name 分 flagSet）、`ApplyTo`（填充 `kubectrlmgrconfig.KubeControllerManagerConfiguration`）、`Validate`（校验并拒绝 v1.31+ 内置云厂商）、`Config`（生成最终 `config.Config`）。每个控制器对应一个 `<controller>.go` 文件，遵循相同的 `AddFlags`/`ApplyTo`/`Validate` 三方法模式，字段直接指向 `pkg/controller/apis/config` 下的内部配置结构体。
