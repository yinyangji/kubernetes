本模块位于 cmd/kube-controller-manager/app 包内，是控制面启动期对下游控制器实现进行统一包装的轻量框架：
- controller_descriptor.go 定义核心契约：Controller 接口（Name/Run）、ControllerConstructor 工厂函数、ControllerDescriptor 元数据容器（名称、构造器、requiredFeatureGates、aliases、isDisabledByDefault、requiresSpecialHandling），并通过 NewControllerDescriptors() 集中注册所有内置控制器；BuildController 在调用构造器前按 requiredFeatureGates 做 feature gate 检查并注入带 controller name 的 klog context。
- controller_utils.go 提供 runFunc/runFuncSlice 并发执行器与 newControllerLoop 适配器，让普通函数快速实现 Controller 接口，便于组合多个 goroutine 并在 ctx 取消时统一退出。
- import_known_versions.go 通过空白导入 k8s.io/kubernetes/pkg/apis/*/install 将 apps/authentication/authorization/autoscaling/batch/certificates/coordination/core/events/extensions/policy/rbac/resource/scheduling/storage 等 API Group 注册到全局 scheme，供 legacy RESTMapper 使用。
依赖方向单向向外：本模块仅依赖 apimachinery、apiserver featuregate、klog 与 component-base，不反向依赖具体控制器实现，由上层 app/cmd 通过 NewControllerDescriptors 获取并编排。
