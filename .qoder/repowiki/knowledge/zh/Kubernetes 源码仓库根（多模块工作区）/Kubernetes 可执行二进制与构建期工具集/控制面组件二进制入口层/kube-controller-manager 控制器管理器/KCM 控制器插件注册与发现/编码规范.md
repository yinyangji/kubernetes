- 每个控制器以“描述符函数 + 构造函数”成对定义在独立文件中，描述符返回 `&ControllerDescriptor{name, aliases, constructor}`，构造函数通过 `controllerContext.NewClient` 获取客户端并用 `newControllerLoop` 包装 Run 循环。
- 控制器描述符的 `name` 字段取自 `k8s.io/kubernetes/cmd/kube-controller-manager/names` 包中的常量，别名列表用于命令行短名匹配。
- 控制器启动时优先从 `controllerContext.ComponentConfig.<ControllerName>.ConcurrentXXXSyncs` 读取并发度，再传入对应控制器的 `Run(ctx, int(...))`。
- 持久卷插件通过显式 import 子包并在 `probeControllerVolumePlugins` 中调用各自的 `ProbeVolumePlugins` 进行静态注册，再通过闭包 filter 按接口类型筛选出所需子集。
