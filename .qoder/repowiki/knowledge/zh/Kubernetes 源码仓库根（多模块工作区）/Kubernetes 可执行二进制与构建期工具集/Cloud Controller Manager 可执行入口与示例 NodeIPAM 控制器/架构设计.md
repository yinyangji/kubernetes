本模块是 `cloud-controller-manager` 二进制的单一包，职责集中在三处：
- `main.go`：调用 `k8s.io/cloud-provider/app.NewCloudControllerManagerCommand` 启动框架；通过 `app.DefaultInitFuncConstructors` 获取默认控制器集合，再覆盖/新增控制器（示例中注册了 `nodeipam`）；实现 `cloudInitializer` 使用 `cloudprovider.InitCloudProvider(name, configFile)` 动态加载云厂商插件，并在未设置 ClusterID 时根据 `AllowUntaggedCloud` 拒绝或告警。
- `providers.go`：空占位文件，注释说明 out-of-tree 云厂商无需在此导入 in-tree provider，仅作为 `import _ "k8s.io/legacy-cloud-providers/<provider>"` 的参考位置。
- `nodeipamcontroller.go`：以示例形式展示如何编写外部控制器——定义 `nodeIPAMController` 结构体持有配置与选项，实现 `StartNodeIpamControllerWrapper(initContext, completedConfig, cloud) app.InitFunc` 回调，内部校验 `--cidr-allocator-type=CloudAllocator` 与 cloud provider 的关系、解析 `ClusterCIDR`/`ServiceCIDR`/`SecondaryServiceCIDR` 并做双栈一致性检查，最终委托 `pkg/controller/nodeipam.NewNodeIpamController` 创建并运行控制器。
依赖方向单向向外：本包只依赖 `k8s.io/cloud-provider/app`、`k8s.io/kubernetes/pkg/controller/nodeipam` 等上层库，不反向被其他 k8s 包引用。
