本模块由三个互不依赖的独立 `package main` 可执行程序组成，均作为 hack/build 流水线中的代码生成与校验步骤：
- `fieldnamedocscheck`：通过 `k8s.io/apimachinery/pkg/runtime.ParseDocumentationFrom` 解析指定 Go 源文件中的类型注释，用正则提取反引号包裹的字段名并与 tag 名称做大小写无关比对，不一致则 stderr 输出并 exit 1，用于 CI 强制字段文档一致性。
- `genswaggertypedocs`：同样基于 `apimachinery/pkg/runtime`，读取类型源码并通过 `WriteSwaggerDocFunc` / `VerifySwaggerDocsExist` 生成或校验 Swagger 文档函数，支持 `-f/-` 将输出重定向到文件或 stdout。
- `genfeaturegates`：通过空导入 `_ "k8s.io/kubernetes/pkg/features"`、`_ "k8s.io/apiserver/pkg/features"`、`_ "k8s.io/apiextensions-apiserver/pkg/features"` 触发各组件注册其 `FeatureGate`，再经由 `utilfeature.DefaultMutableFeatureGate.GetAllVersioned()` 与 `Dependencies()` 枚举所有版本化特性规格，按 stage（alpha/beta/ga/deprecated）计算 toVersion 区间，并以 markdown 表格或 JSON 结构输出；meta-feature `AllAlpha`/`AllBeta` 被显式跳过。

三者均为纯命令行工具，无共享包，仅依赖 `k8s.io/*` 公共库与 `spf13/pflag`/标准库，依赖方向单向指向 apimachinery/component-base/apiserver 等上层仓库。
