- CLI 入口一律通过 `os.Args` 手动解析参数而非 flags 库，以避免 flags 污染影响 man 页面生成。
- 在生成前设置 `HOME=/home/username` 环境变量，确保 kubectl 等命令的输出在不同执行环境下保持一致。
- 对需要注入上下文或 IO 流的命令构造器（如 `NewKubectlCommand`、`NewKubeadmCommand`、`NewKubeletCommand`），传入空 `bytes.Reader` 和 `io.Discard` 作为 In/Out/ErrOut，避免交互式行为。
- 通过 `switch module` 分发不同组件，新增组件时需在 genkubedocs 与 genman 两处同步添加分支。
