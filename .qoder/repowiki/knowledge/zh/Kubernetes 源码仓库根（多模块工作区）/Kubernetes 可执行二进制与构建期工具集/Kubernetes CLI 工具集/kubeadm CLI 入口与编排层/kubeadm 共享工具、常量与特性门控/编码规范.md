- 平台相关实现通过 `_unix.go` / `_windows.go` 后缀的文件对同一包进行编译期选择，避免 if runtime.GOOS 分支。
- 错误统一经 `app/util/errors` 包装后返回，外层用 `errors.Wrapf` 附加上下文，保持错误链可读。
- 对外暴露的“已知集合”以包级 `var` 或 `const` 形式集中声明（如 `SupportedEtcdVersion`、`InitFeatureGates`、`defaultKnownCRISockets`），并由 getter 函数访问，禁止散落在业务逻辑中。
- 需要重试的操作围绕 `constants.PullImageRetry` / `RemoveContainerRetry` 等常量循环，并在每次失败后记录 klog 日志，最后聚合为 `errorsutil.NewAggregate`。
- 配置反序列化为先走 `scheme.Default` 填充默认值，再用 `strict.VerifyUnmarshalStrict` 校验未知/重复字段，随后才 DecodeInto 内部类型。
- 测试数据统一放在子目录 `testdata/` 下，配合 `testing/` 子包提供 fake 实现，便于单元测试隔离外部依赖。
