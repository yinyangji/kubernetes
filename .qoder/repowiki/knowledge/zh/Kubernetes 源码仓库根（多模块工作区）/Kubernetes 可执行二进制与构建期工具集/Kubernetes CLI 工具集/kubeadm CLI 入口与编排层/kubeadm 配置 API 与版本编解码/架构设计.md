本模块按 Kubernetes API 约定组织为 `GroupName=kubeadm.k8s.io` 下的多版本子包：
- 顶层 `kubeadm/` 暴露内部组 `SchemeGroupVersion=runtime.APIVersionInternal`，仅包含不可序列化的共享类型（如 `InitConfiguration` 中嵌入的私有结构）和 SchemeBuilder；对外通过 `scheme/scheme.go` 聚合所有版本，设置 `SetVersionPriority(v1beta4, v1)` 使 v1 成为最高优先写入版本。
- `kubeadm/v1beta4/` 与 `kubeadm/v1/` 各自持有同构的 `InitConfiguration`、`ClusterConfiguration`、`JoinConfiguration`、`ResetConfiguration`、`UpgradeConfiguration` 等可序列化类型，并通过 `zz_generated.deepcopy.go` / `zz_generated.conversion.go` / `zz_generated.defaults.go` 由 code-generator 生成 DeepCopy 与互转逻辑；`conversion.go` 中的手写转换函数用于处理内部私有结构（如 `kubeadm.InitConfiguration` ↔ `v1.InitConfiguration`）之间的桥接。
- `kubeadm/validation/` 提供平台无关的验证器，按 OS 拆分为 `util_unix.go` / `util_windows.go`，由上层命令在解析后调用。
- `bootstraptoken/v1/` 是独立的小 API 组，被 kubeadm 各版本类型引用（如 `BootstrapTokens []bootstraptokenv1.BootstrapToken`），实现跨版本的 token 结构复用。
- `kubeadm/fuzzer/` 基于 apimachinery 的 fuzzer 框架生成随机输入，配合 `fuzzer_test.go` 做回归测试。
- `argument.go`、`apiendpoint.go`、`timeoututils.go` 提供对 `[]Arg`、`APIEndpoint`、超时处理的纯函数式工具，不依赖 scheme。
依赖方向：`v1`/`v1beta4` → `kubeadm`（内部类型）→ `bootstraptoken/v1`；`scheme/scheme.go` 是唯一汇总入口，其他包只 import 具体版本或内部包，避免循环。
