- 每个工作流目录（init/join/reset/upgrade）在自身 data.go 中定义一个 `Data` 接口，phase 的 Run 函数通过类型断言消费该接口，避免直接引用父级结构体。
- 每个 Data 接口都配套一个同包的 `testData{}` 或 `testInitData{}` 空实现，并用 `var _ Interface = &testXxxData{}` 编译期校验满足关系。
- phase 通过 `workflow.Phase{Name:..., Run: func(data workflow.RunData) error {...}}` 字面量声明并挂入 Runner，而非单独编写 cobra Command。
- 公共 flag 通过 `options.Add*Flag` 系列函数集中注册，phase 只调用这些装配器而不自行重复定义 flag 字符串。
- 需要强制子命令的父 cobra 命令统一调用 `cmdutil.RequireSubcommand(cmd)` 设置空 RunE 并返回 `ErrorSubcommandRequired`。
- phase 间依赖通过 `Dependencies []string` 声明，由 `Runner.computePhaseRunFlags` 在执行前校验未解析依赖并汇总报错。
