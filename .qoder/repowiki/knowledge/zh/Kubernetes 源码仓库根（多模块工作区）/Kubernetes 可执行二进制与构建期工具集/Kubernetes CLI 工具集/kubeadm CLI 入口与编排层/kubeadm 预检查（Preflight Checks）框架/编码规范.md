- 每个检查以 struct 形式实现 `Checker` 接口，`Name()` 返回人类可读标签，未显式设置时按 `Type-Path/Port` 模式自动生成。
- `Check()` 方法统一返回 `(warnings, errorList []error)` 二元组，warning 表示可继续执行的告警，error 表示阻断性失败。
- 平台相关行为通过 `//go:build <os>` 文件拆分，并以 `addOSValidator`、`addExecChecks`、`addIPv4Checks`、`addIPv6Checks`、`addSwapCheck` 等钩子函数注入，避免在通用代码中使用 runtime.GOOS 分支。
- 对外部 I/O（exec.LookPath、容器运行时、HTTP 请求等）通过结构体字段暴露可插桩接口（如 `exec utilsexec.Interface`、`impl utilruntime.Impl`、`listenFunc`），便于单元测试替换。
- 错误信息统一经 `k8s.io/kubernetes/cmd/kubeadm/app/util/errors` 包装后返回，保持消息风格一致。
