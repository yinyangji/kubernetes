本模块由四个并列子包组成，共同构成 kubeadm 上层命令（init/join/upgrade/reset）的底层支撑层，依赖方向单向向下：
- `constants`：集中声明所有路径、端口、组件名、默认版本、超时、标签/注解键等“硬编码”常量，并通过 `getSkewedKubernetesVersion` 基于构建期注入的 `component-base/version` 派生当前/最小支持版本；按平台拆分 `constants_unix.go` / `constants_windows.go`。
- `features`：基于 `k8s.io/component-base/featuregate` 实现 kubeadm 自有 FeatureGate（如 `RootlessControlPlane`），提供 `FeatureList`、`NewFeatureGate`、`ValidateVersion`、`CheckDeprecatedFlags` 等 API，供 init/join 命令在解析 flag 时校验兼容性。
- `images`：根据 `ClusterConfiguration` 与 `constants.SupportedEtcdVersion` 拼装 control-plane、CoreDNS、etcd、pause 等镜像完整引用，是 `kubeadm config images pull/list` 的核心逻辑。
- `util`：最丰富的共享库，按功能拆分子目录：
  - `config/` + `config/strict/`：负责从 ConfigMap/kubelet 配置文件反序列化 Init/Join/Reset/UpgradeConfiguration，并调用 `strict.VerifyUnmarshalStrict` 做未知字段/重复字段校验；`common.go` 提供动态默认值填充。
  - `apiclient/`：封装带重试/超时的 API 调用、幂等写入、条件等待（`wait.go`、`idempotency.go`、`dryrun.go`）。
  - `runtime/`：定义 `ContainerRuntime` 接口，通过 CRI gRPC 连接 containerd/crio/docker，`fake_impl.go` 用于测试替换。
  - `staticpod/`：构造 system-node-critical 优先级的控制面 Pod 模板，结合 `patches/` 应用 StrategicMergePatch，再写盘到 `/etc/kubernetes/manifests`。
  - `pkiutil/`、`crypto/`、`pubkeypin/`：证书签发、公钥指纹校验。
  - `kubeconfig/`、`users/`、`initsystem/`、`files/`、`endpoint.go`、`arguments.go`、`template.go`、`version.go`、`env.go`：分别处理 kubeconfig 读写、用户/组创建、systemd 交互、文件复制、端点解析、命令行参数合并、Go template 渲染、版本转换与环境变量。

依赖关系：`images` 依赖 `constants`；`config` 依赖 `constants`、`apiclient`、`strict`、`kubeconfig`、`output`、`runtime`；`staticpod` 依赖 `constants`、`patches`、`users`、`util` 顶层函数；`util/runtime` 依赖 `constants` 中的 CRI socket 列表；`util/apiclient` 依赖 `errors`。整个模块不直接依赖任何 kubeadm 命令入口，仅被 `cmd/kubeadm/app/phases` 与 `cmd/kubeadm/app/cmd` 消费。
