模块以 `Checker` 接口为核心抽象（`Check() (warnings, errorList []error)` + `Name() string`），每个具体检查实现为一个结构体类型，通过返回 warnings/errorList 区分可恢复警告与阻断性错误。

- 通用检查逻辑集中在 `checks.go`：包括 CRI 连接与版本兼容 (`ContainerRuntimeCheck` / `ContainerRuntimeVersionCheck`)、端口占用 (`PortOpenCheck`)、文件/目录存在性与内容 (`FileAvailableCheck` / `DirAvailableCheck` / `FileExistingCheck` / `FileContentCheck`)、PATH 中二进制 (`InPathCheck`)、主机名与 DNS 解析 (`HostnameCheck`)、HTTP/代理连通性 (`HTTPProxyCheck` / `HTTPProxyCIDRCheck`)、etcd 外部版本 (`ExternalEtcdVersionCheck`)、镜像拉取 (`ImagePullCheck`)、CPU/内存 (`NumCPUCheck` / `MemCheck`)、内核系统验证 (`SystemVerificationCheck`)、Kubernetes/kubelet 版本 (`KubernetesVersionCheck` / `KubeletVersionCheck`)、swap 状态 (`SwapCheck`)、服务启用状态 (`ServiceCheck`) 与 firewalld 检测 (`FirewalldCheck`) 等。
- 平台差异通过 Go build tag 拆分到 `checks_linux.go`、`checks_windows.go`、`checks_darwin.go`、`checks_other.go`，由 `addOSValidator`、`addIPv4Checks`、`addIPv6Checks`、`addSwapCheck`、`addExecChecks` 等函数注入平台相关检查项；`MemCheck` 在 Linux 上调用 `syscall.Sysinfo`，在 Windows 上为空操作。
- `InitNodeChecks` 是入口装配器，根据 `InitConfiguration` 组装完整的 Checker 列表，并在末尾调用各 `add*` 钩子完成平台扩展。
- 辅助能力位于 `utils.go`（如 `GetKubeletVersion` 通过执行 `kubelet --version` 解析语义版本）。
- 依赖方向单向向外：仅依赖 `cmd/kubeadm/app/util/*`、`k8s.io/system-validators`、`k8s.io/utils/exec`、`k8s.io/apimachinery/pkg/util/version` 等基础库，不反向依赖上层命令层。
