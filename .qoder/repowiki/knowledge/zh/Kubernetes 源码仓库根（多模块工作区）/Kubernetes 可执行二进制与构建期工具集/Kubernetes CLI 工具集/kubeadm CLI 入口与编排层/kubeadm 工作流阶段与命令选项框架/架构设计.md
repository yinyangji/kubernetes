本模块由三个协作子包构成，围绕 `workflow` 提供的可组合阶段引擎组织：
- `phases/workflow`：核心抽象。`Phase` 结构体描述一个阶段的元信息（Name/Aliases/Short/Long/Example/Hidden）、可选的 Run/RunIf 函数、依赖声明、父子嵌套 Phases 列表及 pflag.FlagSet 本地标志；`Runner` 负责把一组 Phase 构造成带层级路径（以 `/` 分隔）的命令树，支持 `--skip-phases` / `--filter-phases` 过滤、依赖预检查、按顺序遍历执行，并通过 `BindToCommand` 把每个 leaf phase 注册为 cobra 子命令。
- `phases/{init,join,reset,upgrade}`：具体工作流的阶段实现。每个顶层目录对应一个 kubeadm 子命令族，内部按职责拆分为多个文件（如 `preflight.go`、`certs.go`、`controlplane.go`、`kubeconfig.go`、`uploadconfig.go` 等），并在各自的 `data.go` 中定义面向该工作流的 `Data` 接口（例如 `InitData`、`upgrade.Data`），phase 的 `Run` 通过类型断言消费该接口，从而将“配置 + 运行时状态”与“阶段逻辑”解耦。
- `cmd/options`：集中暴露 `AddKubeConfigFlag`、`AddIgnorePreflightErrorsFlag`、`AddControlPlanExtraArgsFlags`、`AddPatchesFlag` 等 flag 装配器，供各 phase 复用；`argslice.go` 提供自定义 `pflag.Value` 解析 `key=value` 形式的 ExtraArgs。
- `cmd/util`：通用 cobra 辅助（`RequireSubcommand`、`ValidateExactArgNumber`、`GetKubeConfigPath`、`InteractivelyConfirmAction`、`ValueFromFlagsOrConfig`）和文档生成（`documentation.go`）。
依赖方向：`phases/*` → `workflow` → `options` / `util`；各 phase 实现不直接依赖上层 `app/cmd/*.go`，而是通过 `data.go` 中的接口反向被父级组装，形成松耦合的可插拔阶段模型。
