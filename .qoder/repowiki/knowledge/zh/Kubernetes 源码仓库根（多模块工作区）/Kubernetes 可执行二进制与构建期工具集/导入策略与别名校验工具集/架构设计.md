本模块由三个互不依赖的独立 `main` 程序组成，各自承担一种导入治理职责：
- `cmd/import-boss/main.go`：核心引擎。通过 `golang.org/x/tools/go/packages` 加载目标包及其依赖图，递归向上查找 `.import-restrictions`（YAML）规则文件至 `go.mod` 根目录；对每个包的直接/间接导入执行正向规则匹配（`Rules`），并基于反向依赖图（Warshall 传递闭包）执行逆向规则匹配（`InverseRules`），支持 `Transitive` 控制是否作用于间接依赖。
- `cmd/importverifier/importverifier.go`：轻量验证器。调用外部 `go list -json` 解析包树，依据 `ImportRestriction`（`baseImportPath` + `ignoredSubTrees` + `allowedImports`）判断某包下的所有导入是否落入禁止集合，仅做“允许白名单”式检查。
- `cmd/preferredimports/preferredimports.go`：别名规范化器。使用 `go/parser` + `go/ast` 遍历源码，按 `-import-aliases` JSON 中的正则模板替换 import alias，并在 `-confirm` 模式下回写 AST 格式化后的文件。
三者均遵循“CLI → 解析配置/包 → 遍历导入 → 输出错误并 exit 1”的统一流程，但分别采用 packages API、`go list` 子进程、AST 三种不同的包发现手段。
