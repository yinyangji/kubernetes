本模块为纯静态产物目录，不包含可执行代码，按产出物类型分为三层：
- `api/openapi-spec/`：由 `hack/update-codegen.sh` 生成的 OpenAPI v3 规范（`v3/` 下按 group/version 拆分到独立 `_openapi.json` 文件，顶层保留聚合版 `swagger.json`），并附带自定义 vendor extension（`x-kubernetes-group-version-kind`、`x-kubernetes-action`、`x-kubernetes-list-map-keys`、`x-kubernetes-patch-strategy` 等）的说明。
- `api/discovery/`：API Server Discovery 端点返回的 JSON 快照（`apis.json`、`api.json`、各 group 及 version 子集），用于客户端快速枚举可用 API 组与版本。
- `api/api-rules/`：kube-openapi 规则检查产出的已知违规清单（`violation_exceptions.list` 及各子包例外），通过 Make 目标在生成阶段比对，阻止引入新违规；新增/移除条目需显式提交而非隐藏在“生成变更”中。
- `docs/kubernetes-internals-deep-dive.md`：社区编写的中文深度解析文档，不属于构建产物。
依赖方向单向：上游 Go 源码 + kube-openapi 生成器 → 本模块下的 JSON/Markdown 文件，本模块不反向依赖任何源码。
