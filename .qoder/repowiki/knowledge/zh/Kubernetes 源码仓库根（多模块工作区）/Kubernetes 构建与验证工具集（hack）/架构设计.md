目录按职责分层组织：
- `lib/` 为纯 Bash 共享库（init.sh 统一设置 KUBE_ROOT/KUBE_OUTPUT、source util/logging/version/golang/etcd/protoc/test），所有顶层脚本通过 `source hack/lib/init.sh` 复用，避免重复逻辑。
- `make-rules/*.sh` 是 make 目标对应的实现（build/cross/test/update/verify 等），由根 Makefile 调用；顶层 `verify-all.sh` / `update-all.sh` 仅做重定向到 `make verify|update`。
- 顶层大量 `verify-*.sh` / `update-*.sh` 是一维扁平编排脚本，每个负责单一检查或生成任务（如 boilerplate、gofmt、golangci-lint、openapi、codegen、vendor、licenses 等），依赖关系通过 shell 顺序而非显式模块系统表达。
- `tools/` 下嵌入独立的 Go 工作区（go.mod/go.sum/go.work），包含 golangci-lint 自定义插件、metrics instrumentation、traceparse、publishing-verifier 等可编译二进制，由 `update-tools.sh` 安装。
- `jenkins/` 封装 Docker 化执行环境（test-dockerized.sh 等），供 CI 以容器方式运行相同脚本。
- `e2e-internal/` 提供集群扩缩容、状态查询等 e2e 辅助脚本；`testdata/` 存放各子命令的 YAML/JSON 输入样例。
- 依赖方向单向：顶层脚本 → `make-rules/*` → `lib/*`；`tools/` 与 `jenkins/` 不反向依赖顶层脚本。
