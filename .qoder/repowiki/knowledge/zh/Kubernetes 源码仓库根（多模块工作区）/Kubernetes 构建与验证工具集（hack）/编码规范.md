- 每个脚本顶部统一声明 `set -o errexit; set -o nounset; set -o pipefail`，并通过 `KUBE_ROOT=$(dirname "${BASH_SOURCE[0]}")/..` 定位仓库根。
- 共享函数采用 `kube::<namespace>::<function>` 命名空间前缀（util/log/golang/test 等），通过 `source hack/lib/init.sh` 一次性加载。
- 顶层 `verify-*.sh` / `update-*.sh` 保持单职责：只做参数解析与调用，具体实现下沉到 `make-rules/*.sh` 或 `lib/*.sh`。
- 需要外部工具时先 `command -v` 探测并给出 `kube::log::usage` 提示，失败直接 exit 1，禁止静默降级。
- 输出统一走 `kube::log::status/error/fatal` 等日志函数，不使用裸 echo。
- Makefile 入口脚本（verify-all.sh、update-all.sh）仅打印迁移提示并重定向到 `make verify|update`，不再承载业务逻辑。
