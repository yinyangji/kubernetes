- cobra 子命令采用 NewXxxOptions + NewCmdXxx(f, ioStreams) 工厂模式，Run 回调中顺序调用 Complete 与业务方法，错误统一由 cmdutil.CheckErr 包装。
- 通过空白导入 _ ".../install" 的方式注册 API group Scheme，而非显式构造 Scheme，确保所有已知 GVK 可用。
- 对外部可插拔组件（Builder、Validator、Printer）一律通过 Factory 或 PrintFlags 注入，避免在命令内部硬编码实例化。
- 对 NotRegisteredError 采取“警告并继续”策略（写入 ErrOut 后 continue），保证单个对象转换失败不影响整批文件处理。
