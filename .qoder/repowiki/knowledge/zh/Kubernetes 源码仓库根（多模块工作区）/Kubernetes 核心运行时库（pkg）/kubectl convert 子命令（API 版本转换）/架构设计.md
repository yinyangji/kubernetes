本模块是 kubectl 的 convert 子命令实现，位于 pkg/kubectl/cmd/convert 包内，遵循 cobra + options 模式：
- convert.go 定义 ConvertOptions 结构体与 NewCmdConvert/NewConvertOptions 工厂函数，通过 Complete → RunConvert 两阶段执行；使用 cmdutil.Factory 注入 resource.Builder、Validator 等依赖，并通过 genericclioptions.PrintFlags 统一输出。
- import_known_versions.go 以空白导入方式注册所有内置 API group 的 Scheme，使 scheme.Scheme 具备各 GVK 的编解码能力；同时显式引入 legacy extensions/v1beta1 并置于末尾，保证新版本优先（见注释引用 issue #112682）。
- 核心转换流程：resource.Builder 读取输入 → 内部版本对象 → tryConvert 依次尝试指定 output-version 或该 Group 的所有 PrioritizedVersions → 单对象直接返回，多对象包装为 api.List → printers.ResourcePrinter 输出。
- 对不在 api.Scheme 中的对象走 runtime.Unknown 回退路径（先编码为 JSON 再包裹 Unknown），避免强依赖。
- 顶层 doc.go 声明该目录为 kubectl 工具函数的测试友好型组织；.import-restrictions 白名单仅允许本包访问 pkg/apis/*/install 与 legacyscheme，防止随意引入其他 pkg 代码。
