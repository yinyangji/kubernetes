本模块按 API Group 垂直切分为 pkg/apis/<group>/ 子包，每个子包遵循统一结构：顶层 types.go + register.go 声明内部对象与 SchemeGroupVersion，vX/ 子目录提供外部版本 Go 类型及 zz_generated.conversion.go / defaults.go / validations.go；install/ 子包通过 import 副作用将各版本注册到全局 runtime.Scheme。pkg/api/ 则不定义新 API 组，而是集中放置跨资源的通用 helper（pod/service/pv/pvc/node/storage/resourceclaim 的 util.go）、legacy scheme（legacyscheme/scheme.go 暴露 Scheme/Codecs/ParameterCodec）以及测试基础设施（testing/install.go 一次性 _import 所有 install 包以在单测中完成全量注册）。各 child 之间无直接依赖，唯一耦合点是共享 apimachinery/runtime.Scheme 与统一的 zz_generated.* 生成产物约定，由上层 kube-apiserver 启动时串联这些 install 包完成全局注册。
