模块按 Kubernetes API 约定分层：
- `types.go` 声明核心结构体（`APIGroupDiscoveryList`、`APIGroupDiscovery`、`APIVersionDiscovery`、`APIResourceDiscovery`、`APISubresourceDiscovery`）以及枚举 `ResourceScope`、`DiscoveryFreshness`，并通过 `+k8s:deepcopy-gen` 注解生成 `zz_generated.deepcopy.go`。
- 根包 `register.go` 以 `SchemeGroupVersion = apidiscovery.k8s.io/internal` 注册内部版本对象到 scheme，供 apiserver 内部使用。
- `v2/` 与 `v2beta1/` 子包分别将 `apidiscovery.k8s.io/v2` 和 `apidiscovery.k8s.io/v2beta1` 的 `SchemeBuilder` 暴露为本地 `AddToScheme`，自身不重复定义类型，仅做版本别名与 Scheme 转发；每个版本目录还包含 `doc.go`、`zz_generated.conversion.go`、`zz_generated.defaults.go`、`zz_generated.validations.go`，由 code-generator 从外部 `k8s.io/api/apidiscovery` 包同步生成。
- 依赖方向单向：版本子包 → `k8s.io/api/apidiscovery/<version>`（只读引用其 `SchemeBuilder`），根包 → `apimachinery/pkg/runtime` 与 `schema`，无反向依赖，保证该 leaf 模块仅承载类型与注册逻辑。
