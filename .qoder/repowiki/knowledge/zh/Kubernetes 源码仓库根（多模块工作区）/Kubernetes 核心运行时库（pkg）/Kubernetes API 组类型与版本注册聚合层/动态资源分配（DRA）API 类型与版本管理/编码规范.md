- 每个对外版本子包都遵循相同的文件骨架：`doc.go` 声明 generator 注释、`register.go` 导出 `AddToScheme` 并注册手动编写的 `addDefaultingFuncs`/`addConversionFuncs`、`conversion.go`/`defaults.go` 放置手写逻辑、`zz_generated.*.go` 由 codegen 产出，以便缺少生成文件时仍可编译。
- 版本号通过 `schema.GroupVersion{Group: GroupName, Version: "vX"}` 常量表达，并在 `install/install.go` 中以 `SetVersionPriority` 显式排序，稳定版（v1）排在最前。
- 校验函数成对出现：`ValidateXxx` 用于创建路径、`ValidateXxxUpdate` 用于更新路径、`ValidateXxxStatusUpdate` 用于 status 更新，分别复用 `corevalidation.ValidateObjectMeta*` 与 `apimachineryvalidation.ValidateImmutableField`。
- oneOf 互斥字段通过收集已设置字段名到 `setFields` 切片，再根据长度分支返回 Required/Invalid 错误，而非逐个字段单独报错。
- 大小与唯一性校验统一封装在 `validateSet`/`validateSlice`/`validateMap` 辅助函数中，并以 `sizeCovered`、`uniquenessCovered` 等标记配合 declarative validation 覆盖。
- feature gate 控制的字段在类型定义处用 `// +featureGate=...` 注解标注，校验侧通过 `utilfeature.DefaultFeatureGate.Enabled(features.XXX)` 条件化执行额外约束。
