模块按 Kubernetes API 标准分层组织：
- 根包 `pkg/apis/resource` 存放内部（internal）类型的完整定义（`types.go`）、Group/Version 常量与 Scheme 注册（`register.go`，`SchemeGroupVersion = resource.k8s.io/internal`），并通过 `zz_generated.deepcopy.go` 提供 deepcopy。
- 每个子目录 `v1alpha3`、`v1beta1`、`v1beta2`、`v1` 对应一个对外暴露的 API 版本，其中 `v1` 是 stable 版本；这些版本的 Go 类型实际位于外部依赖 `k8s.io/api/resource/<version>`，本模块仅通过 `conversion.go`、`defaults.go`、`zz_generated.*.go` 提供跨版本转换、默认值注入和字段标签转换（如 v1beta2 对 ResourceSlice 的 field label 映射）。
- `install/install.go` 作为统一入口，在 `init()` 中调用 `Install(legacyscheme.Scheme)`，将 internal + 所有版本注册到 legacy scheme，并用 `SetVersionPriority(v1, v1beta2, v1beta1, v1alpha3)` 指定优先级。
- `validation/` 包集中实现所有资源的业务校验（命名、互斥 oneOf、size 上限、CEL 表达式编译与成本限制、状态更新不变性等），被 apiserver 的 declarative validation 与 webhook 共同消费。
- `fuzzer/fuzzer.go` 为 fuzzing 测试生成随机对象。
依赖方向：`install` → 各版本包 → `resource` internal 包；`validation` 反向依赖 internal 包以访问常量与结构体；各版本包不直接依赖 `validation`，由上层 apiserver 组合调用。
