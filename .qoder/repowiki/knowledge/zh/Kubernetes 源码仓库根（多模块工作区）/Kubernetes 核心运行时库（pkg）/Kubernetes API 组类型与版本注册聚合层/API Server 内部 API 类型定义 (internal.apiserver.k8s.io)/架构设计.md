模块按 Kubernetes API 包标准分层组织：
- 根目录 `types.go` + `register.go` + `doc.go` 声明内部版本 (`runtime.APIVersionInternal`) 的 `StorageVersion`/`StorageVersionList` 及其 status、condition 结构体，并通过 `+k8s:deepcopy-gen` 注解生成 `zz_generated.deepcopy.go`。
- `v1alpha1/` 子包作为对外暴露的稳定版本，通过代理方式复用 `k8s.io/api/apiserverinternal/v1alpha1` 的 SchemeBuilder，并生成转换/默认值/校验代码（`zz_generated.conversion.go`、`zz_generated.defaults.go`、`zz_generated.validations.go`）。
- `validation/` 提供 `ValidateStorageVersion*` 系列函数，对 name 格式（`<group>.<resource>`）、apiServerID、encoding/decodable/served versions 一致性以及 conditions 唯一性做运行时校验。
- `install/install.go` 在 `init()` 中把 internal 与 v1alpha1 注册到 `legacyscheme.Scheme`，并把 v1alpha1 设为版本优先级；`fuzzer/fuzzer.go` 为 fuzz 测试提供随机对象构造。
依赖方向：`install` → `apiserverinternal` + `v1alpha1`；`validation` 仅依赖根包类型；`v1alpha1` 反向引用 `k8s.io/api/apiserverinternal/v1alpha1` 以复用其 scheme 注册逻辑。
