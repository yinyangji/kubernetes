模块遵循 kubernetes 标准 API group 分层：
- 顶层 `pkg/apis/discovery` 声明内部 GroupName、SchemeGroupVersion（指向 `runtime.APIVersionInternal`）、`Kind`/`Resource` 辅助函数以及 `addKnownTypes`，将 `EndpointSlice`、`EndpointSliceList` 注册到内部 scheme。
- `types.go` 是领域模型唯一来源，包含 `EndpointSlice`、`Endpoint`、`EndpointConditions`、`EndpointHints`、`EndpointPort` 等结构体及 `AddressType` 常量；通过 `+k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object` 注解暴露 runtime.Object 接口。
- `v1/` 与 `v1beta1/` 子包分别绑定外部版本 `discovery.k8s.io/v1` 和 `/v1beta1`，各自持有 `SchemeBuilder`、`GroupName`、`SchemeGroupVersion` 并通过 `init()` 仅注册手写默认值函数，其余由 `zz_generated.*` 生成文件完成。
- `install/install.go` 作为入口在 `legacyscheme.Scheme` 上调用 `Install`，按顺序注册 internal → v1beta1 → v1，并用 `SetVersionPriority(v1, v1beta1)` 指定首选版本。
- `fuzzer/` 提供随机数据生成器供 fuzzing 测试使用；`validation/` 集中实现字段级校验逻辑；各版本自带 `defaults.go`/`defaults_test.go`/`conversion.go`/`conversion_test.go` 处理默认值与跨版本转换。
- 依赖方向单向：`v1`/`v1beta1` 反向引用 `k8s.io/api/discovery/v1` 与 `k8s.io/api/discovery/v1beta1` 的 SchemeBuilder，顶层 `types.go` 仅依赖 `apimachinery` 与 `pkg/apis/core`，不依赖任何具体版本子包。
