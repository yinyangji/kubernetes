本模块是 k8s.io/kubernetes 内部的 policy API 组实现，遵循 apimachinery 的“外部类型 + 内部类型”双包模式：
- 顶层 `pkg/apis/policy` 存放**内部运行时类型**（`types.go` 中的 `PodDisruptionBudget`/`PodDisruptionBudgetSpec`/`PodDisruptionBudgetStatus`/`Eviction`），通过 `register.go` 暴露 `GroupName="policy"`、`SchemeGroupVersion=runtime.APIVersionInternal`、`Kind()`/`Resource()` 以及 `SchemeBuilder`/`AddToScheme`。
- `v1/` 与 `v1beta1/` 子包仅作为**转换层**，其 `doc.go` 使用 `+k8s:conversion-gen-external-types=k8s.io/api/policy/vX` 指向 `k8s.io/api/policy` 下的真实 CRD 类型，并通过 `zz_generated.conversion.go` 生成双向转换函数；每个版本包各自维护 `SchemeGroupVersion`（`v1`/`v1beta1`）和 `RegisterDefaults`。
- `install/install.go` 在 `init()` 中把 internal、v1beta1、v1 全部注册到 `legacyscheme.Scheme`，并显式设置 `SetVersionPriority(v1, v1beta1)`，保证 v1 为默认版本。
- `validation/validation.go` 对内部类型的 spec/status 做业务校验（minAvailable/maxUnavailable 互斥且为 0~100%、selector 合法性、`UnhealthyPodEvictionPolicy` 白名单等），并对 v1beta1 做降级兼容。
- `helper.go` 提供 v1beta1→v1 迁移时用于 selector 语义对齐的辅助常量与 `StripPDBV1beta1Label`。
- `fuzzer/fuzzer.go` 由 go-fuzz 使用，基于内部类型生成随机 PDB/Eviction 对象。
依赖方向单向：`install → v1/v1beta1 → api/policy/internal types`，不反向依赖上层 `k8s.io/api/policy`。
