- 每个外部版本子包都遵循固定文件布局：`register.go` 绑定外部 `SchemeBuilder`，`conversion.go` 手写双向转换，`defaults.go` 写手动默认值，`zz_generated.*` 由代码生成器产出，且 `init()` 中仅注册手写函数以保证缺少生成文件时仍可编译。
- 对外部版本的转换函数统一采用 `Convert_autoscaling_Xxx_To_vN_Yyy` / `Convert_vN_Yyy_To_autoscaling_Xxx` 命名模式，并在入口调用 `autoConvert_*` 生成的辅助函数后再执行手写逻辑。
- HPA 在 v1↔internal 转换时使用 `DropRoundTripHorizontalPodAutoscalerAnnotations` 清理 round-trip 注解，再按需写入 `MetricSpecsAnnotation`、`BehaviorSpecsAnnotation`、`ToleranceScaleDownAnnotation`、`ToleranceScaleUpAnnotation`、`MetricStatusesAnnotation`、`HorizontalPodAutoscalerConditionsAnnotation` 等专用注解以承载 v1 无法表达的新字段。
- 所有错误均通过 `field.ErrorList` 累积并以 `.MarkCoveredByDeclarative()` 标注是否已由声明式校验覆盖，便于上层 REST 层合并结果。
- 常量枚举（如 `ScalingPolicySelect`、`MetricSourceType`、`MetricTargetType`）以 Go 字符串常量形式定义，并在 `validation` 包中以 `sets.NewString(...)` 构建白名单用于 `NotSupported` 校验。
