本模块是 kubernetes 内部对 `extensions` 实验性 API 组的历史兼容层，按以下层次组织：
- 顶层 `pkg/apis/extensions`：声明 `GroupName="extensions"`、`SchemeGroupVersion`（使用 `runtime.APIVersionInternal`）、`Kind/Resource` 辅助函数以及 `SchemeBuilder.AddToScheme`；`addKnownTypes` 将实际业务类型从 `apps`、`networking`、`autoscaling` 三个子包重新注册到 extensions 组下，作为向后兼容的别名。
- `install/install.go`：在 `init()` 中调用 `Install(legacyscheme.Scheme)`，把 extensions 内部版本和 `v1beta1` 都加入 legacy scheme，并设置 `v1beta1` 为最高优先级版本。
- `v1beta1/`：对外暴露的稳定版本目录，包含 `register.go`（注册 `SchemeGroupVersion=extensions/v1beta1`）、`defaults.go`（通过 `SetDefaults_*` 函数配合 `zz_generated.defaults.go` 注入默认值）、`conversion.go`（手写转换逻辑，桥接 `k8s.io/api/extensions/v1beta1` 与内部 `pkg/apis/apps|networking|autoscaling` 类型），以及 `zz_generated.conversion.go`（由 deepcopy-gen 生成的自动转换）。`fuzzer/fuzzer.go` 仅返回空 fuzzer 列表，供测试框架发现。
依赖方向：`extensions` → `apps` / `networking` / `autoscaling`（反向引用以完成别名注册），`v1beta1/conversion.go` 同时依赖外部 `k8s.io/api/extensions/v1beta1` 与内部各子包，实现双向编解码桥接。
