模块遵循 k8s.io/apimachinery 标准 API 包分层：
- 根包 `abac` 仅承载内部不可变结构体 `Policy`/`PolicySpec`，通过 `+k8s:deepcopy-gen=package` 注解生成 DeepCopy，并维护独立 Scheme（带 TODO 注释表明未来应移除）。
- `v0`、`v1beta1` 子包分别暴露对外 JSON 序列化版本，字段随版本演进（v0 缺少 APIGroup/NonResourcePath；v1beta1 引入这些字段并将所有字段标记为 optional）。
- 每个版本包包含 `register.go`（注册 GroupVersion）、`conversion.go`（手写 Convert_*_To_abac_Policy 将版本对象归一化到内部 abac.Policy）、以及 `zz_generated.*` 生成的 deepcopy/conversion/model_name 代码。
- `latest/latest.go` 通过空白导入同时初始化 abac、v0、v1beta1 三个包的 init，使外部只需 import latest 即可获得全部版本编解码能力。
- 依赖方向单向：各版本 → 内部 abac 包；latest 聚合所有版本；abac 根包不反向依赖任何版本包。
