模块按 Kubernetes API group 标准分层组织：
- 顶层 `types.go` + `register.go` 声明共享的枚举常量（`ScopeType`、`FailurePolicyType`、`MatchPolicyType`、`SideEffectClass`、`OperationType` 等）以及 `SchemeGroupVersion = admissionregistration.k8s.io/internal`，并通过 `addKnownTypes` 注册所有 Kind；`doc.go` 通过 `+groupName=admissionregistration.k8s.io` 和 `+k8s:deepcopy-gen=package` 驱动代码生成。
- 三个版本子包 `v1`、`v1beta1`、`v1alpha1` 各自包含 `register.go`（向 scheme 注册本版本）、`defaults.go`（`SetDefaults_*` 注入默认值）、`zz_generated.*`（由 deepcopy/conversion/default/validation 生成器产出），其中 `v1/conversion.go` 显式实现与外部 `k8s.io/api/admissionregistration/v1` 之间 `RuleWithOperations` 的 copy-only 转换桥接，解决 build tag 导致的不可见问题。
- `install/install.go` 在 `init()` 中调用各版本的 `AddToScheme`，并设置版本优先级 `v1 > v1beta1 > v1alpha1`，供 apiserver 启动时自动注册。
- `validation/` 是纯函数式校验层，不依赖任何版本 sub-package 的类型，仅消费顶层 internal types；它集中维护 `supportedFailurePolicies`、`supportedMatchPolicies`、`supportedSideEffectClasses`、`supportedOperations`、`validValidationActions` 等白名单集合，并提供 `ValidateValidatingWebhookConfiguration` / `ValidateMutatingWebhookConfiguration` / `ValidateValidatingAdmissionPolicy` / `ValidateValidatingAdmissionPolicyBinding` / `ValidateMutatingAdmissionPolicy` / `ValidateMutatingAdmissionPolicyBinding` 等入口，同时通过 `preexistingExpressions` 缓存已编译的 CEL 表达式以复用编译器实例。
- `fuzzer/fuzzer.go` 为 fuzzing 提供随机对象生成器。
依赖方向：`install` → 各版本 `register` → 顶层 `register` → `types`；`validation` 反向依赖顶层 `types` 但不依赖版本包；`v1/conversion.go` 是唯一跨到 `k8s.io/api/admissionregistration/v1` 外部包的出口。
