- 每个版本子包遵循统一文件布局：`doc.go`（包注释）+ `register.go`（`AddToScheme`）+ `defaults.go`（`SetDefaults_*`）+ `zz_generated.*`（生成产物），新增字段需同步更新 defaults 与 validation。
- 枚举类型采用 string 别名 + 包级常量形式（如 `FailurePolicyType`、`SideEffectClass`、`ValidationAction`），并在 `validation/` 中以 `sets.NewString(...)` 白名单集中校验合法性。
- 对外部 `k8s.io/api/admissionregistration/v1` 类型的互转通过 `+k8s:conversion-fn=copy-only` 标记的手动 wrapper 函数桥接到 `autoConvert_*`，避免直接依赖导致构建期不可见。
- 校验函数返回 `field.ErrorList`，通过 `field.Required/Invalid/NotSupported/Duplicate/TooMany` 等构造器配合 `fldPath.Child(...).Index(i)` 逐层定位错误路径。
- 创建/更新分离：对外暴露 `Validate*` 与 `Validate*Update` 两套入口，后者内部根据旧对象状态计算 `validationOptions`（如是否允许 sideEffects、是否忽略 matchConditions）再复用同一核心校验逻辑。
