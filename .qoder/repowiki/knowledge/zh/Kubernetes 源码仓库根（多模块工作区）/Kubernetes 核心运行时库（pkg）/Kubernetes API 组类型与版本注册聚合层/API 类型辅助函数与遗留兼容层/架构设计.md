本模块按资源族划分子包，每个子包仅暴露纯函数形式的工具方法，不持有状态：
- `legacyscheme/scheme.go` 导出全局的 `Scheme`、`Codecs`、`ParameterCodec`，作为整个 pkg/api 下所有对象编解码与转换的唯一入口，被 testing 和 v1/* 子包复用。
- 顶层目录以资源名组织：`pod/`、`service/`、`persistentvolume/`、`persistentvolumeclaim/`、`node/`、`storage/`、`resourceclaimspec/`、`job/`、`servicecidr/`，各自包含 `util.go`（字段访问/默认值/兼容性裁剪）与可选的 `warnings.go`（弃用注解/字段告警），并配套 `*_test.go`。
- `v1/` 子目录是上述资源的 v1 版本专用辅助实现（如 `v1/pod/util.go`、`v1/service/util.go`），供 versioned validation 调用；顶层 util 则面向内部版本。
- `testing/` 是独立的测试基础设施包，通过 `VerifyVersionedValidationEquivalence` / `VerifyValidationEquivalence*` 在 Beta/Standard/AllDeclarative 三种声明式校验模式下断言错误一致，并用 `skippedEquivalenceGroupVersions` 白名单跳过未声明式化的 GVK；`compat/compatibility_tester.go` 负责跨版本序列化/反序列化回归。
- 依赖方向单向向外：各资源 util 可反向调用同属本模块的其他资源 util（例如 `pod/warnings.go` 调用 `pvcutil.GetWarningsForPersistentVolumeClaimSpec`），但绝不反向依赖本模块；对外仅依赖 `pkg/apis/core` 内部类型与 `apimachinery` 公共库。
