- 每个资源子包采用 `util.go` + `*_test.go` 的最小文件布局，必要时将告警逻辑拆到同包的 `warnings.go`，保持单一职责。
- 对 Pod 类字段的废弃处理统一遵循「FeatureGate 关闭且旧对象未使用 → 置 nil」模式，由 `DropDisabledTemplateFields`/`DropDisabledPodFields` 集中调用若干 `drop*InUse` 辅助函数完成裁剪。
- 验证选项通过 `GetValidationOptionsFromPodSpecAndMeta` 从新/旧 Spec 与 Meta 推导，对旧对象中已存在的“宽松”用法一律透传允许，以保证向后兼容。
- 跨版本验证测试统一通过 `VerifyVersionedValidationEquivalence` 驱动，并在需要时借助 `WithNormalizationRules` 对 field path 做正则归一化后再比对。
- 对多容器集合的遍历统一封装为 `ContainerType` 掩码 + `VisitContainers`/`ContainerIter` 迭代器，避免在各处重复展开 Init/Containers/EphemeralContainers 三段循环。
