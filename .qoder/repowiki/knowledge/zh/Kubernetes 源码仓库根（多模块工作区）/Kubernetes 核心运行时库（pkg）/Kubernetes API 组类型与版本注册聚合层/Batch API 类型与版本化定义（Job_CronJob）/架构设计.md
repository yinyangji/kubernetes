模块按 Kubernetes API 约定分层：
- `pkg/apis/batch` 根包声明 GroupName="batch"，并通过 `SchemeGroupVersion` 指向内部版本 `runtime.APIVersionInternal`，集中注册 `Job`、`JobList`、`CronJob`、`CronJobList` 到 Scheme。
- `types.go` 是单一类型源文件，包含所有 GA 结构体（Job/CronJob/Spec/Status 及枚举），并附带 `+k8s:deepcopy-gen=package` 注解，由 `zz_generated.deepcopy.go` 生成 DeepCopy。
- `v1/` 与 `v1beta1/` 子包分别对应公开 API 版本，每个版本内通过 `register.go` 暴露 `AddToScheme`，并在 `init()` 中调用 `localSchemeBuilder.Register(addDefaultingFuncs, addConversionFuncs)`；`conversion.go` 仅注册 FieldLabelConversion（如 Job 的 `metadata.name`、`status.successful`），其余转换/默认值/验证由 `zz_generated.*.go` 提供。
- `install/` 子包聚合各版本的 `AddToScheme` 供 apiserver 启动时一次性安装。
- `validation/` 独立于版本，直接依赖根包 `batch` 类型，实现 `ValidateJob`/`ValidateCronJob` 等函数，使用 `field.ErrorList` 返回结构化错误，并通过 `JobValidationOptions`/`JobStatusValidationOptions` 控制可选校验路径（如索引作业限制、不可变字段检查）。
- `fuzzer/` 为 fuzz 测试提供随机输入构造器。
依赖方向：`v1`/`v1beta1` → `batch`（根类型）→ `apimachinery`；`validation` 反向依赖根 `batch` 但不依赖任何具体版本包，保证跨版本复用。
