模块按 Kubernetes API 组标准分层组织：
- 顶层 `pkg/apis/node` 定义**内部类型**（`types.go` 中的 `RuntimeClass`/`Overhead`/`Scheduling`），并通过 `register.go` 的 `SchemeGroupVersion` 使用 `runtime.APIVersionInternal` 注册到 Scheme，作为所有外部版本的归一化目标。
- `v1alpha1`/`v1beta1`/`v1` 三个子包分别对应公开 API 版本，每个包包含 `doc.go`、`register.go`（导出 `SchemeGroupVersion` 和 `AddToScheme`）、以及由代码生成器产出的 `zz_generated.conversion.go` 与 `zz_generated.validations.go`；其中 `v1alpha1/conversion.go` 手写覆盖自动转换以处理 spec 结构从嵌套到扁平化的不兼容变更。
- `install/install.go` 是统一入口，在 `init()` 中把 node 内部类型与各版本通过 `utilruntime.Must(...AddToScheme)` 注入到 `legacyscheme.Scheme`，并用 `scheme.SetVersionPriority(v1, v1beta1)` 指定默认偏好版本。
- `validation/validation.go` 基于内部类型提供 `ValidateRuntimeClass`/`ValidateRuntimeClassUpdate`，并暴露 `NodeNormalizationRules` 正则规则，将 v1alpha1 的 `spec.runtimeHandler`/`spec.scheduling` 规范化为 v1 的顶层字段，使同一份验证逻辑可复用跨版本。
依赖方向单向：各版本包 → 内部 node 包 → apimachinery/core，install 聚合所有版本，validation 仅依赖内部类型。
