本模块是 Kubernetes `networking.k8s.io` API 的源码实现，采用“内部类型 + 外部版本包”的经典分层：
- 根目录 `types.go` 定义所有资源的内部结构体（`NetworkPolicy`、`Ingress`、`IngressClass`、`IPAddress`、`ServiceCIDR` 及其 List 类型），并通过 `+k8s:deepcopy-gen=package` 和 `+groupName=networking.k8s.io` 注解驱动代码生成；`register.go` 通过 `SchemeGroupVersion = GroupName + runtime.APIVersionInternal` 将上述内部类型注册到运行时 Scheme。
- `v1/` 与 `v1beta1/` 子包分别绑定公开版本 `networking.k8s.io/v1` 与 `v1beta1`，各自持有独立的 `register.go`（导出 `GroupName`、`SchemeGroupVersion`、`AddToScheme`）以及由 `zz_generated.*` 生成的 DeepCopy/Default/Validation 函数；`v1/defaults.go` 与 `v1beta1/defaults.go` 提供版本特定的默认值注入。
- 跨版本兼容集中在 `v1beta1/conversion.go`，手动覆写 `autoConvert_*` 以处理 `IngressBackend` 中 `ServiceName/ServicePort` → `Service/Port` 等字段迁移，其余路径由 `zz_generated.conversion.go` 自动生成。
- 业务校验统一收敛在 `validation/validation.go`，按资源拆出 `ValidateNetworkPolicy*`、`ValidateIngress*`、`ValidateIngressClass*`、`ValidateIPAddress*`、`ValidateServiceCIDR*` 系列函数，并暴露 `NetworkPolicyValidationOptions` / `IngressValidationOptions` 等选项结构体控制 beta→GA 过渡行为（如 `AllowInvalidSecretName`、`AllowRelaxedServiceNameValidation`）。
- `fuzzer/` 为 fuzz 测试提供随机样本，`install/install.go` 供 apiserver 启动时调用，`util/helpers.go` 存放共享辅助函数。
依赖方向：`v1`/`v1beta1` → 根 `networking`（内部类型）→ `apimachinery`/`core`；`validation` 反向依赖根 `networking` 但不被其反向引用，形成单向边界。
