本模块采用「内部类型 + 版本化桥接」的两层结构：
- `pkg/apis/core` 为内部包，`doc.go` 声明 `+k8s:deepcopy-gen=package`，由 `zz_generated.deepcopy.go` 提供 DeepCopy；`register.go` 通过 `SchemeGroupVersion{Group:"", Version:runtime.APIVersionInternal}` 注册 Pod/Service/Node/PV/PVC/Secret/ConfigMap 等所有核心资源的内部 Go 结构体到 runtime.Scheme。
- `pkg/apis/core/v1` 为对外 v1 版本的桥接包，`doc.go` 使用 `+k8s:conversion-gen=k8s.io/kubernetes/pkg/apis/core`、`+k8s:defaulter-gen`、`+k8s:validation-gen` 注释驱动代码生成；`register.go` 的 `AddToScheme` 仅注册手写 `addDefaultingFuncs` 与 `addConversionFuncs`，其余由 `zz_generated.defaults.go`、`zz_generated.conversion.go`、`zz_generated.validations.go` 注入。`conversion.go` 手工实现跨包字段标签转换（Pod/Node/Event/Namespace/Secret/Service）以及 `ReplicationController ↔ apps.ReplicaSet` 的双向转换，并处理 `PodCIDR` 单值↔数组、`StringData→Data` 覆盖等兼容逻辑。
- `pkg/apis/core/install` 作为入口，`init()` 调用 `Install(legacyscheme.Scheme)` 把 core 内部包与 v1 同时加入 legacy scheme，并将 v1 设为最高优先级。
- 辅助子目录按职责切分：`helper/` 与 `pods/` 提供通用与 Pod 专用工具函数；`validation/` 集中命名、事件等校验；`fuzzer/` 提供 fuzz 测试数据；`v1/helper/qos/` 复用顶层 helper 的 QoS 计算逻辑。
- 依赖方向单向：`v1` → `k8s.io/api/core/v1`（外部类型）+ `pkg/apis/core`（内部类型）；`install` → `core` + `v1`；顶层 helper/validation 不反向依赖 v1。
