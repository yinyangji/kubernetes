模块遵循 Kubernetes API 组的通用分层结构：
- 顶层 `types.go` + `register.go` 声明内部运行时对象（`PriorityClass`、`Workload`、`PodGroup`、`CompositePodGroup` 等），通过 `SchemeGroupVersion = GroupName + /APIVersionInternal` 注册到 Scheme，供控制面组件在内存中使用。
- `v1/`、`v1beta1/`、`v1alpha3/` 三个外部版本包各自包含 `doc.go`（带 `+k8s:conversion-gen`、`+k8s:defaulter-gen`、`+k8s:validation-gen` 注释）、`register.go`、`defaults.go` 以及 `zz_generated.*` 生成文件；它们通过 `conversion-gen` 与顶层 internal types 互转，并通过 `defaulter-gen`/`validation-gen` 生成默认值填充与字段级校验。
- `install/install.go` 提供 `Install(scheme)` 入口，统一调用各版本的 `AddToScheme` 并设置版本优先级（v1 > v1beta1 > v1alpha3）。
- `validation/validation.go` 集中实现跨版本的业务校验（如 PriorityClass 的系统前缀/最大值限制、Workload 模板深度≤4、模板名唯一性、优先级一致性、ResourceClaim 状态匹配等），依赖 `apimachinery validation` 与 `pkg/apis/core/validation`。
- `fuzzer/fuzzer.go` 为 fuzzing 测试提供随机输入生成器。
依赖方向：外部版本 → conversion → internal types；校验层同时消费 internal 与 v1 类型；`install` 仅聚合注册，无业务逻辑。
