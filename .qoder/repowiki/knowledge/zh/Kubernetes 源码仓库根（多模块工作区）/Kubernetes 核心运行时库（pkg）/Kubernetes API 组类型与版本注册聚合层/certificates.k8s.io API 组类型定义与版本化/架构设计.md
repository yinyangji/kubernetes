模块按 Kubernetes API 组标准分层组织：
- 顶层 `types.go`/`register.go`/`doc.go` 声明内部 GroupName=`certificates.k8s.io`、SchemeGroupVersion=internal，并集中注册 CertificateSigningRequest、ClusterTrustBundle、PodCertificateRequest 及其 List 类型；`zz_generated.deepcopy.go` 由 deepcopy-gen 生成。
- `v1`、`v1beta1`、`v1alpha1` 三个版本子包各自包含 `register.go`（导出 SchemeGroupVersion）、`conversion.go`（跨版本转换）、`defaults.go`（默认值注入）以及 `zz_generated.*` 生成的转换/默认/校验代码；其中 v1 的 register 通过 `localSchemeBuilder.Register(addDefaultingFuncs, addConversionFuncs)` 将手写函数与生成函数合并注册。
- `install/install.go` 是聚合入口，在 init 中调用 `Install(legacyscheme.Scheme)`，把 internal + v1/v1beta1/v1alpha1 全部注册到 legacy scheme，并通过 `SetVersionPriority(v1 > v1beta1 > v1alpha1)` 确定首选版本。
- `validation/validation.go` 提供独立的业务校验器（ValidateCertificateSigningRequest*、ValidateClusterTrustBundle*、ValidatePodCertificateRequest*），使用 `field.ErrorList` 返回错误，并通过 `certificateValidationOptions` 结构体对历史兼容场景（如 legacy signerName、重复 condition type、未知 usages）做宽容处理。
- `helpers.go` 暴露 `ParseCSR` 等共享工具供校验层复用；`fuzzer/fuzzer.go` 为 fuzz testing 提供随机构造器。
依赖方向：各版本子包 → 顶层 types/register → validation/helpers；install 反向聚合所有版本，形成单向树形依赖。
