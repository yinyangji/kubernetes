模块按 Kubernetes API 组标准分层组织：
- 顶层 `pkg/apis/rbac` 定义内部存储用的核心类型（Role、ClusterRole、RoleBinding、ClusterRoleBinding、PolicyRule、Subject、AggregationRule 等），并通过 `register.go` 的 `SchemeGroupVersion` 指向 `runtime.APIVersionInternal`，作为集群内序列化版本。
- `v1`、`v1beta1`、`v1alpha1` 三个子包分别对应对外暴露的版本，每个子包通过 `+k8s:conversion-gen-external-types=k8s.io/api/rbac/<version>` 注释把外部 `k8s.io/api/rbac/<version>` 的类型映射到本模块的内部类型，并生成 `zz_generated.conversion.go`、`zz_generated.defaults.go`、`zz_generated.validations.go` 完成互转。
- `install/install.go` 是统一入口，在 `init()` 中调用各版本的 `AddToScheme` 并把 v1 → v1beta1 → v1alpha1 设为版本优先级；同时注入到 `legacyscheme.Scheme`。
- `validation/validation.go` 集中实现创建/更新校验（含 `ValidatePolicyRule`、`ValidateRoleBindingSubject` 等可导出函数供外部复用）；`fuzzer/fuzzer.go` 提供 fuzz 测试数据生成。
- 依赖方向单向：`vN` 包依赖顶层内部类型和 `k8s.io/api/rbac/vN` 外部类型，`install` 聚合所有版本，顶层不反向依赖任何版本子包。
