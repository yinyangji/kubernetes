- 对外版本子包的 `doc.go` 统一以 `+k8s:conversion-gen-external-types=k8s.io/api/rbac/<version>` 配合 `+k8s:conversion-gen=k8s.io/kubernetes/pkg/apis/rbac` 声明双向转换目标，由 code-generator 产出 `zz_generated.conversion.go`。
- 每个版本子包通过 `localSchemeBuilder = &rbacvN.SchemeBuilder` 复用外部 `k8s.io/api/rbac/vN` 已注册的 Scheme，再在 `init()` 中仅注册手写函数（如 `addDefaultingFuncs`），使缺失生成文件时仍可编译。
- 顶层内部类型通过 `+k8s:deepcopy-gen=package` 和 `+groupName=rbac.authorization.k8s.io` 标记，并由 `register.go` 中的 `addKnownTypes` 集中向内部 Scheme 注册 Role/ClusterRole 及其 List 类型。
- 构造辅助结构体（`PolicyRuleBuilder`、`ClusterRoleBindingBuilder`、`RoleBindingBuilder`）统一标注 `+k8s:deepcopy-gen=false`，避免被 deepcopy 生成器处理，并在 `*OrDie()` 方法中对构造错误 panic。
- 校验函数采用 `ValidateXxx` / `ValidateXxxUpdate` 成对命名，返回 `field.ErrorList`，并通过 `MarkCoveredByDeclarative()` 标注可由声明式校验覆盖的错误。
