本模块是 Kubernetes 的 `storage.k8s.io` API 组，采用标准的 apimachinery 分层：
- 根包 `pkg/apis/storage` 定义所有对象的 Go 结构体（`types.go`）并通过 `register.go` 中的 `SchemeGroupVersion = {Group: "storage.k8s.io", Version: runtime.APIVersionInternal}` 注册到内部 scheme；`doc.go` 使用 `+k8s:deepcopy-gen=package` / `+groupName=storage.k8s.io` 注解驱动代码生成。
- 每个已发布版本位于独立子目录（`v1`、`v1beta1`、`v1alpha1`），各自包含 `register.go`（导出 `SchemeGroupVersion` 和 `AddToScheme`）、`defaults.go`（手动默认值）、`zz_generated.*`（由 deepcopy/conversion/default/validation 生成器产出）。`v1/register.go` 通过 `localSchemeBuilder = &storagev1.SchemeBuilder` 复用 `k8s.io/api/storage/v1` 中生成的 SchemeBuilder，仅在此处注册手写函数，使缺失生成文件时仍可编译。
- `install/install.go` 作为聚合入口，在 `init()` 中将 v1/v1beta1/v1alpha1 全部注入 `legacyscheme.Scheme`，并用 `SetVersionPriority(v1, v1beta1, v1alpha1)` 确定序列化优先级。
- `validation/validation.go` 集中实现各资源的字段级校验与更新不可变约束，依赖 `pkg/apis/core/validation` 与 `utilfeature` 按 feature gate 分支；`fuzzer/fuzzer.go` 提供 fuzzing 数据生成；`util/helpers.go` 存放共享辅助逻辑。
- 依赖方向单向：`vN` → `storage`（内部类型）→ `apimachinery`/`core`；`install` 聚合所有版本，其他组件只依赖 `install`。
