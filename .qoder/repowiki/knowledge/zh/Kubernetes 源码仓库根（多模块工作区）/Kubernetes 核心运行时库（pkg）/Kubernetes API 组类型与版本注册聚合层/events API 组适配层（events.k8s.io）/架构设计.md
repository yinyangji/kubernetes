本模块是 Kubernetes 内部 API 适配层，遵循 apimachinery 的 scheme/conversion 模式：
- 顶层 `register.go` 定义 GroupName=`events.k8s.io` 与 SchemeGroupVersion=internal，并通过 `addKnownTypes` 将 `core.Event`/`core.EventList` 注册到 internal scheme。
- `install/install.go` 作为统一入口，在 `init()` 中把 events group 连同 v1beta1、v1 版本一并注入 `legacyscheme.Scheme`，并用 `SetVersionPriority(v1, v1beta1)` 指定首选版本。
- `v1/` 与 `v1beta1/` 各自持有对外部包 `k8s.io/api/events/vX` 的 `SchemeBuilder` 引用（`localSchemeBuilder = &eventsv1.SchemeBuilder`），并在 `init()` 中仅注册手写函数 `RegisterDefaults` 与 `AddFieldLabelConversionsForEvent`；其余类型注册由 `zz_generated.*.go` 完成，使生成文件缺失时仍可编译。
- `conversion.go` 实现双向 `Convert_*_Event_To_core_Event` / `Convert_core_Event_To_*_Event`，负责字段重映射（如 `Note↔Message`、`DeprecatedFirstTimestamp↔FirstTimestamp`、`Regarding↔InvolvedObject`、`DeprecatedSource↔Source`），并调用 `k8s_api_v1` 下的 ObjectReference/EventSource 转换器。
- 依赖方向单向向外：本模块依赖 `k8s.io/api/events/vX`（外部类型）、`pkg/apis/core`（内部核心 Event 类型）以及 `apimachinery` 的 runtime/conversion，不反向依赖上层组件。
