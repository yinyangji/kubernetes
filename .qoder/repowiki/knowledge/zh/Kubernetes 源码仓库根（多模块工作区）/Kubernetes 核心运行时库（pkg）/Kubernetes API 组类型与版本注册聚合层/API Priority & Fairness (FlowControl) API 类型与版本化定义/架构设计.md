模块按 Kubernetes API 分层规范组织：
- `types.go` + `register.go` + `doc.go` 声明内部（unversioned）对象模型，作为所有版本的转换目标；`SchemeGroupVersion` 使用 `runtime.APIVersionInternal`。
- `v1`、`v1beta1`、`v1beta2`、`v1beta3` 四个子包分别对应对外暴露的 API 版本，每个子包包含 `doc.go`（通过 `+k8s:conversion-gen` / `+k8s:defaulter-gen` / `+k8s:validation-gen` 注解驱动代码生成）、`register.go`（绑定到 `k8s.io/api/flowcontrol/<ver>` 的 SchemeBuilder）、`defaults.go`、`zz_generated.*.go`（deepcopy/defaulter/validation/conversion）以及可选的手写 `conversion.go`（如 v1beta3 对 `NominalConcurrencyShares==0` 的 roundtrip annotation 处理）。
- `install/install.go` 集中注册所有版本到 scheme，并通过 `SetVersionPriority(v1, v1beta3, v1beta2, v1beta1)` 指定首选版本为 v1。
- `internalbootstrap/default-internal.go` 将 `k8s.io/apiserver/pkg/apis/flowcontrol/bootstrap.Mandatory*` 外部强制资源转换为内部 untyped 对象，供 apiserver 启动时注入。
- `util/helpers.go`、`validation/validation.go`、`fuzzer/fuzzer.go` 提供共享工具、字段级校验与 fuzzing 支持。
依赖方向：各版本包 → 外部 `k8s.io/api/flowcontrol/<ver>` → 本模块内部 `pkg/apis/flowcontrol` 内部类型；`install` 聚合所有版本；`internalbootstrap` 仅依赖 `install` 和外部 bootstrap 数据，不反向依赖具体版本包。
