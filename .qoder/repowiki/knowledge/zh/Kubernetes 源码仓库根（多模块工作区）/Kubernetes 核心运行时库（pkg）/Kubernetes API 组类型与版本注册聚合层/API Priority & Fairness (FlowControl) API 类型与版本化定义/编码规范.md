- 每个对外版本子包在 `doc.go` 顶部集中声明 `+k8s:conversion-gen`、`+k8s:defaulter-gen`、`+k8s:validation-gen` 等注解，由 code-generator 产出 `zz_generated.conversion.go`、`zz_generated.defaults.go`、`zz_generated.validations.go` 与 deepcopy。
- 版本包 `register.go` 中 `AddToScheme` 直接复用外部 `k8s.io/api/flowcontrol/<ver>.SchemeBuilder`，并在 `init()` 中手动 `Register(RegisterDefaults)`，以便在生成的文件缺失时仍可编译。
- 对外版本与内部类型的转换通过手写 `Convert_*_To_flowcontrol_*` 函数包裹自动生成的 `autoConvert_*`，用于在双向转换中对 annotations 做 roundtrip 兼容处理（如 v1beta3 的 `PriorityLevelPreserveZeroConcurrencySharesKey`）。
- 内部对象统一采用 `metav1.TypeMeta` + `metav1.ObjectMeta` + `Spec`/`Status` 三字段结构，并标注 `+genclient`、`+genclient:nonNamespaced`、`+k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object` 以生成 List 与 Object 接口。
- 条件字段遵循标准模式：`Conditions []XxxCondition` 配合 `+listType=map`、`+listMapKey=type`、`+patchMergeKey=type`、`+patchStrategy=merge` 注解，状态常量集中在 `types.go` 顶部的 `const` 块中。
