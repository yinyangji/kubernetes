模块按 Kubernetes API 组标准分层组织：
- 根包 `pkg/apis/authentication` 存放内部版本类型（`types.go` 中的 `TokenReview`/`TokenRequest`/`SelfSubjectReview`）、`SchemeGroupVersion` 注册以及共享常量（如 `Impersonate-*-Header`），并通过 `zz_generated.deepcopy.go` 提供 deep copy。
- `v1`、`v1alpha1`、`v1beta1` 三个子目录分别对应 `k8s.io/api/authentication/vX` 外部类型的本地映射，每个子目录包含 `doc.go`（声明 `+k8s:conversion-gen` / `+k8s:defaulter-gen` / `+k8s:validation-gen` 注释）、`register.go`（复用上游 `SchemeBuilder`）、`defaults.go`、`conversion.go` 及 `zz_generated.*.go` 生成文件。
- `install/install.go` 负责把各版本 Scheme 注入到全局 scheme；`fuzzer/fuzzer.go` 为 fuzzing 提供随机构造器。
- `validation/validation.go` 集中实现跨版本的业务校验（如 `MinTokenAgeSec=600` 秒下限），由上层 REST 层调用。
依赖方向：各版本包仅依赖上游 `k8s.io/api/authentication/vX` 和 `apimachinery`，不反向依赖具体实现；根包通过 `runtime.SchemeBuilder` 聚合所有已知类型。
