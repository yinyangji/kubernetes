本模块是 coordination.k8s.io API 组的类型声明与注册层，按标准 Kubernetes API 包结构分层：
- 顶层 `types.go` 集中声明内部版本（`runtime.APIVersionInternal`）的 `Lease`、`LeaseList`、`LeaseCandidate`、`LeaseCandidateList` 及其 Spec/Strategy 等核心结构体，并通过 `+k8s:deepcopy-gen=package` 注解驱动代码生成；`register.go` 暴露 `GroupName`、`SchemeGroupVersion`、`Kind`/`Resource` 辅助函数以及将上述类型注册到 Scheme 的 `addKnownTypes`。
- `v1`、`v1beta1`、`v1alpha2` 三个子包分别对应公开版本，每个子包的 `register.go` 仅持有对 `k8s.io/api/coordination/<version>.SchemeBuilder` 的引用并注册本地 `RegisterDefaults`，实际类型定义由外部 `k8s.io/api/coordination` 仓库提供；各版本目录下 `zz_generated.*.go` 由 deepcopy/conversion/defaults/validation 生成器产出，保证跨版本双向转换与字段默认值。
- `install/install.go` 作为统一入口，在 `init()` 中把本组及所有版本注册进 `legacyscheme.Scheme`，并通过 `SetVersionPriority(v1 > v1beta1 > v1alpha2)` 指定默认序列化优先级。
- `validation/validation.go` 实现独立的业务校验逻辑（如 `leaseDurationSeconds > 0`、`binaryVersion ≥ emulationVersion`、`strategy` 白名单 + 可选扩展名），不依赖任何版本子包，仅消费顶层 `coordination.CoordinatedLeaseStrategy` 常量。
依赖方向：`install` → 各版本 `AddToScheme` → 顶层 `coordination.AddToScheme`；`validation` 仅依赖顶层类型，不被版本子包反向依赖。
