模块按 k8s.io/kubernetes/pkg/apis 标准分层组织：
- 根包 `pkg/apis/apps` 通过 `register.go` 的 `SchemeBuilder` 把内部类型注册到 `runtime.Scheme`，并暴露 `GroupName="apps"`、`Kind()`/`Resource()` 等 Group 级构造器；`types.go` 集中声明所有内部结构体（含 List 与 Condition），由 `zz_generated.deepcopy.go` 提供 DeepCopy。
- 每个版本子目录 `v1`、`v1beta1`、`v1beta2` 对应一个公开 APIVersion，其 `doc.go` 用 `+k8s:conversion-gen-external-types=k8s.io/api/apps/vN` 指向外部 `k8s.io/api/apps/vN` 包，并通过 `+k8s:defaulter-gen` / `+k8s:validation-gen` 生成 `zz_generated.defaults.go` 与 `zz_generated.validations.go`；`conversion.go` 手写覆盖 auto-generated 转换以处理注解往返（如 Deployment 的 `DeprecatedRollbackTo` ↔ `spec.rollbackTo`、DaemonSet 的 `DeprecatedTemplateGeneration` ↔ `spec.templateGeneration`）以及 StatefulSet 的 PVC apiVersion/Kind 兼容行为。
- `install/install.go` 负责把各版本的 SchemeGroupVersion 注入全局 scheme；`fuzzer/fuzzer.go` 为 fuzzing 提供随机数据。
- 依赖方向单向：版本包 → 根内部包 → `k8s.io/api/apps/vN` 外部类型；验证逻辑集中在独立的 `validation/validation.go`，被控制器与 APIServer 共同复用。
