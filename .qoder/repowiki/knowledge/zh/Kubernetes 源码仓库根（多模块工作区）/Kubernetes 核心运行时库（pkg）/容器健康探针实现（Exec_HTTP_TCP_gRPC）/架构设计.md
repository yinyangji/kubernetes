模块以 `probe.Result`（success/warning/failure/unknown）为跨协议统一返回值，按协议拆分为四个子包：
- `exec`: 通过 CRI client 在容器内执行命令，依据退出码映射到 Success/Failure，超时走 `remote.ErrCommandTimedOut`。
- `http`: 构造禁用代理与压缩、TLS 校验关闭的 `http.Transport`，并通过 `probe.ProbeDialer()` 注入自定义 Dialer；支持可选跟随非本地重定向，否则返回 Warning。
- `tcp`: 基于 `DoTCPProbe` 直接拨号指定 host:port，错误一律转为 Failure。
- `grpc`: 使用 `grpc.health.v1.Health` 协议，User-Agent 带 kubelet 版本，连接失败或状态非 SERVING 均返回 Failure。
每个子包暴露 `New() Prober` + `Probe(...)` 接口，同时导出 `Do*Probe` 供外部直调。根目录的 `dialer_others.go` / `dialer_windows.go` 通过 build tag 区分，`ProbeDialer()` 将 SO_LINGER 设为 1s 缩短 TIME-WAIT，被 http/tcp/grpc 三方共享。`util.go` 提供 `ResolveContainerPort` 把 `intstr.IntOrString` 解析为容器端口号，是上层编排探针参数时的公共工具。依赖方向单向：各子包 → `k8s.io/kubernetes/pkg/probe`，不反向依赖。
