本模块是 `pkg/proxy` 下的共享子包集合，按职责拆分为六个独立 Go package，被上层具体 proxier 实现和 `cmd/kube-proxy/app` 引用：
- `apis/config`：基于 `+groupName=kubeproxy.config.k8s.io` 的自定义 API 组，定义 `KubeProxyConfiguration` 及 Linux/Windows/IPTables/IPVS/NFTables/Winkernel 等配置结构体；通过 `scheme/` 注册 scheme、`v1alpha1/` 做版本转换与默认值注入、`validation/` 做字段校验、`fuzzer/` 生成随机配置，`zz_generated.deepcopy.go` 由 deepcopy-gen 生成。
- `config`：独立的配置聚合层，将多个配置源（etcd、文件等）增量合并并提供统一视图，解耦数据源与 proxier。
- `conntrack`：Linux-only（`//go:build linux`）内核 conntrack 表清理工具，暴露 `Interface` 抽象并通过 `fake.go` 提供测试替身，核心逻辑在 `cleanup.go` 中扫描并删除 UDP 服务失效条目，同时上报 `ReconcileConntrackFlows*` 指标。
- `healthcheck`：对外暴露 `/healthz` 与 `/livez` HTTP 端点，通过 `QueuedUpdate`/`Updated` 接口感知 proxier 同步延迟，结合 node taint 判断节点可调度性，内部用 `listener`/`httpServerFactory`/`nodeManager` 接口切分依赖以打破循环引用。
- `metrics`：集中声明所有 `kubeproxy` subsystem 的 Prometheus 指标，通过 `RegisterMetrics(mode)` 按 `ProxyMode` 分支注册 iptables/ipvs/nftables/winkernel 专属指标，并使用 `nfacctMetricCollector` 实现 `StableCollector` 从 netfilter accounting 拉取计数。
- `kubemark`：为 kubemark 测试环境提供 `HollowProxy`，构造一个 `ProxyMode("fake")` 的 `proxyapp.ProxyServer` 配合空实现的 `FakeProxier`，使测试能复用真实启动流程而不操作内核。

依赖方向单向向下：`conntrack`/`healthcheck`/`kubemark` 均只依赖 `apis/config` 与 `metrics`，不被它们反向引用，从而形成稳定的共享层。
