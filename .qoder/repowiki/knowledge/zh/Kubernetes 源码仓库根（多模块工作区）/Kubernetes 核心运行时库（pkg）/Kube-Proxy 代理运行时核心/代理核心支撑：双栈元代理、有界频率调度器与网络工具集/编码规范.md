- 跨平台差异通过 Go build tag 文件拆分（如 `*_linux.go` / `*_others.go`），公共 API 留在无标签文件中暴露。
- 对外能力以 interface 定义在独立文件中（`nfacct.Interface`、`LocalTrafficDetector`），并提供 New* 工厂返回具体实现，便于测试注入。
- 事件分发型结构体（如 `metaProxier`）直接实现上层 `proxy.Provider` 接口，方法体只做简单的族选择转发，不包含业务逻辑。
- 周期性任务统一采用 `clock.NewTimer` + select loop 模式，并通过构造函数注入 clock 以便单元测试。
