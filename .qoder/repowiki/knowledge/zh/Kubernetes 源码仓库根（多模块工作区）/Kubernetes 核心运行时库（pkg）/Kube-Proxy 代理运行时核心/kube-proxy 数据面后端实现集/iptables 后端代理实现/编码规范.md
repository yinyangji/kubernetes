- 所有 iptables 链名常量统一以 `kubeXxxChain utiliptables.Chain = "KUBE-..."` 形式定义在 proxier.go 顶部，并在 `iptablesJumpChains` 表中集中声明跳转关系，便于同步与清理复用。
- 动态链名通过 `portProtoHash(servicePortName, protocol)` 对名称+协议做 sha256→base32→截断 16 位生成，确保长度 ≤28 且唯一，各前缀（KUBE-SVC-/KUBE-SVL-/KUBE-SEP-/KUBE-FW-/KUBE-EXT-）对应独立函数。
- 规则构建采用 `LineBuffer` 批量写入后再一次性 `Restore`，而非逐条调用 `EnsureRule`，以降低系统调用开销。
- 对外部不可达场景（无 endpoint 或无本地 endpoint）统一在 filter 表插入 DROP/REJECT 规则，并通过 `serviceNoLocalEndpointsTotalInternal/External` 计数器上报指标。
- 大集群优化：当 `totalEndpoints > largeClusterEndpointsThreshold` 时关闭 `-m comment --comment` 输出，并将 `largeClusterMode` 标记用于跳过部分调试路径。
- 非 linux 平台通过空的 `doc.go` 占位包避免 build constraint 报错，真实实现全部带 `//go:build linux` 标签。
