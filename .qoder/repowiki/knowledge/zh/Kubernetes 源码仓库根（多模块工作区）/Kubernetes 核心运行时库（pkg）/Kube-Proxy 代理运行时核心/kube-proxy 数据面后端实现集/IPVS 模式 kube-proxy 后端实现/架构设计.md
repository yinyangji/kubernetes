本模块是 `pkg/proxy` 下的一种 Proxier 实现，通过 `NewDualStackProxier` 为 IPv4/IPv6 各构造一个单栈 `Proxier`，再交由 `metaproxier.NewMetaProxier` 组合成双栈入口。

- 核心状态机在 `proxier.go`：`Proxier` 持有 `ServiceChangeTracker` / `EndpointsChangeTracker`，由 `runner.BoundedFrequencyRunner` 周期性触发 `syncProxyRules`，将 Service/EndpointSlice 增量同步到内核。
- 依赖注入接口化：`utilipvs.Interface`（`util/ipvs.go`）、`utilipset.Interface`（`ipset/types.go`）、`utiliptables.Interface`、`conntrack.Interface`、`LocalTrafficDetector` 等均以接口形式传入，便于测试替换。
- 子包分层：
  - `util/`：定义 `Interface` 及 `VirtualServer`/`RealServer` 数据结构，`ipvs_linux.go` 提供基于 libipvs 的实际实现；`netlink_linux.go` 封装 dummy 设备创建/删除。
  - `ipset/`：抽象 ipset 操作并集中声明所有 KUBE-* 集合名（如 `KUBE-CLUSTER-IP`、`KUBE-NODE-PORT-TCP` 等），上层仅通过 `Interface` 使用。
  - 顶层文件按职责划分：`supported.go` 暴露 `CanUseIPVSProxier` 探测能力；`cleanup.go` 提供进程退出时的幂等清理；`graceful_termination.go` 处理 TCP/SCTP 连接优雅下线；`ipset.go`/`ipset_test.go` 管理集合生命周期。
- 依赖方向单向向外：本模块只依赖 `k8s.io/kubernetes/pkg/util/iptables`、`component-helpers/node/util/sysctl`、`proxy/healthcheck`、`proxy/metrics` 等公共库，不反向依赖上层 runner 或 API server。
- 构建约束：所有源文件以 `//go:build linux` 标记，非 Linux 平台不可编译，运行时通过 `modprobe ip_vs` 探测内核支持。
