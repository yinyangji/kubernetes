模块以 `Proxier` 结构体为核心，实现 `proxy.Provider` 接口，通过 `NewDualStackProxier` 组合 IPv4/IPv6 两个单栈 Proxier 并由 `metaproxier.NewMetaProxier` 聚合，从而支持双栈。

- 入口与生命周期：`proxier.go` 中的 `NewProxier` 负责 sysctl 校验（`route_localnet`、`nf_conntrack_tcp_be_liberal`）、创建 `BoundedFrequencyRunner` 驱动 `syncProxyRules`、启动 `utiliptables.Interface.Monitor` 监听 mangle/NAT/filter 表变化并触发强制全量同步；`Sync()` / `SyncLoop()` 暴露给上层 runner 调用。
- 事件到规则的流水线：`OnService*` / `OnEndpointSlice*` 回调仅写入 `serviceChanges` / `endpointsChanges` 变更追踪器，由 `syncProxyRules` 在持有 `mu` 锁时合并为 `svcPortMap` / `endpointsMap`，再按服务维度遍历生成 NAT 与 filter 链。
- 规则生成策略：使用 `proxyutil.LineBuffer` 预分配 buffer 批量拼装 `*nat` 和 `*filter` 表内容，最后通过 `iptables.Restore` 原子替换；当总端点数超过 `largeClusterEndpointsThreshold=1000` 时进入 large cluster mode，丢弃注释以压缩规则体积。
- 命名空间隔离：所有 KUBE-* 顶级链（KUBE-SERVICES、KUBE-NODEPORTS、KUBE-FORWARD、KUBE-MARK-MASQ、KUBE-POSTROUTING、KUBE-PROXY-FIREWALL、KUBE-EXTERNAL-SERVICES）在 `iptablesJumpChains` 中集中声明，并通过 `portProtoHash`（sha256+base32 截断 16 位）派生 KUBE-SVC-/KUBE-SVL-/KUBE-SEP-/KUBE-FW-/KUBE-EXT- 子链名，保证 ≤28 字符且可逆识别。
- 清理边界：`cleanup.go` 提供 `CleanupLeftovers`，反向遍历同一份 chain 列表，用 `SaveInto` + `GetChainsFromTable` 扫描现有链并按前缀匹配删除，同时移除 mangle 表的 canary 链；`doc.go` 是空包文件，用于在非 linux 平台避免“build constraints exclude all Go files”错误。
- 依赖方向：本模块只向上暴露 `proxy.Provider`，向下依赖 `utiliptables.Interface`、`conntrack.Interface`、`nfacct.Interface`、`healthcheck.ServiceHealthServer`、`runner.BoundedFrequencyRunner` 等抽象，不直接调用 iptables 二进制。
