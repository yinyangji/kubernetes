- 所有源文件顶部使用 `//go:build windows` 构建标签，确保仅参与 Windows 构建。
- 对底层 HNS/HCN 的访问一律通过 `HcnService` / `HostNetworkService` 接口进行，生产用 `hcnImpl`/`hns` 实现，测试用 `testing.HcnMock` 注入，禁止直接调用 `hnslib`。
- 错误分类采用有限枚举类型（如 `lbErrorType`、`loadBalancerType`）配合 `classifyLBError` 函数，再写入 metrics label，避免错误字符串直接入指标。
- LB 标识使用 `loadBalancerIdentifier` 结构体（protocol/internalPort/externalPort/vip/endpointsHash/isIPv6）+ SHA1 XOR 计算后端集合哈希，保证无序端点集下的幂等匹配。
- 服务/端点生命周期方法成对出现：创建返回带 `hnsID` 的内部对象，清理时先尝试删除并在失败时记录到 `mapStaleLoadbalancers` 延迟重试，而非立即报错。
- 日志输出统一使用 `klog.V(level).InfoS(...)` 结构化字段形式，关键调试路径（如端点变更）默认 V(5) 高冗长级别。
