模块按职责分为四层：
- 适配层 `hcnutils.go`：定义 `HcnService` 接口并给出 `hcnImpl` 直接封装 `github.com/Microsoft/hnslib/hcn`，作为对外唯一依赖点；`testing/hcnutils_mock.go` 提供同接口的内存 mock，供单元测试注入。
- 网络抽象层 `hns.go`：定义 `HostNetworkService` 接口及 `hns` 实现，将 HCN 的 Network/Endpoint/LoadBalancer 资源统一为 `endpointInfo`、`loadBalancerInfo`、`hnsNetworkInfo` 等内部结构，负责端点去重（含 Windows 本地/远端 IP 冲突清理）、LB 创建/更新/删除以及基于 endpoint ID XOR-SHA1 的 `loadBalancerIdentifier` 去重。
- 策略编排层 `proxier.go`：实现 `proxy.Provider`，通过 `proxy.ServiceChangeTracker` / `proxy.EndpointsChangeTracker` 接收上游变更，经 `syncProxyRules` + `runner.BoundedFrequencyRunner` 节流后调用 `serviceInfo`/`endpointInfo` 的 `cleanupAllPolicies` 与 LB 操作，维护 `svcPortMap`、`endpointsMap`、`mapStaleLoadbalancers`、`terminatedEndpoints` 等状态；`NewDualStackProxier` 使用 `metaproxier.NewMetaProxier` 组合 IPv4/IPv6 单栈实例。
- 能力探测层 `supported.go`：`KernelCompatTester`/`StackCompatTester` 通过 `hnslib.HNSListPolicyListRequest`、`Ipv6DualStackSupported`、Overlay 特性门控决定是否启用本后端。
依赖方向：`proxier → hns → hcnutils → hnslib/hcn`，测试通过 `HcnService` 接口反向注入 mock，避免真实 HNS 调用。所有文件以 `//go:build windows` 限定仅在 Windows 构建。
