本模块是 kube-proxy 的“可插拔数据面”集合：父包 pkg/proxy/config 定义 ServiceHandler/EndpointSliceHandler/NodeTopologyHandler/ServiceCIDRHandler 四个事件接口，并由 config.ServiceConfig、config.EndpointSliceConfig、config.NodeConfig、config.ServiceCIDRConfig 等 informer-backed 控制器将 API Server 变更广播给所有已注册的 handler；每个子目录（iptables/ipvs/nftables/winkernel）各自实现一个满足上述接口的具体 proxier，并在自身 supported.go 中声明平台能力，由上层根据配置选择其一作为唯一的 Provider 实例。四个后端彼此互斥、仅共享同一套事件契约与公共类型（如 ServicePortName），不互相 import，从而保证 Linux iptables/IPVS/nftables 与 Windows HNS/HCN 两套数据面在编译期按 build tag 隔离。
