本模块是各独立可执行程序（kube-apiserver、kube-controller-manager、kube-scheduler、kubelet、kube-proxy）共同依赖的 Go 包集合，通过以下层次协作：
- api_types + registry_layer 构成资源契约层：前者定义所有 API Group 的类型与 Scheme 安装入口，后者为每个 GroupVersion 装配 REST storage/strategy/authz 并挂载到版本化路由；两者一一对应，由 apiserver 启动时统一 import。
- control_plane_orchestrators 作为编排枢纽，在 apiserver_bootstrap 中串联 API Server、内置控制器与 KCM 控制器集合，使上述两类包以插件形式被注入。
- kubelet_node_agent 与 volume_plugins、credential_provider、probe_healthcheck、security_context 等构成节点侧运行时，彼此通过共享接口组合并由 NewMainKubelet 驱动。
- proxy、scheduler 各自自包含完整组件运行时，但同样消费 api_types/registry_layer 提供的对象模型。
- auth_authorization、admission_webhooks、quota_evaluator、features_flags、printers、routes 等为上层组件提供横切能力，被 apiserver/kubelet/scheduler 多处复用。
- client_library、kubectl_convert、kubemark、shared_utils 属于工具/测试支撑层，不改变主运行路径。
