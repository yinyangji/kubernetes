模块按资源族拆分为两个并列子包，均复用 staging 层的通用清单加载循环：
- `plugin/webhook/manifest/loader`：封装 `stagingloader.LoadManifests`，通过自定义 `acceptObject[T]` 泛型分发器将已解码对象路由到 `extractValidating` / `extractMutating`，再经 `defaultAndValidateVWC` / `defaultAndValidateMWC` 完成 scheme.Default → 内部类型转换 → `validation.Validate*` → 反向转换的标准流水线。
- `plugin/policy/manifest/loader/{mutating_loader.go,validating_loader.go}`：分别封装 `policyloader.LoadPolicyManifests`，以 `acceptMAP`/`acceptMAPB` 和 `acceptVAP`/`acceptVAPB` 作为 acceptor 回调，配合各自的 `defaultAndValidate*` 处理 MutatingAdmissionPolicy/Binding 与 ValidatingAdmissionPolicy/Binding。
每个子包在 init 中通过 `admissionregistrationinstall.Install(scheme)` 注册 v1 类型并添加 unversioned List/Status，使用 `serializer.NewCodecFactory(scheme, serializer.EnableStrict)` 进行严格反序列化；所有错误统一用 `%w` 包装原始 error 以便调用方判断。依赖方向单向指向 staging loader 与 kubernetes 内部的 `pkg/apis/admissionregistration/install` 与 `validation`，不反向依赖上层 apiserver。
