本模块由四个相互协作的子包构成，围绕 `fwk.APIDispatcher` 接口解耦对 apiserver 的同步写入：
- `api_dispatcher/`：实现 `fwk.APIDispatcher`，内部通过 `callQueue` + `goroutines_limiter` 将 Bind/Patch 等写操作排队并发执行，并暴露 `SyncObject` 做双向同步。
- `cache/`：定义 `Cache` 接口（`interface.go`），以 pod-centric 增量事件驱动维护 NodeInfo、AssumedPods、PodGroup/CompositePodGroup 状态；`cacheImpl` 用 RWMutex 保护内存结构，并通过 `nodeTree` 聚合节点资源视图，`UpdateSnapshot` 在每轮调度前导出只读快照。
- `queue/`：核心是 `PriorityQueue`（`scheduling_queue.go`），组合 `activeQ`（基于 `heap.Heap[T]`）、`backoffQ`（退避堆）和 `unschedulableEntities`，支持 GenericWorkload 下的 `workloadForest` 与 `podGroupMemberPods` 维持 PodGroup 生命周期不变式；通过 `PreEnqueuePlugin` 与 `QueueingHintFn` 按 profile/event 精细控制入队策略。
- `heap/`：无锁泛型堆 `Heap[T Item]`，仅依赖 `container/heap`，由上层队列加锁，提供 O(1) keyIndex 查找。
依赖方向：`queue` → `heap`；`cache` 与 `queue` 均通过 `fwk.APIDispatcher` 间接依赖 `api_dispatcher`，避免循环引用。
