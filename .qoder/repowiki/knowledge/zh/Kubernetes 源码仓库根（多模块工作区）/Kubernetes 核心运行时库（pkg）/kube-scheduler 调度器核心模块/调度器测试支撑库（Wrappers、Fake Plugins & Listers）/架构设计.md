模块按职责拆分为两个子包：
- `pkg/scheduler/testing`（顶层 package testing）：通过 `wrappers.go` 暴露 `PodWrapper`、`NodeWrapper`、`ContainerWrapper`、`LabelSelectorWrapper`、`NodeSelectorWrapper`、`PersistentVolumeClaimWrapper`、`PodDisruptionBudgetWrapper` 等 builder，统一以 `MakeXxx()` + 链式 setter + `Obj()` 模式生成 v1 资源；`workload_prep.go` 提供 `MakeNodesAndPodsForEvenPodsSpread` 用于 EvenPodsSpread 特性测试。
- `pkg/scheduler/testing/framework`（package framework）：面向 scheduler framework 的测试桩——`fake_plugins.go` 实现 Filter/PreFilter/Reserve/PostFilter/Permit/PreBind/Score/PreScore 等所有 extension point 的 fake 插件并配套 `NewXxxPlugin` 工厂返回 `frameworkruntime.PluginFactory`；`fake_extender.go` 提供 `FakeExtender` 及一组 `FitPredicate`/`PriorityFunc` 闭包，模拟旧版 extender/v1 行为；`fake_listers.go` 用切片实现 client-go lister 接口（ServiceLister、ReplicaSetLister、StatefulSetLister、PersistentVolumeClaimNamespaceLister 等），配合 `GetPod*` 辅助方法按 label selector 反查控制器；`framework_helpers.go` 提供 `RegisterXxxPlugin` 注册器与 `NewFramework`，把 fake 插件注入到 `schedulerapi.KubeSchedulerProfile` 并通过 `runtime.NewFramework` 组装出可运行的 `framework.Framework`。
依赖方向单向向外：本模块只依赖 `k8s.io/api/*`、`client-go/listers/*`、`kube-scheduler/framework`、`kubernetes/pkg/scheduler/apis/config` 与 `framework/runtime`，不反向依赖被测调度器核心逻辑，仅作为被 import 的纯测试辅助包。
