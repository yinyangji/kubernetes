本模块按职责划分为四层并通过包级依赖自顶向下串联：
- `apis/config` 定义 KubeSchedulerConfiguration 内部类型并桥接 v1 外部版本，供上层启动时解析。
- `backend` 提供 Informer 驱动的 Cache、优先级 Queue（含 PodGroup/Backoff/WorkloadForest）以及异步 API Dispatcher，是调度器的数据与事件源。
- `framework` 暴露 Plugin 接口、CycleState、Preemption 引擎与全部内置插件，由 `runtime` 在调度周期中按阶段调用。
- `core_orchestration`（scheduler.go / schedule_one*.go）作为入口，从 Queue 取 Pod/PodGroup，驱动 Framework 执行 Filter→Score→Bind 等阶段，并将结果落回 Cache。
- `metrics` 在所有层注册 Prometheus 指标；`util` 提供跨层共享的 AssumeCache、资源计算等工具；`testing` 为各子层提供 Fake Listers/Plugins/Wrappers。
启动时由 apis/config 加载配置 → 初始化 backend 缓存/队列 → 注册 framework 插件 → 进入 core 主循环，形成单向依赖链 config→backend→framework→core。
