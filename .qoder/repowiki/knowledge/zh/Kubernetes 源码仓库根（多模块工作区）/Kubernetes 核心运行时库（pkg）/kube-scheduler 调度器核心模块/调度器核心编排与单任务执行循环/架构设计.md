本模块是调度器的运行时编排层，围绕 `Scheduler` 结构体组织，职责分层如下：
- 生命周期与装配（scheduler.go）：`New()` 通过 Option 模式注入 profile、extenders、parallelism、snapshot 等；构建 in-tree + out-of-tree plugin registry，初始化 `internalcache.Snapshot`、`SchedulingQueue`、可选的 `APIDispatcher`/DRA manager/CSI manager，并调用 `addAllEventHandlers` 注册所有 informer handler。
- 调度入口与分发（schedule_one.go）：`Run()` 启动 queue 与 APIDispatcher，并在独立 goroutine 中循环调用 `ScheduleOne`；`ScheduleOne` 根据实体类型分派到 `scheduleOnePod` 或 `scheduleOnePodGroup`；Pod 路径走 `schedulingAlgorithm`(Filter→Score→PostFilter) → `assumeAndReserve` → `prepareForBindingCycle`(Permit) → 异步 `runBindingCycle`(PreBind→Bind→PostBind)。
- PodGroup 批量调度（schedule_one_podgroup.go）：在 `podGroupCycle` 中先运行 PlacementFeasible 插件校验约束，再对每个 pod 复用 `schedulingAlgorithm`/`assumeAndReserve`，并通过 `revertFns` LIFO 撤销中间假设；支持 CompositePodGroup 递归遍历与父级失败向子级传播；最终由 `submitPodGroupAlgorithmResult` 统一提交结果并更新 PodGroup Condition。
- 事件驱动（eventhandlers.go）：为 Pod/Node/PodGroup/CompositePodGroup 及 DRA/CSI/PV/PVC/DeviceClass 等 GVK 注册 Add/Update/Delete handler，将集群变更转化为 `fwk.ClusterEvent` 并调用 `SchedulingQueue.MoveAllToActiveOrBackoffQueue` 触发重调度；`addAllEventHandlers` 按 profile 的 `EnqueueExtensions.EventsToRegister` 动态注册动态资源 informer。
- 外部扩展（extender.go）：`HTTPExtender` 实现 `fwk.Extender`，通过 JSON+HTTP 调用 filter/prioritize/preempt/bind verb，支持 nodeCacheCapable 只传 NodeName 列表以减少带宽，并按 `managedResources` 做 IsInterested 短路。
依赖方向：本模块仅依赖 `backend/cache`、`backend/queue`、`framework`、`profile`、`metrics` 等下游包，不反向被它们引用，属于 leaf 编排层。
