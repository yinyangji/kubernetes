本模块是 scheduler 的纯工具包，按职责拆分为三个文件（目录）：
- `utils.go`：无状态的调度期 API 客户端封装与对象操作工具，包括 `BindPod`、`PatchPodStatus`、`PatchPodGroupStatus`（基于 `strategicpatch.CreateTwoWayMergePatch` 生成差异并配合 `retry.OnError` 重试）、`Retriable`/`RetriableWithConflict` 错误分类、`GetHostPorts`、`IsScalarResourceName`/`IsDRAExtendedResourceName` 资源名校验、`MoreImportantPod`/`GetEarliestPodStartTime` 优先级比较、以及泛型 `As[T]` 类型断言辅助。
- `pod_resources.go`：仅暴露 `DefaultMilliCPURequest`、`DefaultMemoryRequest` 两个常量，用于零请求 Pod 的优先级计算。
- `assumecache/assume_cache.go`：独立的 `AssumeCache` 实现，包装一个 `cache.SharedInformer`，内部用 `objInfo` 同时持有 `latestObj`（可被 `Assume()` 覆盖的内存版本）和 `apiObj`（informer 同步的版本），通过 `sync.RWMutex` + `buffer.Ring[func()]` 事件队列 + `sync.Cond` 串行化事件分发，避免 handler 访问 cache 时死锁；对外暴露 `Get`/`List`/`Assume`/`Restore`/`AddEventHandler`，并提供 `AddTestObject`/`UpdateTestObject`/`DeleteTestObject` 测试注入点。
依赖方向：本模块不依赖 scheduler 核心包，仅依赖 `k8s.io/api`、`client-go`、`component-helpers/scheduling/corev1`、`apimachinery` 等基础库，供上层 scheduler 各阶段复用。
