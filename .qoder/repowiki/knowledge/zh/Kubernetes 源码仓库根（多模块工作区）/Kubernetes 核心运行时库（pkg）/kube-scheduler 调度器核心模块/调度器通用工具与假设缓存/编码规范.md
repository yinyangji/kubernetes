- 对 API Server 的写操作统一封装为带 `retry.OnError(retry.DefaultBackoff, ...)` 的重试函数，并通过 `Retriable`/`RetriableWithConflict` 白名单判定可重试错误。
- 需要并发安全的缓存结构使用 `sync.RWMutex` 保护数据，并通过独立的事件队列（如 `buffer.Ring[func()]`）+ `sync.Cond` 串行化事件派发，避免在持有写锁时调用可能再次加锁的 handler。
- 自定义错误采用 sentinel error + 具名 struct 组合的方式，并为每个错误类型实现 `Is(err) bool` 以支持 `errors.Is` 语义。
- 面向 informer 的缓存类提供 `AddTestObject`/`UpdateTestObject`/`DeleteTestObject` 等以 `Test` 后缀的写入入口，明确标注仅用于单元测试。
