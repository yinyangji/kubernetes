本模块是 kube-scheduler PostFilter 阶段的抢占子系统，按职责拆分为四层：
- 类型与抽象层（types.go）：定义 `Victim`/`DomainVictim`/`Candidate` 三个核心接口，将单 Pod 与跨节点 PodGroup 统一为可比较、可排序的抢占单元；`domain` 抽象了抢占作用域（单节点或集群），`candidateList` 提供带原子索引的并发候选收集。
- 评估器层（preemption.go + podgrouppreemption.go）：`Evaluator` 负责普通 Pod 抢占，通过 `Interface` 注入具体策略（默认在 scheduler 内部实现），流程为 `evaluate → findCandidates → DryRunPreemption(并行) → SelectCandidate → executor.actuate*`；`PodGroupEvaluator` 负责集群级抢占，先移除所有潜在 victim 后调用外部 `podGroupSchedulingFunc` 验证可行性，再尝试 reprieve 尽可能多的 victim。
- 执行器层（executor.go）：`Executor` 封装 actuation，支持同步与异步两种模式（由 `feature.Features.EnableAsyncPreemption` 控制），异步路径使用 goroutine + `parallelize` 并发删除 victim，并通过 `preempting`/`lastVictimsPendingPreemption` 双状态防止 preemptor 卡死；`PreemptPod` 钩子优先走 WaitingPod/preBind 内存拒绝，否则打 DisruptionTarget 条件并调用 API 删除。
- 工具层（util.go）：提供优先级计算、PDB 冲突判定（泛型 `FilterVictimsWithPDBViolation[T Victim]`）、victim 重要性排序等无状态辅助函数。
依赖方向单向：evaluator → executor → 框架 Handle/Lister/API；不反向依赖上层插件。
