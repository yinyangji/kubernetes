本模块是 `k8s.io/kube-scheduler/framework` 中 `APICall` 接口的内置实现集合，位于 `pkg/scheduler/framework/api_calls` 包下。
- 入口与注册：`api_calls.go` 定义两个常量类型 `PodStatusPatch`、`PodBinding`（类型为 `fwk.APICallType`），并导出全局变量 `Relevances`（按对象类型划分的优先级映射）和 `Implementations[*PodStatusPatchCall, *PodBindingCall]`（类型到构造函数的映射）。两者均标注可在 out-of-tree plugin 的 `init()` 中被替换。
- 具体实现：`pod_binding.go` 中的 `PodBindingCall` 通过 `util.BindPod` 执行绑定；`pod_status_patch.go` 中的 `PodStatusPatchCall` 使用 `util.PatchPodStatus` 对 Pod Status 做 patch，内部用 `sync.Mutex` + `executed` 标志协调 `Execute` 与 `Sync` 之间的竞态，并通过 `Merge` 合并条件与 NominatingInfo。
- 依赖方向：仅向上依赖 `k8s.io/kube-scheduler/framework` 定义的接口与类型，向下复用 `k8s.io/kubernetes/pkg/scheduler/util` 与 `k8s.io/kubernetes/pkg/api/v1/pod` 完成实际 etcd 写入，自身不持有 clientset，由框架在 `Execute(ctx, client)` 注入。
