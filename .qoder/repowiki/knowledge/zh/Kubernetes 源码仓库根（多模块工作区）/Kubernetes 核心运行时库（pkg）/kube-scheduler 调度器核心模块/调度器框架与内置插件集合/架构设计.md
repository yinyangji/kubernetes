本目录按职责切分为四层：
- 顶层 `interface.go`/`types.go`/`cycle_state.go` 定义 Framework 接口、NodeInfo/QueuedEntity/CycleState 等跨层共享类型（core_framework_api）；
- `runtime/` 实现插件注册表、扩展点生命周期与 Permit/PreBind 状态机（framework_runtime），由所有插件通过 registry 获取实例；
- `preemption/` 作为独立抢占引擎，在框架的 PreFilter/Filter/Score 阶段被 runtime 调用，选择并触发 Pod/PodGroup 抢占；
- `plugins/` 下每个子目录是一个可插拔实现，统一通过 `plugins/registry.go` 的 `RegisterDefaultPlugins` 注入到 runtime 的默认链；`api_calls/` 提供 APICall 接口的 PodBinding/PodStatusPatch 实现，`parallelize/` 为各插件提供 workqueue 并行封装。
四层通过 `Framework` 接口与 `CycleState` 解耦：插件只依赖接口与 CycleState 键值，不感知 runtime 内部状态机细节。
