本模块按 API Group 目录划分（core、apps、rbac、batch、networking、storage、admissionregistration、authentication、authorization、autoscaling、certificates、coordination、discovery、events、flowcontrol、node、policy、resource、scheduling、storagemigration、apiserverinternal），每个 Group 下再按资源子目录组织：
- `rest/storage_<group>.go`：该组的入口 Provider（如 `StorageProvider`、`RESTStorageProvider`、`legacyProvider`），实现 `genericapiserver.PostStartHookProvider` 与 `NewRESTStorage(apiResourceConfigSource, restOptionsGetter) (APIGroupInfo, error)`，负责调用子资源 `NewStorage/NewREST` 并把 `/<resource>`、`/<resource>/status`、`/<resource>/scale` 等映射到 `VersionedResourcesStorageMap`。
- `<resource>/storage/storage.go`：基于 `k8s.io/apiserver/pkg/registry/generic/registry.Store` 构造具体 REST，封装 etcd 访问；复杂资源（如 service、pod）在此额外注入分配器、代理、subresource 处理器。
- `<resource>/strategy.go`：实现 `rest.CreateStrategy/UpdateStrategy/DeleteStrategy/ResetFieldsStrategy`，定义校验、默认值、字段重置逻辑。
- `<resource>/authz.go`：针对 RBAC 聚合场景的授权策略（如 rbac 下的 policybased 包装）。
- `<resource>/doc.go`：声明 `+genclient:nonNamespaced` 等生成注解。
- 特殊路径：`core/rest/storage_core_generic.go` 提供通用 core 资源装配模板；`core/service/{ipallocator,portallocator,allocator}` 内嵌 IP/端口分配器及其 controller/repair 循环；`rbac/validation` 提供规则解析器供 policybased 存储使用；`registrytest` 提供测试辅助（etcd、endpoint、service 等）。依赖方向单向：`rest` → `storage` → `strategy/authz` → apiserver generic registry，不反向引用上层 server。
