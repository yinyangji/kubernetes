- 每个资源目录固定包含 `storage/storage.go`（构造 `*Store`）、`strategy.go`（实现 Create/Update/Delete/ResetFields Strategy）、`doc.go`（genclient 注解）三件套，测试文件以 `_test.go` 配对存放。
- Group 级入口统一暴露 `NewRESTStorage(apiResourceConfigSource, restOptionsGetter) (APIGroupInfo, error)`，内部用 `apiResourceConfigSource.ResourceEnabled(GV.WithResource(...))` 做条件开关，再将返回的 `Deployment/Status/Scale` 等对象挂入 `storage["<resource>"]` / `storage["<resource>/status"]` 键。
- 复杂资源在 `storage/storage.go` 中通过 `store.BeginCreate/BeginUpdate/AfterDelete/Decorator` 钩子注入业务前置逻辑（如 Service 的 IP/端口分配、Pod 的日志/执行代理），而非直接重写 Store 方法。
- 需要集群初始化或后台协程的资源组（core、rbac、certificates 等）实现 `PostStartHookProvider`，在 `PostStartHook()` 中返回 HookName 与 `PostStartHookFunc`，由 apiserver 启动时串行执行。
- RBAC 相关资源采用 `policybased.NewStorage(底层Storage, Authorizer, RuleResolver)` 包装模式，将权限检查与数据存取解耦。
