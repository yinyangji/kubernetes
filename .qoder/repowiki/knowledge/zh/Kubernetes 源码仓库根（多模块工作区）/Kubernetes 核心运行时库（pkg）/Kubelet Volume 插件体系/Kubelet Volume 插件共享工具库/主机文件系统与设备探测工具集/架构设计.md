模块以 `HostUtils` 接口（hostutil.go）为中心，定义 DeviceOpened / PathIsDevice / MakeRShared / GetFileType / PathExists / EvalHostSymlinks / GetOwner / GetSELinuxSupport / GetMode / GetSELinuxMountContext 等统一方法；具体实现按构建标签拆分：`hostutil_linux.go` 使用 `golang.org/x/sys/unix` 调用 `O_EXCL` 检测独占打开、解析 `/proc/self/mountinfo` 判断 rshared 传播并通过 `syscall.Mount(MS_BIND|MS_SHARED)` 修正挂载点，同时借助 `github.com/opencontainers/selinux/go-selinux` 读取 seclabel/context 选项；`hostutil_windows.go` 对设备相关方法返回 false/no-op，仅保留路径存在性与基本权限查询；`hostutil_unsupported.go` 为其他平台提供空实现。`fake_hostutil.go` 提供基于内存映射的 `FakeHostUtil` 供单元测试替换真实系统调用。公共辅助函数如 `ExclusiveOpenFailsOnDevice`、`DoMakeRShared`、`GetSELinux`、`getDeviceNameFromMount` 被多文件复用，Linux 与 Windows 共享同一份 `getDeviceNameFromMount` 逻辑但各自处理路径归一化差异。依赖方向单向指向 `k8s.io/mount-utils`、`k8s.io/utils/path`、`klog/v2`，本模块不反向依赖上层 volume 代码。
