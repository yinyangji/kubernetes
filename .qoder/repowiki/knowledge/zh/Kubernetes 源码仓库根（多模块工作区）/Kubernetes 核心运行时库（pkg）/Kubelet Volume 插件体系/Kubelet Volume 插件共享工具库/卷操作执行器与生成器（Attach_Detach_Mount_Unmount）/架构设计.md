模块采用「执行器 + 生成器」两层拆分：
- `operation_executor.go` 暴露 `OperationExecutor` 接口，内部持有 `nestedpendingoperations.NestedPendingOperations` 作为全局锁，确保同一 volume/pod/node 三元组上不会并行触发相同操作；所有公开方法（AttachVolume/DetachVolume/MountVolume/UnmountVolume/UnmountDevice/VerifyVolumesAreAttached/ExpandInUseVolume/ReconstructVolumeOperation）先通过 `operationGenerator` 生成 `volumetypes.GeneratedOperations`，再交给 `pendingOperations.Run` 异步执行，失败仅记录日志，由上层重试。
- `operation_generator.go` 实现 `OperationGenerator` 接口，封装对 `volume.VolumePluginMgr`、`clientset.Interface`、`record.EventRecorder`、`BlockVolumePathHandler` 的调用，按 Filesystem/Block 两种 volumeMode 分支构造挂载、卸载、映射、设备挂载/卸载、控制器附加校验等操作函数，并通过 `util.OperationCompleteHook` 注入指标钩子。
- `node_expander.go` 独立实现 `NodeExpander`，负责在 MountVolume 流程中调用 `volume.NodeExpandableVolumePlugin.NodeExpand`，结合 PVC 的 `AllocatedResourceStatuses` 做前置检查、PVC 状态标记（InProgress/Failed/Finished/Infeasible）以及 RWX 多节点扩容的特殊路径。
- `fakegenerator.go` 提供测试用的 `FakeOperationGenerator`。
依赖方向单向：executor → generator → volume plugin manager / API server / host utils；对外只暴露 executor 接口与若干数据载体（`VolumeToAttach`、`VolumeToMount`、`MountedVolume`、`AttachedVolume`、`MarkVolumeOpts`、`ActualStateOfWorld*Updater` 接口），不直接依赖 kubelet 上层组件。
