- 通过 `//go:build linux` / `//go:build windows` 等构建标签在同一包内按平台拆分实现，共享同一 `Interface` 与 `Subpath` 结构体。
- 对外只暴露 `Interface` + `New(mount.Interface) Interface` 工厂函数，平台相关细节封装在私有 `subpath` 结构体中。
- 所有路径操作前先 `filepath.EvalSymlinks` 或显式检查 `ModeSymlink`，并用 `mount.PathWithinBase` / `StartsWithBackstep` 校验最终路径仍在 base 内。
- 对需要跨竞态窗口保持稳定的资源（Linux 的 fd、Windows 的文件句柄）通过闭包形式的 `cleanupAction func()` 或显式 handle 列表返回给调用方管理生命周期。
- 关键可变行为（如 `lazyUnmountFn`）以包级变量形式暴露，便于测试时替换而不 fork 二进制。
