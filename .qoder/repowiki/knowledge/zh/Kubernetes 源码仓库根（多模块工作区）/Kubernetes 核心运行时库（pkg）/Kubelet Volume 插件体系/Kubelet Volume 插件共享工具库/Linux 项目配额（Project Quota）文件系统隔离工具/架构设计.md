模块按平台切分为三个层次：
- `common/`：跨平台共享类型与接口。`quota_common.go` 定义通用 `QuotaID` 及特殊值；`quota_common_linux.go` 声明 `LinuxVolumeQuotaProvider` / `LinuxVolumeQuotaApplier` 两个面向实现的接口以及 `MountsFile`、`FirstQuota` 等常量；`quota_common_linux_impl.go` 提供默认实现 `VolumeProvider` + `linuxVolumeQuotaApplier`，通过 `statfs` magic 识别 XFS/ext4，并调用外部命令 `/sbin/xfs_quota`、`/usr/bin/lsattr` 完成实际配额操作。
- 顶层 `quota.go` 暴露 `Interface`（GetQuotaOnDir / SupportsQuotas / AssignQuota / GetConsumption / GetInodes / ClearQuota），并通过 `enabledQuotasForMonitoring()` 受 FeatureGate `LocalStorageCapacityIsolationFSQuotaMonitoring` 控制开关。
- 平台实现文件以 build tag 区分：`quota_linux.go` + `project.go` 是 Linux 主实现，负责路径→mountpoint→backing device 的探测缓存、设备级 applier 选择（`devApplierMap`）、目录级 applier 缓存（`dirApplierMap`）、Pod UID ↔ quota ID 的多对多映射与引用计数（`podDirCountMap`），以及 `/etc/projects` / `/etc/projid` 的原子写入（先写临时文件再 `os.Rename`，并用 `unix.Flock` 加锁）。`quota_unsupported.go` 在非 Linux 下返回 `errNotImplemented` 占位实现。
- 依赖方向：顶层 → `common` 接口；Linux 实现 → `common` 默认实现；`project.go` 仅依赖 `common.QuotaID`，不直接调用外部命令。
