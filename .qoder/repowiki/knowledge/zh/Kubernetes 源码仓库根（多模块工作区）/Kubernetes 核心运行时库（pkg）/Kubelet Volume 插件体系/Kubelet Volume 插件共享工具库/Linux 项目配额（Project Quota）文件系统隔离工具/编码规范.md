- 跨平台差异通过 Go build tag 分离：`//go:build linux` 与 `//go:build !linux` 分别提供完整实现与返回 `errNotImplemented` 的占位函数，保持包 API 一致。
- 对外部可执行命令（xfs_quota、lsattr）采用“首次发现后缓存”模式，配合 `sync.RWMutex` 保护全局变量，避免重复 stat 查找。
- 对共享状态映射（如 `dirApplierMap`、`quotaSizeMap`、`supportsQuotasMap`、`backingDevMap`）统一使用独立的 `sync.RWMutex` 细粒度加锁，而非单一大锁。
- 修改系统文件（`/etc/projects`、`/etc/projid`）遵循“读入内存 → 标记无效行 → 写入同权限临时文件 → `os.Rename` 原子替换”的原子更新模式，并在打开时先用 `Lstat` 校验非符号链接。
- 错误码约定使用 `common.BadQuotaID` 表示无效/未知配额 ID，`UnknownQuotaID = -1` 表示无法判断是否处于配额中，调用方据此区分“无配额”和“检查失败”。
