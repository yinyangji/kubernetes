- 所有对外错误信息统一通过本地 `log(msg, parts...)` 包装，自动加上 `kubernetes.io/csi:` 前缀以便日志过滤。
- 对 CSI gRPC 调用的错误一律用 `isFinalError` 判断是否属于 Canceled/DeadlineExceeded/Unavailable/ResourceExhausted/Aborted 等可重试状态，非终态错误再包装为 `volumetypes.NewUncertainProgressError` 交由上层重试。
- 需要访问 API Server 或 Listers 的路径先通过类型断言检查 host 是否实现 `volume.KubeletVolumeHost` / `volume.CSIDriverVolumeHost` / `volume.AttachDetachVolumeHost`，缺失则回退为 standalone 模式并记录 Warning。
- 跨进程恢复场景（Unmounter/Unmapper/Construct*Spec）采用“创建时写 `vol_data.json`，销毁时读该文件”的磁盘持久化约定，键名集中在 `volDataKey.*` 常量中。
- CSI 驱动名称与地址使用强类型别名 `csiDriverName string` / `csiAddr string`，避免裸字符串传递导致语义丢失。
