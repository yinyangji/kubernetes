本模块是 Kubelet 中 `kubernetes.io/csi` 插件的核心实现，围绕四个文件形成清晰分层：
- `csi_plugin.go`：对外暴露 `ProbeVolumePlugins()` 并实现 `volume.VolumePlugin` / `AttachableVolumePlugin` / `BlockVolumePlugin` 等接口；通过全局 `RegistrationHandler` 响应 kubelet pluginwatcher 的 Validate/Register/DeRegister 回调，将驱动端点与最高支持版本写入全局 `DriversStore`，并在注册后调用 `nodeinfomanager.InstallCSIDriver` 同步到 CSINode；同时维护 `csiNodeUpdaterVar` 以在 `MutableCSINodeAllocatableCount` 特性开启时动态更新 CSINode。
- `csi_client.go`：定义 `csiClient` 接口并由 `csiDriverClient` 基于 `github.com/container-storage-interface/spec/lib/go/csi` 的 gRPC Node 服务实现所有 Node RPC（Stage/Publish/Expand/Unstage/Unpublish/GetStats/Capabilities），使用 `newGrpcConn` 经 unix socket + insecure credentials 连接，并通过 `csiClientGetter` 做带 RWMutex 的双检缓存；错误分类由 `isFinalError` 统一处理，区分可重试与终态错误。
- `csi_drivers_store.go`：线程安全的 `DriversStore`（内部 `store map[string]Driver` + `sync.RWMutex`），保存每个已注册 CSI Driver 的 endpoint 与最高 CSI 版本，作为驱动发现的全局单例。
- `csi_util.go`：提供跨文件复用的工具函数——从 `volume.Spec` 解析 CSIVolumeSource/CSIPersistentVolumeSource、持久化/加载 `vol_data.json` 元数据、构造 `/plugins/kubernetes.io/csi/volumeDevices/{specVolID}/{dev,data}` 路径、读取 Secret 注入 secrets、生成 PodInfo 属性等。
依赖方向：plugin → client/store/util；client 仅依赖 CSI gRPC 协议包与 metrics 拦截器；util 被其余三文件共同消费，不反向依赖。
