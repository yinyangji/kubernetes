本模块由三个面向不同生命周期的结构体组成，均通过接口断言声明与上层 volume 框架的契约：
- `csiAttacher`（csi_attacher.go）：实现 `volume.Attacher` / `volume.Detacher` / `volume.DeviceMounter` / `volume.DeviceUnmounter`。Attach/Detach 不直接操作底层存储，而是创建/删除 `VolumeAttachment` CRD，并通过 lister 轮询其 `Status.Attached` 或等待对象被删除来确认完成；`MountDevice`/`UnmountDevice` 仅在驱动支持 `STAGE_UNSTAGE_VOLUME` 时调用 `NodeStageVolume`/`NodeUnstageVolume`，并将元数据持久化到 `{pluginDir}/{driver}/{sha256(volHandle)}/globalmount/vol_data.json`。
- `csiMountMgr`（csi_mounter.go）：实现 `volume.Volume` / `volume.Mounter` / `volume.Unmounter`。根据 `CSIPersistentVolumeSource` 或 inline `VolumeSource` 分支处理 Persistent/Ephemeral 两种模式，调用 `NodePublishVolume`/`NodeUnpublishVolume`；当驱动不支持 `VOLUME_MOUNT_GROUP` 且满足 FSGroup 条件时回退为 Kubelet 自行 `chown`；同时注入 Pod 信息与服务账户令牌到 `volume_attributes` 或 `node_publish_secrets`。
- `csiBlockMapper`（csi_block.go）：实现 `volume.BlockVolumeMapper` / `CustomBlockVolumeMapper` / `BlockVolumeUnmapper` / `CustomBlockVolumeUnmapper`，负责 block 卷的 staging/publish/unstage/unpublish 四阶段，路径遵循 `volumeDevices/{specName}/{staging|publish}` 约定，并在 `TearDownDevice` 中清理孤儿文件。
依赖方向：三者都持有 `*csiPlugin` 和 `kubernetes.Interface`，通过 `csiClientGetter` 获取 gRPC 客户端调用 CSI Node Service；对 API Server 仅读取 `VolumeAttachment`/`CSIDriver`/Secret，写操作仅限 `Attach` 阶段创建 VA。错误统一包装为 `volumetypes.TransientOperationFailure` 以触发 kubelet 重试。
