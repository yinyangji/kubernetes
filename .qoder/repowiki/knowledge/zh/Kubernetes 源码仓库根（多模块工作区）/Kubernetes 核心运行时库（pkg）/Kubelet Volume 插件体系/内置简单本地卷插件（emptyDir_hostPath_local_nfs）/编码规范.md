- 每个包暴露包级函数 `ProbeVolumePlugins(...)` 返回 `[]volume.VolumePlugin`，作为卷框架发现该插件的唯一入口。
- 插件结构体通过 `var _ volume.VolumePlugin = &xxxPlugin{}` 编译期断言实现接口，并在 `Init(host)` 中注入 `VolumeHost`。
- 通过 `CanSupport(spec)` 检查 `spec.Volume.*` 或 `spec.PersistentVolume.Spec.*` 是否为对应 Source 类型来决定是否认领该 spec。
- Mounter/Unmounter 成对定义（如 `hostPathMounter`/`hostPathUnmounter`），且 `TearDown()` 一律委托给 `TearDownAt(GetPath())` 以保持行为一致。
- 路径构造统一通过 `host.GetPodVolumeDir(uid, utilstrings.EscapeQualifiedName(pluginName), volName)` 生成 Pod 卷目录，避免硬编码路径。
- SetUp 流程遵循“先 `IsNotMountPoint` 判断幂等 → 必要时 `MkdirAll` → 调用 `MountSensitiveWithoutSystemd` → 失败时回滚 unmount 并删除目录”的模式。
