两个子包 `fc` 与 `iscsi` 结构完全对称，均通过 `ProbeVolumePlugins()` 暴露单一插件实例，并实现 `volume.VolumePlugin` / `volume.PersistentVolumePlugin` / `volume.BlockVolumePlugin` 三个接口。

每个包内部按职责拆分为四个文件：
- `*.go`：定义 `*Plugin`（如 `fcPlugin`/`iscsiPlugin`），实现插件生命周期方法（`Init`、`GetPluginName`、`CanSupport`、`NewMounter`、`NewBlockVolumeMapper`、`NewUnmounter`、`NewBlockVolumeUnmapper`、`ConstructVolumeSpec`、`ConstructBlockVolumeSpec`）；同时维护一个共享的 `*Disk` 结构体（`fcDisk`/`iscsiDisk`）承载 WWN/LUN 或 Portal/IQN/Lun 等卷标识。
- `attacher.go`：实现 `volume.Attacher`/`Detacher` 及 `DeviceMounter`/`DeviceUnmounter`，由外部 Attach/Detach 控制器调用，核心在 `WaitForAttach` 中委托给 `diskManager.AttachDisk`，并在 `UnmountDevice` 中执行最终卸载与目录清理。
- `disk_manager.go`：声明 `diskManager` 抽象接口（`MakeGlobalPDName`、`MakeGlobalVDPDName`、`AttachDisk`、`DetachDisk`、`DetachBlock*Disk`），并提供通用的 `diskSetUp` 函数完成“检查是否已挂载 → mkdir → bind mount 到 Pod volumeDir → 可选 FSGroup 权限变更”的流程。
- `*_util.go`：提供 `diskManager` 的具体实现（`fcUtil`/`ISCSIUtil`），封装对 `multipath`、`iscsiadm`、`sg_inq` 等系统命令的调用；`*_test.go` 对应 mock 这些 util。

依赖方向：`plugin` → `attacher/detacher` → `diskManager` 接口 → `*_util` 具体实现；mount/exec/device handler 通过构造时注入（`host.GetMounter()`、`utilexec.New()`、`util.NewDeviceHandler`），使测试可替换。iSCSI 额外通过 `keymutex.KeyMutex` 对同一 target 做并发锁，并通过 Secret 读取 CHAP 认证信息。两者都支持 SELinux 上下文挂载（受 `features.SELinuxMountReadWriteOncePod` 特性门控制）。
