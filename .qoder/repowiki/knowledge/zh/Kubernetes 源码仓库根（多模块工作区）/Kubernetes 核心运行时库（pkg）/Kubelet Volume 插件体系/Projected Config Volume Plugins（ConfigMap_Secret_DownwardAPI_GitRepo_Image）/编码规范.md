- 每个插件包都导出 `ProbeVolumePlugins() []volume.VolumePlugin` 作为注册入口，并定义形如 `<type>PluginName = "kubernetes.io/<type>"` 的常量。
- 路径计算统一通过 `getPath(uid, volName, host)` 封装 `host.GetPodVolumeDir(uid, utilstrings.EscapeQualifiedName(pluginName), volName)`，避免直接拼接字符串。
- `CanSupport` 采用 `spec.Volume != nil && spec.Volume.<Type> != nil` 的模式匹配；`RequiresRemount` 对 configmap/secret/downwardapi 返回 true，git_repo/image 返回 false。
- `SetUpAt` 固定流程：先 `host.NewWrapperMounter(wrappedVolumeSpec())` 创建 EmptyDir，再 `volumeutil.MakeNestedMountpoints`，随后用 `volumeutil.NewAtomicWriter(dir, ...).Write(payload, setPerms)` 原子落盘，失败时 defer 中构造 Unmounter 清理。
- 权限变更统一通过 `volume.NewVolumeOwnership(..., volumeutil.FSGroupCompleteHook(b.plugin, nil)).ChangePermissions()` 完成，而非直接 chown/chmod。
- 可选资源（Secret/ConfigMap 的 Optional、ClusterTrustBundle.Optional）在 API 不存在时构造空对象继续走后续逻辑，错误收集使用 `utilerrors.NewAggregate(errlist)` 汇总。
