模块按职责分为两层：
- `evaluator/core`：具体资源评估器实现，每个资源一个文件（pods.go、services.go、persistent_volume_claims.go、resource_claims.go），统一通过 `quota.Evaluator` 接口暴露 `GroupResource/Handles/Matches/MatchingResources/Usage/UsageStats` 等方法；`registry.go` 中的 `NewEvaluators` 是入口工厂，根据 `isEnabled` 回调与 FeatureGate 动态组装 evaluator 列表，并为 configmaps/secrets/replicationcontrollers/resourcequotas 等历史资源注册 `generic.NewObjectCountEvaluator` 别名。
- `install`：对外装配层，提供 `NewQuotaConfigurationForAdmission` 与 `NewQuotaConfigurationForControllers` 两个构造器，分别对接 admission 控制器与后台 controller，内部调用 `core.NewEvaluators` 并注入 `DefaultIgnoredResources()` 忽略虚拟资源（bindings、reviews、events 等）。
依赖方向单向：`install` → `evaluator/core` → `k8s.io/apiserver/pkg/quota/v1/generic`；`update_filter.go` 仅作为 informer 事件过滤桥接，复用 core 包的 `IsTerminating/QuotaV1Pod/GetQuotaServiceType/RequiresQuotaReplenish` 等导出函数判断是否需要 replenish。
