- 每个资源评估器以独立文件 + 结构体实现 `quota.Evaluator` 接口，并在文件末尾用 `var _ quota.Evaluator = &xxxEvaluator{}` 做编译期接口校验。
- 通过 `NewXxxEvaluator(f, ...)` 构造函数接收 `quota.ListerForResourceFunc` 与可选 clock/informer，避免在 evaluator 内直接持有 clientset。
- 对可开关的资源使用 `isEnabled(gvr)` 回调包裹构造逻辑，配合 `utilfeature.DefaultFeatureGate.Enabled(features.XXX)` 双重条件决定是否启用。
- Scope 匹配逻辑集中在 `podMatchesScopeFunc` 等 switch 函数中，按 `corev1.ResourceQuotaScope*` 常量分支处理，未命中时返回 false。
- 历史对象计数类资源通过 `legacyObjectCountAliases` 映射到 `generic.ObjectCountQuotaResourceNameFor` 生成的别名，保持向后兼容。
