本包是独立的 Go 包，通过 `Install(*restful.Container)` 方法将 WebService 注册到外部传入的 go-restful Container，遵循“可插拔路由”模式。
- `logs.go`：定义 `Logs` 结构体，挂载 `/logs/` 路径，使用 `{logpath:*}` 通配参数配合 `http.ServeFile` 直接暴露宿主机 `/var/log` 目录，并通过 `logFileNameIsTooLong` 拦截超长文件名避免 500 误报。
- `openidmetadata.go`：定义 `OpenIDMetadataServer`，依赖 `serviceaccount.OpenIDMetadataProvider` 接口获取 JSON 配置与 JWKS；按注释说明，该包放在 routes 是为了让上层 controller 能 import `pkg/serviceaccount` 而不被 import-boss 禁止引入 go-restful，因此仅做 HTTP 渲染，密钥逻辑全部委托给 serviceaccount 包。
- `const_other.go` / `const_windows.go`：通过 build tag 区分平台常量 `fileNameTooLong`（非 Windows 用 `syscall.ENAMETOOLONG`，Windows 用 `golang.org/x/sys/windows.ERROR_FILENAME_EXCED_RANGE`），供 logs 路由复用。
- 依赖方向：routes → restful + klog + serviceaccount，不反向被 serviceaccount 依赖，保持单向边界。
