模块围绕三个核心接口组织：
- `DockerConfigProvider`（provider.go）：定义 `Enabled()/Provide(image)` 扩展点，内置 `defaultDockerConfigProvider` 读取本地 `.dockercfg`/`.docker/config.json`，以及可组合的 `CachingDockerConfigProvider` 带 TTL 缓存。
- `DockerKeyring`（keyring.go）：维护 registry→凭据的反向索引，实现 `Lookup(image) ([]TrackedAuthConfig, bool)`；`BasicDockerKeyring` 基于排序后的 index 做最具体路径优先匹配，`providersDockerKeyring` 聚合多个 Provider，`UnionDockerKeyring` 级联多个 Keyring。
- `plugin.Plugin`（plugin/plugin.go）：通过 `RegisterCredentialProviderPlugins` 按 `CredentialProviderConfig` 发现并启动外部二进制插件，以 stdin/stdout JSON 协议与 v1/v1alpha1/v1beta1 版本的 `CredentialProviderRequest/Response` 通信，内部用 `singleflight.Group` 去重并发调用，并使用 `cache.NewExpirationStore` 按 Image/Registry/Global 三级粒度缓存结果，可选注入 Pod ServiceAccount 绑定 token 作为鉴权上下文。

子包职责边界：
- `secrets/`：仅负责把 `v1.Secret`（`kubernetes.io/dockerconfigjson` / `kubernetes.io/dockercfg`）转成 `BasicDockerKeyring` 并与默认 keyring 合并，不直接读磁盘。
- `plugin/`：独立于 core 类型，依赖 `k8s.io/kubelet/pkg/apis/credentialprovider` 的 API 版本，通过 `install.Install(scheme)` 注册编解码器。

依赖方向：`plugin` → `credentialprovider`（core 类型），`secrets` → `credentialprovider`，core 不反向依赖二者，形成稳定的扩展边界。
