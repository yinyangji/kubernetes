- 对外扩展点均以 interface 形式暴露（`DockerConfigProvider`、`DockerKeyring`、`Plugin`），并通过组合或 Union 模式拼装实现，避免硬编码分支。
- 配置/凭据文件读取统一采用“先 config.json，再回退 .dockercfg”的顺序，并在每个候选路径上记录 klog.V(4) 日志便于诊断。
- registry 匹配一律复用 `URLsMatchStr`/`URLsMatch` 配合 `ParseSchemelessURL`，对 `/v1/`、`/v2/` 前缀做规范化后再比较，保证与 docker 客户端行为一致。
- 需要跨请求共享状态时显式声明 `sync.Mutex` 或使用 `singleflight.Group` 去重，并对缓存项附带 `expiresAt` 配合 `cache.ExpirationPolicy` 自动清理。
- 敏感字段（如 ServiceAccount token）在参与 cache key 计算前先经 `getHashIfNotEmpty` 取 sha256，避免明文泄露。
