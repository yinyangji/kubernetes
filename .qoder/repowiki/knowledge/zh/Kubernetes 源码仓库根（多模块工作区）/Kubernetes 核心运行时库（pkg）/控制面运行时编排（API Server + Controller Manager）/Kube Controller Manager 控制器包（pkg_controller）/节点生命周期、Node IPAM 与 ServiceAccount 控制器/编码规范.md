- 控制器构造器统一采用 `NewXxxController(logger, informers..., client, options)` 形式，并在构造函数中完成 Informer 事件注册、Lister 缓存与 HasSynced 闭包的保存。
- Run 方法固定以 `defer utilruntime.HandleCrashWithContext(ctx)` 开头，随后调用 `cache.WaitForNamedCacheSyncWithContext` 等待所有 Informer 同步后再启动 worker goroutine。
- 工作队列一律使用 `workqueue.NewTypedRateLimitingQueueWithConfig` + `DefaultTypedControllerRateLimitier[T]()` 创建强类型队列，并按不同资源维度拆分多个队列（如 tokens controller 的 serviceAccountQueue 与 secretQueue）。
- 配置对象放在独立的 `config/` 包下，并提供 `config/v1alpha1` 子目录含 `conversion.go`、`defaults.go`、`register.go` 以及 `zz_generated.conversion.go`/`zz_generated.deepcopy.go` 自动生成文件，形成标准的 versioned config 模式。
- 错误处理统一走 `utilruntime.HandleErrorWithLogger` / `HandleErrorWithContext`，并将可重试项通过 `AddRateLimited` 重新入队而非直接返回。
