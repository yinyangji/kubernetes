本模块包含三个并列的控制器子包，各自遵循相同的 informer+workqueue 经典模式，但职责互补：
- `endpoint/`：监听 Pod/Service/Endpoints，按 Service.Spec.Selector 计算并维护 v1.Endpoints 资源；通过 `staleEndpointsTracker` 检测 informer 缓存过期，使用 `TriggerTimeTracker` 输出 `EndpointsLastChangeTriggerTime` 注解，并在超过 `maxCapacity=1000` 时截断 subsets 并打上 `endpoints.kubernetes.io/over-capacity` 注解。
- `endpointslice/`：监听 Pod/Service/Node/EndpointSlice，调用 `k8s.io/endpointslice` 包的 `Reconciler` 生成 discovery.EndpointSlice；额外维护 Node 拓扑分布以支持 TopologyAwareHints，并通过 `topologycache.TopologyCache` + 独立 topologyQueue 触发重平衡。
- `endpointslicemirroring/`：监听 Endpoints/Service/EndpointSlice，仅对无 Selector 且未 skip-mirror 的 Endpoints 做镜像，内部 `reconciler.reconcile()` 按端口映射把每个 EndpointSubset 映射到单个 EndpointSlice（受 `maxEndpointsPerSubset` 限制），并通过本地 `metrics.Cache` 统计效率指标。

每个子包都提供独立的 `config/types.go`（含 `v1alpha1` 版本及 conversion/defaults/register），由上层 kube-controller-manager 注入配置；Controller 结构体统一持有 clientset、eventBroadcaster/recorder、多个 Listers、TypedRateLimitingInterface 队列、`endpointUpdatesBatchPeriod` 批处理延迟以及 `endpointSliceTracker` 用于追踪期望的 EndpointSlice 集合以判断是否 stale。错误处理统一采用 `maxRetries=15` 指数退避 + `AddRateLimited`，并在达到上限后 drop 项。
