模块以泛型结构 `ClusterTrustBundlePublisher[T clusterTrustBundle]` 为核心，通过类型约束同时支持 certificates/v1、v1beta1、v1alpha1 三个 API 版本；具体版本的差异由 `clusterTrustBundleHandlers` 接口（`gaHandlers`/`betaHandlers`/`alphaHandlers`）实现，分别负责创建、更新、比较和命名该版本的 ClusterTrustBundle。

- 入口：`NewGAClusterTrustBundlePublisher` / `NewBetaClusterTrustBundlePublisher` / `NewAlphaClusterTrustBundlePublisher` 构造对应版本的 informer、listers、clientset 子资源并调用统一的 `newClusterTrustBundlePublisher`。
- 事件源：对 ClusterTrustBundle informer 的 Add/Update/Delete 事件以及 `dynamiccertificates.CAContentProvider` 的 Listener 都会向 `workqueue.TypedRateLimitingInterface[string]` 入队空 key，触发单 worker 循环 `runWorker -> processNextWorkItem -> syncClusterTrustBundle`。
- 同步逻辑：从 `ca.CurrentCABundleContent()` 读取 CA PEM，用 `constructBundleName`（signerName + SHA256 前 12 字节）计算目标名称，若不存在则 Create，若存在但 `.spec.TrustBundle` 不同则 Update；随后 List 所有 ClusterTrustBundle，删除与本 signer 无关的旧 bundle，保证幂等且仅保留一个。
- 指标：`metrics.go` 在 `init()` 中注册 `sync_total` 与 `sync_duration_seconds` 两个 ALPHA 级别指标，按 HTTP status code 打标签，由 `recordMetrics` 在每次 sync 结束时调用。
- 依赖方向：本模块只读 `dynamiccertificates.CAContentProvider` 与 client-go 的 certificates v1/v1beta1/v1alpha1 informer/lister/clientset，不反向依赖上层 controller 框架，通过 `PublisherRunner.Run(ctx)` 暴露生命周期。
