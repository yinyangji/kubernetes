本模块由三个相互独立的控制器包组成，均遵循 Kubernetes 标准控制器模式（Informer-Lister + workqueue.TypedRateLimitingInterface + Run(ctx, workers)）：

- `ttl/`：监听 Node 事件，维护 nodeCount/boundaryStep/desiredTTLSeconds 三状态，按预置边界表计算并 StrategicMergePatch 节点上的 `ObjectTTLAnnotationKey`，供 Kubelet 用作缓存建议。
- `ttlafterfinished/`：监听 Batch/v1 Job，仅对已终止且设置了 `spec.ttlSecondsAfterFinished` 的 Job 通过 `enqueueAfter` 延迟入队；到期后以 UID 前置条件 Foreground 级联删除，并通过 `metrics.JobDeletionDurationSeconds` 暴露耗时指标。其 `config/v1alpha1` 子目录使用 controller-gen 生成的 conversion/deepcopy 代码提供自定义配置类型。
- `devicetainteviction/`：最复杂的一个，同时监听 Pod、ResourceClaim、ResourceSlice、DeviceClass 及可选 DeviceTaintRule（受 FeatureGate 控制），在内存中维护 pools（按 driver/pool 聚合最新 generation 的 ResourceSlice）、allocatedClaims、deletePodAt 等索引，将阻塞性 API 调用下沉到 `tainteviction.TimedWorkerQueue`，事件处理函数只加锁更新内存结构并 enqueue workItem；通过 `uidCache` 去重已驱逐 Pod，并通过 ApplyStatus 周期性更新 DeviceTaintRule 的 `EvictionInProgress` Condition。

依赖方向：三者都依赖 `k8s.io/kubernetes/pkg/controller` 中的通用工具（如 `KeyFunc`），`devicetainteviction` 额外复用 `pkg/controller/tainteviction` 的 TimedWorkerQueue 与 NamespacedObject 抽象；每个包内 `metrics/` 子包负责注册 Prometheus 指标，`config/v1alpha1` 子包负责 CRD 类型的版本化与转换。
