- 控制器统一采用 `New*Controller(client, informer) -> Run(ctx, workers)` 接口，内部使用 `wait.UntilWithContext` 驱动 worker 循环并以 `sync.WaitGroup` 协调多 worker 退出。
- worker 中先通过 Lister.List(labels.Everything()) 拉取全量资源，再逐个调用 handle 方法，错误通过 `klog.FromContext(ctx).Error` 或 `utilruntime.HandleErrorWithContext` 记录而不中断遍历。
- 清理判定函数以 `is*` 前缀命名且仅返回布尔值，由 `handle` 集中组合判断后再执行 Delete，保持“判定”与“副作用”分离。
- 删除操作使用 `metav1.Preconditions.UID` 作为幂等保护，并对 `IsNotFound` 视为成功处理，避免重复删除报错。
