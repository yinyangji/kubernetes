本模块提供两个独立的、基于 informer/listers 的控制器：
- `cleaner.go` 中的 `CSRCleanerController`：依赖 `certificates/v1` 的 `CertificateSigningRequestInterface` 与 `Lister`，通过 `wait.UntilWithContext` 以固定 `pollingInterval=1h` 轮询全量 CSR，对每个对象依次调用 `isIssuedPastDeadline`/`isDeniedPastDeadline`/`isFailedPastDeadline`/`isPendingPastDeadline`/`isIssuedExpired`/`isApprovedUnissuedPastDeadline` 判定是否满足清理条件（已签发超期、被拒绝/失败超期、Pending 超期、已签发但证书 NotAfter 过期、已批准但未签发超期），满足则调用 `Delete`。
- `pcrcleaner.go` 中的 `PCRCleanerController`：针对 `certificates/v1beta1` 的 `PodCertificateRequest`，构造带 UID 的 `Preconditions.DeleteOptions` 做幂等删除，避免并发误删；支持注入 `clock.PassiveClock` 以便测试。
两个控制器均遵循 Kubernetes 标准控制器模式：构造函数接收 client/informer，暴露 `Run(ctx, workers)` 入口，内部用 `sync.WaitGroup` 管理 worker goroutine，并通过 `utilruntime.HandleCrash` 捕获 panic。依赖方向单向指向 `client-go` 的 certificates v1/v1beta1 typed client、informer 与 lister，不反向依赖上层控制面逻辑。
