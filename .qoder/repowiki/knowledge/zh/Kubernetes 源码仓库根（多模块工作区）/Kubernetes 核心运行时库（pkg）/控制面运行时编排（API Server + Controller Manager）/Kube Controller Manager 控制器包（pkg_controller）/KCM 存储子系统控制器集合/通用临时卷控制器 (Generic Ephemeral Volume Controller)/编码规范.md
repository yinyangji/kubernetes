- 控制器通过 `NewController(ctx, client, podInformer, pvcInformer)` 工厂函数返回抽象 `Controller` 接口，隐藏内部 `ephemeralController` 结构体。
- 事件处理中所有非预期错误统一经 `runtime.HandleError` 记录，避免 panic 传播到 informer 回调。
- Pod 更新与删除事件一律忽略（spec 不可变、PVC 回收走 OwnerReference/GC），仅在 Add 与 PVC Delete 上挂 handler。
- 对外暴露的配置类型放在独立 `config/` 子包并按 v1alpha1 分组，配套 `defaults.go`、`conversion.go` 与 `zz_generated.deepcopy.go`。
