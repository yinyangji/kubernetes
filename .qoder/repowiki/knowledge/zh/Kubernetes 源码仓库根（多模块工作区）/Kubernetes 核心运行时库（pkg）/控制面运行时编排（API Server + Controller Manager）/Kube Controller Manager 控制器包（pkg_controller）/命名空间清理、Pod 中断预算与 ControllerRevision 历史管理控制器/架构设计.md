本模块由三个彼此独立的 controller 包组成，均遵循 client-go informer+workqueue 的标准控制器模式，但职责互不依赖：
- `namespace/`：`NamespaceController` 监听 Namespace informer，仅对设置了 DeletionTimestamp 的对象入队，通过 5s 的 `namespaceDeletionGracePeriod` 延迟处理以容忍 HA apiserver/etcd 复制延迟；实际删除逻辑委托给子包 `deletion.NamespacedResourcesDeleterInterface`（位于 `deletion/namespaced_resources_deleter.go`），后者使用 metadata client + APIResource discovery 动态发现并删除所有命名空间内对象，并通过 `status_condition_utils` 更新条件。
- `disruption/`：`DisruptionController` 同时监听 Pod、PDB、RC、RS、Deployment、StatefulSet 六个 informer，维护主队列 `queue`、重查队列 `recheckQueue` 和脏条件清理队列 `stalePodDisruptionQueue`；通过 `podControllerFinder` 函数列表（`getPodReplicationController/getPodDeployment/getPodReplicaSet/getPodStatefulSet/getScaleController`）按 OwnerReference 反查控制器 scale，支持 scale subresource 回退；`NewDisruptionController` 与 `NewDisruptionControllerInternal` 分离构造器以便注入 fake clock。
- `history/`：暴露 `history.Interface`（List/Create/Delete/Update/Adopt/Release），提供 `realHistory`（基于 clientset + lister + indexer）与 `fakeHistory`（基于 informer indexer）两套实现；通过 `AddControllerRevisionControllerIndexer` 在 SharedIndexInformer 上注册按 `namespace/kind/name/uid` 分片的索引键，加速按 ownerRef 查询；CR 名称采用 `prefix-hash` 格式，冲突时递增 `collisionCount` 重新哈希重试。
依赖方向：三者均只依赖 `k8s.io/client-go`、`k8s.io/api`、`k8s.io/apimachinery` 及本仓库 `pkg/controller` 公共工具，彼此之间无相互 import。
