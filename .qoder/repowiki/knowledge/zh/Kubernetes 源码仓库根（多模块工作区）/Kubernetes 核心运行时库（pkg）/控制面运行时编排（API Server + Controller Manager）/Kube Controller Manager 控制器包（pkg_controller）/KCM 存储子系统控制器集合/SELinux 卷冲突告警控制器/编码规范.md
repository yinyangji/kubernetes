- 所有 informer 回调统一签名 `(logger klog.Logger, obj interface{})`，并在入口处做类型断言 `obj.(*v1.Pod)` 失败即 return，避免 panic。
- 对不可变字段（如 Pod.Spec.Volumes、SELinuxOptions）仅在状态机进入终态（PodFailed/PodSucceeded）时才触发删除逻辑，更新路径先比较 Phase 快速跳过无关变更。
- 冲突检测采用“双向对称”策略：发现一对冲突时同时向两个 Pod 各生成一条 Conflict，保证事件与指标的双向一致性。
- 热路径上的 SELinux 标签解析结果以 `[4]string` 缓存于 `podInfo.seLinuxParts`，并通过 `ConflictsParsed` 传入，避免重复字符串分配。
- 对外暴露的接口（如 `VolumeCache`、`ControllerSELinuxTranslator`）均以 `_ = Interface(&struct{})` 形式在文件内做编译期实现检查。
