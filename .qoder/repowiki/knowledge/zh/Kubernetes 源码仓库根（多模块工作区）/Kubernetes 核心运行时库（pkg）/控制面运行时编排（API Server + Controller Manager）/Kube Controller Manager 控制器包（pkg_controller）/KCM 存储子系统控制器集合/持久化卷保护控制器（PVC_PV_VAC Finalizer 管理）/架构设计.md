本模块由三个并列的独立控制器组成，每个对应一种可保护资源：
- `pvcprotection/pvc_protection_controller.go`：监听 PVC 与 Pod informer，基于 `common.PodPVCIndex` 索引在缓存中查找引用该 PVC 的 Pod；若未命中则按命名空间发起一次“惰性”live List 作为补充，避免大规模并发删除时 API 风暴；支持 `PersistentVolumeClaimUnusedSinceTime` FeatureGate，维护 `PersistentVolumeClaimUnused` 条件。
- `pvprotection/pv_protection_controller.go`：仅监听 PV informer，依据 `pv.Status.Phase == Bound` 判断是否被 PVC 绑定，从而决定是否移除 `PVProtectionFinalizer`。
- `vacprotection/vac_protection_controller.go`：监听 VAC、PV、PVC informer，并为 PV/PVC informer 注册自定义 index `volumeAttributesClassName`，通过闭包 `getPVsAssignedToVAC`/`getPVCsAssignedToVAC` 查询引用某 VAC 的资源集合；注释明确承认其仅依赖 informer 存在最终一致性窗口。

三者共享相同的控制循环骨架：`New*Controller` 注入 clientset + informer → `Run(ctx, workers)` 等待 cache sync → 启动 N 个 worker 调用 `processNextWorkItem` → 核心处理函数先判 `IsDeletionCandidate` 再判 `NeedToAddFinalizer`，分别走 `removeFinalizer` / `addFinalizer` 路径，错误统一走 `AddRateLimited` 重试。finalizer 常量来自 `pkg/volume/util`，判定逻辑来自 `pkg/controller/util/protectionutil`，体现对上层 volume 子系统的单向依赖。
