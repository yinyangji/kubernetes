本目录是 kube-controller-manager 的控制器代码根，按功能域拆分为工作负载、批处理、网络、节点、存储、证书、GC、资源策略、Pod 生命周期、引导/认证等子模块；它们不互相 import，而是通过以下跨子模块契约耦合：
- 配置层：每个子控制器在自身 `config/v1alpha1` 下声明可选项，由顶层 `apis/config/v1alpha1/defaults.go` 集中注册默认值并生成 deepcopy，形成统一的 v1alpha1 配置协议。
- 运行时框架：所有控制器共享 `controller_utils.go` 中的 Expectations/TTLStore、`controller_ref_manager.go` 中的 PodControllerRefManager / ReplicaSetControllerRefManager 等引用管理骨架，以及 `testutil/test_utils.go` 测试桩，确保 informer/workqueue 驱动模式一致。
- 启动编排：外部入口（不在本 scope 内）按 feature gate 选择启用哪些子控制器，注入相同的 shared informers、clientset、event recorder 和 controller manager options，再调用各子模块的 `RunXxx` 方法。
- 子系统边界：storage、certificate 等复杂领域内部再细分 subpackage（如 `volume/attachdetach`、`certificates/signer`），对外仅暴露一个控制器入口，避免上层依赖过深。
