本模块由三个并列的控制器包组成，每个包遵循相同的 informer-controller 分层：
- 入口控制器（`deployment_controller.go`、`stateful_set.go`、`replica_set.go`）负责构造 `New*Controller`，注册 Deployment/RS/Pod（StatefulSet 额外监听 PVC、ControllerRevision）Informer 事件，将 key 放入带退避的 `workqueue.TypedRateLimitingInterface[string]`，并通过 `Run(ctx, workers)` 启动多 worker 循环。
- 同步逻辑按职责拆分到独立文件：Deployment 拆为 `sync.go`（主流程）、`rolling.go`/`recreate.go`（两种策略）、`progress.go`（进度与超时）、`rollback.go`；StatefulSet 拆为 `stateful_set_control.go`（编排核心）、`stateful_pod_control.go`（Pod/PVC 操作）、`stateful_set_status_updater.go`（状态更新）；ReplicaSet 集中在 `replica_set.go` + `replica_set_utils.go`。
- 控制面通过接口解耦：StatefulSet 定义 `StatefulSetControlInterface` 并由 `defaultStatefulSetControl` 实现，组合 `StatefulPodControl`、`StatefulSetStatusUpdaterInterface` 和 `history.Interface`；Deployment 直接调用 `controller.RSControlInterface` 操纵 RS；ReplicaSet 使用 `controller.PodControlInterface`。
- 共享依赖方向单向向外：三个控制器均依赖 `k8s.io/kubernetes/pkg/controller` 中的通用 Pod/RS 控制接口、`pkg/controller/history` 管理 ControllerRevision、`pkg/controller/util/consistency` 在 FeatureGate 开启时记录写一致性信息，并各自在 `config/v1alpha1` 下维护本地配置类型的 conversion/defaults/register 生成代码。
- 指标收集位于各包自有的 `metrics/metrics.go`，通过 `component-base/metrics` 或 `legacyregistry.Register` 注册。
