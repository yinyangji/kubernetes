- 控制器构造函数采用 `NewXxx(ctx, ...)` + 可选 `NewXxxInternal(...)` 双入口，后者用于注入测试用时间参数。
- Run 方法统一结构：defer `utilruntime.HandleCrash`，记录 start/shutdown 日志，`WaitForNamedCacheSyncWithContext` 等待 informer 同步后启动 `wait.UntilWithContext` worker，并在 defer 中关闭 queue 并 WaitGroup.Wait。
- 工作队列一律通过 `workqueue.NewTypedRateLimitingQueueWithConfig` 显式传入 `Name` 字段，便于 Prometheus 指标区分。
- Informer 事件处理器使用 `AddEventHandlerWithOptions(cache.HandlerOptions{Logger: &logger})` 注入结构化日志上下文。
- 错误处理遵循 `Get -> Done -> sync -> Forget` 模板，失败时 `AddRateLimited` 或根据 `apierrors.SuggestsClientDelay` 使用 `AddAfter` 退避。
