- 每个 sync 入口（`syncClaim`/`syncVolume`）先调用 `update*MigrationAnnotationsAndFinalizers` 修正迁移注解与 finalizer，再进入主状态机分支，确保迁移一致性前置。
- 所有对 API Server 的状态更新都走 `updateClaimStatus`/`updateVolumePhase` 系列函数，内部先比对当前 Status 是否已一致，仅在 dirty 时才发起 UpdateStatus 并回写本地 cache，避免无意义写入。
- 工作项入队统一经 `enqueueWork` 包装，对 `DeletedFinalStateUnknown` 做兜底还原对象，再以 `cache.SplitMetaNamespaceKey` 解析 key，保证 add/update/delete 三路事件收敛到同一 worker。
- 对外部可插拔能力（CSI 名称翻译、插件迁移判断）通过接口 `CSINameTranslator` / `CSIMigratedPluginManager` 注入，而非直接依赖具体实现，便于测试替换。
- 日志采用 `klog.FromContext(ctx)` 获取 logger，并在关键分支前输出包含对象引用与当前状态的结构化信息（`klog.KObj`、`getClaimStatusForLogging`/`getVolumeStatusForLogging`）。
