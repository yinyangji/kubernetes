- 每个 Controller 暴露 `New*Controller(ctx, ...)` 构造函数，接收 Pod/Service/Endpoints/EndpointSlice 等 Informer 与 clientset，在构造阶段完成 eventBroadcaster/recorder 初始化、InformerEventHandler 注册与队列创建。
- 所有控制器均实现 `Run(ctx, workers int)` 方法，启动前调用 `cache.WaitForNamedCacheSyncWithContext` 等待相关 Informer 同步，并使用 `sync.WaitGroup` 管理 worker goroutine 生命周期。
- 事件处理函数统一通过 `controller.KeyFunc(obj)` 获取 key 后入队，并在 `handleErr` 中根据 `NumRequeues < maxRetries` 决定是否 `AddRateLimited` 重试，超限则 `Forget` 并记录日志。
- 通过 `LabelManagedBy` 常量标记自己管理的 EndpointSlice/Endpoints，并在 onEndpointSlice* 回调中使用 `endpointSliceTracker.ShouldSync/Has/HandleDeletion` 过滤非自身变更，避免重复同步。
- Pod 变更走独立的 podQueue，先计算匹配的 Service 列表再经 `AddAfter(endpointUpdatesBatchPeriod)` 批量合并，减少风暴式更新。
- config 子包固定采用 `types.go` + `v1alpha1/{conversion,defaults,register,doc}` + `zz_generated.deepcopy.go` 的结构，配合 OWNERS 文件声明所有者。
