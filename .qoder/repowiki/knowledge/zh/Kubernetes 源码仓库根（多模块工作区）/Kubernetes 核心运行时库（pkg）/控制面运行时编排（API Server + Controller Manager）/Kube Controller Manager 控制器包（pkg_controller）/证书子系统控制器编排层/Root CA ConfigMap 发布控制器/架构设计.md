本模块实现一个基于 client-go informer/workqueue 的标准 Kubernetes 控制器：
- `NewPublisher` 接收 ConfigMapInformer、NamespaceInformer、clientset 以及 apiserver 根 CA 字节切片，注册两类事件处理器：ConfigMap 的 Delete/Update（仅当名称为 `kube-root-ca.crt` 时入队其 namespace）和 Namespace 的 Add/Active Update；`syncHandler` 通过可注入字段暴露以便测试。
- `Run(ctx, workers)` 启动多 worker goroutine，使用 `workqueue.NewTypedRateLimitingQueueWithConfig[string]` 带默认退避速率限制器，队列名为 `root_ca_cert_publisher`；等待 cache 同步后循环消费 key（namespace name），调用 `syncNamespace`。
- `syncNamespace` 逻辑：若目标命名空间不存在或处于 terminating 状态则直接返回；否则用 `reflect.DeepEqual` 对比现有 ConfigMap 的 Data 与期望值 `{"ca.crt": rootCA}` 及 `kubernetes.io/description` 注解，不一致时 DeepCopy 后 Update，保证幂等。
- 指标通过 `metrics.go` 在 `init()` 中注册到 `legacyregistry`，以 subsystem `root_ca_cert_publisher` 暴露 `sync_total`（CounterVec，label=code）与 `sync_duration_seconds`（HistogramVec，指数桶），由 `recordMetrics` 在 sync 结束时按 HTTP StatusError.Code 打标。
- 依赖方向单向向外：仅依赖 `k8s.io/client-go`、`k8s.io/api/core/v1`、`k8s.io/apimachinery` 与 `k8s.io/component-base/metrics`，不反向引用上层 controller-manager 包。
