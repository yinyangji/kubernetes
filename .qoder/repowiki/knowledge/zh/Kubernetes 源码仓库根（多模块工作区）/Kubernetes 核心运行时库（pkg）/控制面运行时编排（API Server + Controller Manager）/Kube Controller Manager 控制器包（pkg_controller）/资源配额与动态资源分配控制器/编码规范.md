- 控制器构造统一暴露 `NewController(...)` 工厂函数，接收 informer 接口而非具体客户端，便于测试时注入 mock informer。
- 工作项入队前统一用 `cache.DeletedFinalStateUnknown` 解包，并对非预期类型做 `runtime.HandleErrorWithLogger` 记录。
- Run 方法中统一使用 `sync.WaitGroup` 管理 worker goroutine，并在 defer 中调用 queue.ShutDown() 等待退出。
- 通过 `cache.WaitForNamedCacheSyncWithContext` 等待所有 informer HasSynced 后再启动 worker，确保首次同步数据完整。
- 错误处理统一走 `utilruntime.HandleErrorWithContext`，并将 key 重新入队 `AddRateLimited` 实现指数退避重试。
- 每个控制器在 `metrics/` 子包中定义独立的 prometheus collector，并在构造函数末尾调用 Register 完成注册。
