本模块由三个相互独立的控制器包组成，均遵循 client-go informer + workqueue 的标准控制循环模式：
- `pkg/controller/bootstrap`：包含 `Signer`（监听 Secret/ConfigMap，用 JWS 对 `cluster-info` ConfigMap 中的 kubeconfig 计算并写入 detached signature）、`TokenCleaner`（按过期时间清理 bootstrap token Secret），通过 `FilteringResourceEventHandler` 精确过滤目标资源，单队列串行合并更新。
- `pkg/controller/tainteviction`：`Controller` 监听 Node/Pod 变更，维护内存中 `taintedNodes` 映射；使用两个 typed queue（node/pod）+ 分片 channel 将同节点对象路由到同一 worker，保证 node 与 pod 处理无竞态；基于 `TimedWorkerQueue` 按最小 tolerationSeconds 调度删除任务，并在删除前注入 `DisruptionTarget` 条件与事件。
- `pkg/controller/scheduling/podgroupprotection`：`podgroup_protection_controller.go` 监听 PodGroup 与 Pod，为 PodGroup 安装 `scheduling.PodGroupProtectionFinalizer`，在删除时通过 Pod informer 的自定义索引 `activePodSchedulingGroup` 判断是否仍有活跃 Pod 引用该组，若无则移除 finalizer。
依赖方向：三者仅依赖 `k8s.io/client-go`、`k8s.io/api`、`k8s.io/apimachinery` 及本仓库内 `controller/util/*` 工具包，彼此之间无互相引用。
