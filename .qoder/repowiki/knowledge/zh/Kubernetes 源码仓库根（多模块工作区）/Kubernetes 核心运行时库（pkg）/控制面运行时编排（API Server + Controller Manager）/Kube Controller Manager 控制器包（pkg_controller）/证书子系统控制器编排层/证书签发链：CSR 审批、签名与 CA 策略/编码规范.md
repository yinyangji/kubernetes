- 控制器统一通过 `certificates.NewCertificateController(ctx, name, client, csrInformer, handle)` 构造，自身不直接管理 informer 或 worker 池。
- handle 入口先快速过滤已签发 (`Status.Certificate != nil`)、已批准/失败或 `SignerName` 不匹配的 CSR，避免不必要的解析与授权调用。
- CSR 识别逻辑以 `recognizer`/`isXxx` 函数形式注册到切片，新增场景只需追加条目而非修改核心流程。
- 错误路径一律通过追加 `CertificateFailed` 条件并调用 `UpdateStatus` 返回，而不是向上抛出未包装错误。
- 时间相关行为通过可选的 `Now func() time.Time` 注入，便于单元测试替换时钟。
