模块由三个并列的控制器包组成，每个包遵循相同的分层结构：
- 顶层 `*_controller.go` 定义 Controller 结构体、New* 构造函数、Run/worker/processNextWorkItem 主循环，并通过 informers/listers 监听资源变更，将 key 入队到 `workqueue.TypedRateLimitingInterface[string]`。
- 策略/算法文件按职责拆分：job 包拆出 `success_policy.go`、`pod_failure_policy.go`、`indexed_job_utils.go`、`backoff_utils.go`、`tracking_utils.go`；daemon 包拆出 `update.go` 实现滚动更新逻辑；cronjob 包通过 `injection.go` 暴露注入点并由 `utils.go` 提供辅助函数。
- 配置位于各包的 `config/types.go` + `config/v1alpha1/{conversion,defaults,register}.go`，以 v1alpha1 版本注册 API 并通过 conversion 提供默认值与转换。
- 指标统一放在各自包的 `metrics/metrics.go`，在 NewController 时调用 `Register()` 注册 prometheus 指标。
- 依赖方向单向向外：仅依赖 `k8s.io/client-go`、`k8s.io/api/batch|apps|core`、`k8s.io/apiserver/pkg/util/feature` 以及本仓库内 `pkg/controller`、`pkg/features`、`pkg/controller/util/consistency` 等共享库；cronjob 还复用 job 包的 `job/util` 工具。
- CronJob 使用独立的 `ControllerV2`（基于 DelayingQueue），而 Job/DaemonSet 使用传统 workqueue 模式，体现演进路径。
