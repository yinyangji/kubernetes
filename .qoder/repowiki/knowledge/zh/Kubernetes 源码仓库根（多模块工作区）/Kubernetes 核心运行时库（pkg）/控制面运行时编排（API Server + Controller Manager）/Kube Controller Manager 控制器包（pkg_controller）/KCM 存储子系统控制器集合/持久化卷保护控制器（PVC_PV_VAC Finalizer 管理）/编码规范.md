- 控制器构造器接收 klog.Logger 并在 Run 前用 `cache.WaitForNamedCacheSyncWithContext` 等待所有相关 Informer Synced，再以 `sync.WaitGroup` 启动 N 个 worker goroutine。
- 工作项入队一律使用 `workqueue.NewTypedRateLimitingQueueWithConfig` 并设置 `Name` 字段（如 `"pvcprotection"`），失败路径统一 `AddRateLimited` 配合 `utilruntime.HandleError`。
- 事件处理器只做轻量过滤后把 key 加入 queue，实际业务集中在 `processXxx` 方法内，遵循 Add/Update/Delete 三态分离的 handler 模板。
- finalizer 操作统一走 `protectionutil.IsDeletionCandidate` / `NeedToAddFinalizer` 两个判定，再分别调用 `addFinalizer` / `removeFinalizer`，后者通过 `DeepCopy` 后 `slice.RemoveString` 更新对象。
- 日志输出统一采用 `klog.FromContext(ctx)` 获取 logger，并以 `klog.KObj` / `klog.KRef` 结构化记录资源引用。
