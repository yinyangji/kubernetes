本模块由三个并列的控制器包组成，均遵循 client-go informer+workqueue 的标准控制器模式：
- `pkg/controller/nodelifecycle`：核心是 `node_lifecycle_controller.go`，通过监听 Node、Pod、Lease 事件并维护内部 `nodeHealthMap`（带 RWMutex），将 NodeCondition 映射到 TaintKey（如 NotReady→TaintNodeNotReady、Unknown→TaintNodeUnreachable）；内置 zone 级健康状态机（Initial/Normal/FullDisruption/PartialDisruption）驱动差异化驱逐速率；配置对象位于 `config/types.go`，并通过 `config/v1alpha1` 提供版本化 API 及 conversion/deepcopy 生成。
- `pkg/controller/nodeipam`：顶层 `Controller` 根据 `allocatorType` 在 Run 时选择两条路径——旧版 `legacyIPAM`（IPAMFromCluster/IPAMFromCloud）或新版 `cidrAllocator`；后者委托给 `ipam/` 子包，其中 `cidr_allocator.go` 定义 `CIDRAllocator` 接口并由 `range_allocator.go` 基于 `cidrset.CidrSet` 实现本地位图分配，`sync/` 负责跨集群/云的双向同步，`timeout.go` 封装超时重试。
- `pkg/controller/serviceaccount`：拆分为 `serviceaccounts_controller.go`（确保每个 Namespace 存在 default SA）、`tokens_controller.go`（为 SA 生成/清理 ServiceAccountToken Secret，使用两个 typed workqueue 分别处理 SA 与 Secret 事件）、`legacy_serviceaccount_token_cleaner.go`（清理旧式 token secret 引用）；同样拥有 `config/v1alpha1` 版本化配置。
依赖方向：各控制器仅向上依赖 `k8s.io/client-go`、`k8s.io/api/core/v1`、`cloud-provider` 等外部库，向下不反向被本模块其他包引用，彼此独立运行于同一 kube-controller-manager 进程内。
