- 控制器构造器统一返回 `(Controller, error)`，内部通过 `NewTypedRateLimitingQueueWithConfig` 创建带 Name 的 typed workqueue，并在 `Run` 中调用 `cache.WaitForNamedCacheSyncWithContext` 等待 informer 同步。
- informer 事件处理器使用 `cache.ResourceEventHandlerFuncs` 的 Add/Update/Delete 回调，并对 `DeletedFinalStateUnknown` 做显式类型断言与错误上报，避免 tombstone 丢失。
- 日志统一通过 `klog.FromContext(ctx)` 获取 logger，关键路径以 `V(4)` 输出调试信息，错误通过 `utilruntime.HandleError` 或 `logger.Error(err, ...)` 记录。
- 对外暴露的入口函数采用 `New*` / `NewPodGroupProtectionController` 等构造函数模式，将 clientset、informers、options 作为参数注入，便于测试替换。
