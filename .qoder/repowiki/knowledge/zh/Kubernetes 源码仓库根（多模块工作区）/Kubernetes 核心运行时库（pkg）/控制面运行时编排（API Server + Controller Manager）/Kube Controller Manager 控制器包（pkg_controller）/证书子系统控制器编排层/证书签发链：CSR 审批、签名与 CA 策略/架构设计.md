模块按证书生命周期拆为三个子包，依赖方向单向：approver → signer → authority。
- `authority/`：纯库层，定义 `CertificateAuthority` 结构体与 `SigningPolicy` 接口；`PermissiveSigningPolicy` 负责把 CSR 的 SAN/Usages 映射到 x509 模板并计算 NotBefore/NotAfter（含 backdate 与 short-lived 分支），最终调用 `x509.CreateCertificate` 输出 DER。
- `signer/`：面向控制器的运行时层。`ca_provider.go` 通过 `dynamiccertificates.DynamicCertKeyPairContent` 从磁盘文件轮询 CA 证书/私钥，用 `atomic.Value` 缓存 `authority.CertificateAuthority` 以实现无锁热更新；`signer.go` 暴露 `CSRSigningController`，内部持有 `certificates.CertificateController`，在 `handle` 中根据 `spec.signerName` 分派到 `isKubeletServing/isKubeletClient/isKubeAPIServerClient/isLegacyUnknown` 等验证函数，再调用 `authority.CertificateAuthority.Sign` 完成签发并把 PEM 写回 `csr.Status.Certificate`。`config/v1alpha1/` 提供控制器配置类型的 API 定义及 conversion/deepcopy 生成代码。
- `approver/`：仅包含 `sarApprover`，通过注册一组 `csrRecognizer`（`isNodeClientCert`、`isSelfNodeClientCert`）识别 kubelet 客户端 CSR，对每个匹配项发起 `SubjectAccessReview` 判断是否允许 `create selfnodeclient/nodeclient`，通过后追加 `CertificateApproved` 条件并调用 UpdateApproval。
三者均复用上层 `pkg/controller/certificates` 中的 `CertificateController` 作为事件循环骨架，自身只实现 `func(ctx, *capi.CertificateSigningRequest) error` 的 handle 回调。
