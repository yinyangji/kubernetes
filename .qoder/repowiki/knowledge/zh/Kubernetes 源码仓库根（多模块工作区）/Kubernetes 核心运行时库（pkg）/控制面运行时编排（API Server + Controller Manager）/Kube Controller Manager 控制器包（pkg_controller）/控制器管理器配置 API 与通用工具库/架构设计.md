本模块由三个相互独立的子包组成，均被上层 controller-manager 入口消费：
- `apis/config`：定义内部 API group `kubecontrollermanager.config.k8s.io`，通过 `types.go` 中的 `KubeControllerManagerConfiguration` 聚合来自 `k8s.io/controller-manager/config`、`k8s.io/cloud-provider/config` 以及每个具体控制器（如 `daemon/config`、`deployment/config`、`job/config` 等）的配置结构体；`register.go` 使用 `runtime.NewSchemeBuilder(addKnownTypes)` 注册该类型，`v1alpha1/` 下通过 `+k8s:deepcopy-gen` 注释配合代码生成器产出 `zz_generated.deepcopy.go`、`zz_generated.conversion.go`、`zz_generated.defaults.go`，并由 `scheme/scheme.go` 暴露给外部 scheme。
- `util/`：按功能域拆分子目录——`consistency` 实现基于 `ConsistencyStore` 接口的 `RealConsistencyStore`/`NoopConsistencyStore`，以 owner UID + GroupResource 维度记录写入 RV 并与 store 的 `LastSyncResourceVersion()` 比较，用于多 store 场景下的就绪性校验；`endpointslice/errors.go`、`node/controller_utils.go`、`protectionutils/utils.go`、`selectors/bimultimap.go` 提供节点保护、选择器双键映射等轻量工具。
- `testutil/test_utils.go`：提供 `FakeNodeHandler`（实现 `v1core.NodeInterface`）、`FakeLegacyHandler`、`FakeRecorder` 等 fake 对象，封装 JSON/Merge/StrategicMerge Patch 解析与冲突检测，供控制器单元测试复用。
依赖方向：本模块仅依赖 `k8s.io/apimachinery`、`client-go`、`klog` 等基础库，不反向依赖任何具体控制器实现，属于被消费的纯工具层。
