本目录是证书控制面的顶层编排点：
- `certificate_controller.go` 暴露 `CertificateController` 抽象，负责监听 CertificateSigningRequest informer、入队带退避的 workqueue、多 worker 并行处理以及 ignorableError 语义；各子控制器通过传入自定义 `handler` 复用该骨架。
- `authority/`（本地 CA 加载 + SigningPolicy）、`approver/`（基于 SAR 的自动审批）、`signer/`（可插拔签名者）共同实现 CSR 的签发链路，由上层控制器组合调用。
- `rootcacertpublisher/` 与 `clustertrustbundlepublisher/` 作为独立控制器分别维护 kube-root-ca.crt 与 ClusterTrustBundle，消费 authority/signer 产出的 CA 内容。
- `cleaner/` 按状态与过期阈值扫描并删除过期的 CSR 与 PodCertificateRequest，形成闭环。
- 所有子模块共享 `k8s.io/client-go` informer/listener 模式、`controller.KeyFunc` 键生成、`utilruntime.HandleCrash` 错误处理以及统一的 metrics 注入约定，使它们可在 apiserver 启动时以相同方式注册为控制器。
