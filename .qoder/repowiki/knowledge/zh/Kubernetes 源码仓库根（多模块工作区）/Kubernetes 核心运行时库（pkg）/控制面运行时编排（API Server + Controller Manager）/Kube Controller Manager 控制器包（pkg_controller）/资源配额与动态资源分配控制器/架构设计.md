本模块包含三个并列的独立控制器，均遵循 Kubernetes controller-runtime 风格的 informer + workqueue 模式：
- `resourcequota/`：基于 `ControllerOptions` 注入 `quota.Registry` 和 `NamespacedResourcesFunc`，通过 `QuotaMonitor`（`resource_quota_monitor.go`）按 discovery 结果动态启动各资源的 informers；维护普通队列与 `missingUsageQueue` 双队列以优先处理缺失初始 usage 的 quota；通过 `replenishQuota` 回调由 monitor 触发增量重算。
- `resourceclaim/`：监听 Pod / PodGroup / ResourceClaimTemplate / ResourceClaim 四类对象，借助 `cache.Indexer` 建立 pod→claim、podgroup→claim、claim→owner 等多维索引，并通过 `MutationCache` 在控制器自写后、informer 追上前的窗口内避免重复创建；通过 `controllerFeatures` 结构体配合 feature gate 开关 GenericWorkload / DRAAdminAccess / DRAPrioritizedList / DRAWorkloadResourceClaims 等能力。
- `resourcepoolstatusrequest/`：仅监听 ResourcePoolStatusRequest，按需从 ResourceSlice 与 ResourceClaim lister 两阶段聚合 pool 状态（先取 max generation，再只计数该 generation 的 slice），并带 TTL 后台清理 goroutine 回收过期请求。
三者均通过各自 `metrics/` 子包注册 prometheus metrics，且不互相依赖，仅共享 `k8s.io/kubernetes/pkg/controller` 通用工具。
