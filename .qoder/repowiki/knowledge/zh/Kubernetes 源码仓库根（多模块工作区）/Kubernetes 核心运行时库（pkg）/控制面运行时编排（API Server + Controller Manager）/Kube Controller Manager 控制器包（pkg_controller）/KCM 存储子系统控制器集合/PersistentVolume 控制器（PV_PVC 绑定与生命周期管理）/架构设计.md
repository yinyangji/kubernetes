模块以 `pv_controller.go` 中的 `PersistentVolumeController` 为核心，遵循 Kubernetes controller-runtime 经典模式：
- 框架层在 `pv_controller_base.go` 中通过 `NewController(ControllerParameters)` 构造控制器，注册 PV/PVC/StorageClass/Pod/Node 的 Informer 事件到独立的 `volumeQueue` 与 `claimQueue`，由 `volumeWorker` / `claimWorker` 单 goroutine 串行消费，保证同一 key 不并发；`Run()` 启动 resync、worker 并等待 cache 同步。
- 业务逻辑集中在 `pv_controller.go`：`syncClaim` 分派到 `syncUnboundClaim`（未绑定分支：匹配现有 PV 或调用 `provisionClaim` 动态供给）和 `syncBoundClaim`（已绑定分支：校验 ClaimRef UID、处理 Lost/Misbound），`syncVolume` 处理 PV 侧状态迁移（Available/Bound/Released/Failed）及 `reclaimVolume`/`unbindVolume`。
- 索引与缓存：`index.go` 提供 `persistentVolumeOrderedIndex`（按 StorageClass/AccessModes/Capacity 排序的 PV 有序索引，供 `findBestMatchForClaim` 使用）以及 Pod→PVC 的自定义 Indexer；`volumes` 与 `claims` 本地 Store 用于去抖与版本比较，避免 API Server 冲突风暴。
- 外部依赖边界：`volume_host.go` 让控制器实现 `vol.VolumeHost` 接口（仅暴露 `GetKubeClient`/`GetEventRecorder` 等必要方法，其余返回空/错误，因为 PV 控制器不直接挂载卷）；CSI 迁移通过 `CSINameTranslator` + `CSIMigratedPluginManager` 注入，在 `updateMigrationAnnotations` 中维护 `migrated-to` 注解与 in-tree 删除保护 finalizer。
- 配置分层：`config/types.go` 定义 `PersistentVolumeBinderControllerConfiguration`（含 `VolumeConfiguration`、`PersistentVolumeRecyclerConfiguration`），`config/v1alpha1/` 提供 conversion/defaults/register；`options/options.go` 通过 `pflag` 暴露 CLI 标志（如 `--enable-dynamic-provisioning`、recycler 超时参数），由上层 `controller-manager` 组装为 ControllerParameters。
- 辅助子包：`metrics/` 记录 provision+binding 端到端耗时与队列长度；`testing/` 提供测试桩；`*_test.go` 文件覆盖 binder/delete/provision/recycle 各路径。
