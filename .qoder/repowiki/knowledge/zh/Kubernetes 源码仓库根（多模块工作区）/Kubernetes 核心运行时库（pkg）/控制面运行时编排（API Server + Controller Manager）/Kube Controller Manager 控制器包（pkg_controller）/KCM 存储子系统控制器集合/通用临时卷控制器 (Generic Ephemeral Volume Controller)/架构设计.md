模块围绕 `controller.go` 中的 `ephemeralController` 结构体组织，遵循 Kubernetes controller-runtime 风格：
- 入口与生命周期：`NewController` 注入 `clientset.Interface`、Pod/PVC Informer，注册事件处理器（仅监听 Pod Add 和 PVC Delete），通过 `workqueue.TypedRateLimitingQueue[string]` 驱动多 worker 循环；`Run` 等待 informer cache 同步后启动 goroutine。
- 工作流：`enqueuePod` 过滤含 `vol.Ephemeral != nil` 的 Pod 入队；`syncHandler` 遍历 Pod.Spec.Volumes，对每个 Ephemeral 卷调用 `handleVolume`；后者通过 `ephemeral.VolumeClaimName` 计算 PVC 名称，若不存在则基于 `VolumeClaimTemplate` 构造 PVC 并以 Pod 作为 OwnerReference 创建，利用 GC 在 Pod 删除时级联回收。
- 索引优化：通过 `common.AddPodPVCIndexerIfNotPresent` 安装 Pod→PVC 反向索引，使 `onPVCDelete` 能高效定位引用该 PVC 的 Pod 并重新入队。
- 配置子包 `config/` 暴露 `EphemeralVolumeControllerConfiguration`（v1alpha1 API group），提供默认值与 deepcopy 生成，供 kube-controller-manager 启动参数解析。
- 指标子包 `metrics/` 暴露 `EphemeralVolumeCreateAttempts` / `EphemeralVolumeCreateFailures` 计数器，由 `RegisterMetrics` 统一注册。
- 依赖方向单向向外：本模块依赖 `k8s.io/component-helpers/storage/ephemeral`（PVC 命名/校验）、`pkg/controller/volume/common`（PodPVC 索引）以及标准 client-go informers/listers，不反向被 storage 其他控制器依赖。
