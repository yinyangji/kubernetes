模块由两层组成：
- `pkg/controlplane/apiserver`：面向所有控制面 apiserver 的通用编排层。入口是 `config.go` 中的 `BuildGenericConfig` / `CreateConfig`，把 `options.CompletedOptions` 依次 Apply 到 `genericapiserver.Config`（SecureServing → Features → APIEnablement → EgressSelector → Traces → OpenAPI → LongRunningFunc → Etcd），再调用 `BuildAuthorizer` 构造授权器并注入审计、聚合发现组；`server.go` 的 `Server.New` 在 generic server 之上通过 `AddPostStartHookOrDie` 注册 identity lease、coordinated leader election、peer endpoint reconciler、cluster authentication trust、legacy token tracking 等控制器，并通过 `aggregator.go` / `apiextensions.go` / `apis.go` 挂载聚合 API 与扩展 API。
- `pkg/kubeapiserver`：仅 kube-apiserver 使用的非通用能力，按横切关注点分目录：`options/` 提供 `BuiltInAuthenticationOptions` / `BuiltInAuthorizationOptions` 等带 `AddFlags` / `Validate` / `To*Config` / `ApplyTo` 生命周期的选项对象；`authorizer/modes` 集中声明 `AlwaysAllow|ABAC|Webhook|RBAC|Node` 等模式常量；`admission/exclusion` 维护默认排除资源列表；`authenticator` / `default_storage_factory_builder.go` 提供存储工厂构建与 etcd key prefix 映射（含 `SpecialDefaultResourcePrefixes`、`CohabitatingResources`）。
依赖方向单向向下：`controlplane/apiserver` 依赖 `kubeapiserver/options` 与 `kubeapiserver/authorizer`，而 `kubeapiserver` 不反向依赖 controlplane 层。
