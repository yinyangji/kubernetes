本模块按运行时代角色划分为三层：
- pkg/kubeapiserver/ 提供 API Server 的通用配置组装器（认证/授权/准入/聚合发现），被上层入口消费。
- pkg/controlplane/apiserver 基于 genericapiserver.Config 把上述子组件拼装成可运行的 API Server，并在 PostStartHook 中注册 control_plane_controllers 提供的内置控制器；pkg/controlplane/controller 与 pkg/controlplane/reconcilers 是这些控制器的实现与共享 reconciler。
- pkg/controller 是独立的 KCM 控制器包，通过各自的 config/v1alpha1 选项与 informer/workqueue 框架运行，不依赖 apiserver 生命周期。
三者的耦合点在于 pkg/controlplane/apiserver/server.go 与 instance.go：它们构造 genericapiserver.Config、调用 kubeapiserver 的 options 解析器、再注入 control_plane_controllers 的 Run 函数作为 PostStartHook，从而在同一个 API Server 进程中拉起控制面基础设施。
