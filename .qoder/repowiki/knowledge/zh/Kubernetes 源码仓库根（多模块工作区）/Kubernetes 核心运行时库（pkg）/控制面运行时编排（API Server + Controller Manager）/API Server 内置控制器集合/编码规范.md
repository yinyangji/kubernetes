- Controller 结构体统一持有 `Informer`、`Lister`、`cache.InformerSynced` 及对应的 `clientset.Interface` 或 typed client，构造函数只接收这些依赖而不自行创建 informers（部分控制器例外，如 defaultservicecidr/apiserverleasegc 自建 narrow informer）。
- Run/Start 方法以 `context.Context` 为主入口，内部用 `utilruntime.HandleCrashWithContext(ctx)` 包裹 panic 恢复，并在 defer 中关闭 queue/informer/eventBroadcaster。
- 工作队列使用 `workqueue.NewTypedRateLimitingQueueWithConfig` 并传入带 controllerName 的 `TypedRateLimitingQueueConfig`，错误处理统一走 `utilruntime.HandleErrorWithContext`。
- 对共享资源的写操作前先从 API server 二次 Get 最新对象再判断，避免 HA 下多实例并发导致的竞态删除/更新。
- 日志通过 `klog.FromContext(ctx)` 获取上下文 logger，关键路径使用结构化字段（如 `"controller", controllerName`、`"key", key`）而非拼接字符串。
