本模块按职责拆分为多个子包，每个子包实现一个独立的 Controller，遵循 client-go informer/listener/workqueue 标准模式：
- `leaderelection/`：核心子包，包含基于 LeaseCandidate 的协调式 Leader 选举控制器（`leaderelection_controller.go`）、选举策略选择算法（`election.go`，支持 OldestEmulationVersion 并比较 EmulationVersion/BinaryVersion）、以及通过 `resourcelock` 在 kube-system 命名空间进行传统 LeaderElection 的启动封装（`run_with_leaderelection.go`）。
- `apiserverleasegc/`：周期性扫描过期 apiserver Lease 并删除的轻量 GC 控制器，使用自定义 Narrowed Informer + List + Get 二次校验模式避免并发误删。
- `crdregistration/`：监听 CRD 变化并通过注入的 `AutoAPIServiceRegistration` 接口将 GroupVersion 同步到 APIService 自动注册器，采用 GroupVersion 作为队列 key 去重。
- `defaultservicecidr/`：根据启动参数生成名为 `kubernetes` 的默认 ServiceCIDR，单轮循环驱动，使用 ApplyStatus 设置 Ready 条件。
- `clusterauthenticationtrust/`、`kubernetesservice/`、`legacytokentracking/`、`systemnamespaces/`：各自维护单一资源状态，结构体字段统一持有 informer、listersynced、clientset 等依赖。
依赖方向单向向下：各控制器仅依赖 `k8s.io/client-go`、`k8s.io/apimachinery`、`k8s.io/klog/v2`，不互相引用；`leaderelection` 是唯一同时消费 v1.Lease 与 v1beta1.LeaseCandidate 的子包。
