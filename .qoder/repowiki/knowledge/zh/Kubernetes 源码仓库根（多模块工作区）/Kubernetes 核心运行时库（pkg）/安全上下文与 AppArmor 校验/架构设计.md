模块由两个并列包组成：
- `pkg/security/apparmor`：实现 AppArmor profile 的解析与校验。入口为 `validate.go` 中的 `Validator` 接口与 `NewValidator()`，内部通过 `helpers.go` 的 `GetProfile`/`isRequired` 从 pod/container 字段及废弃注解中推导 profile；`validate_disabled.go` 使用 `//go:build !linux` 在非 Linux 构建中将 `isDisabledBuild` 置为 true，使 `ValidateHost` 直接返回错误，从而以 build tag 隔离平台差异。
- `pkg/securitycontext`：围绕 `api.PodSecurityContext` 和 `api.SecurityContext` 暴露 Accessor/Mutator 接口（`accessors.go`），并通过 `podSecurityContextWrapper`、`containerSecurityContextWrapper`、`effectiveContainerSecurityContextWrapper` 三层包装器分别支持只读、读写以及“容器覆盖 Pod”的有效值合成；`util.go` 提供 `DetermineEffectiveSecurityContext`、`AddNoNewPrivileges`、`ConvertToRuntimeMaskedPaths` 等纯函数，按平台拆分到 `util_linux.go`、`util_darwin.go`、`util_windows.go`（如 Linux 下读取 `/sys/devices/system/cpu/possible` 计算 masked paths）。
依赖方向：apparmor 仅依赖 `k8s.io/api/core/v1` 与 `third_party/forked/libcontainer/apparmor`；securitycontext 仅依赖 `k8s.io/kubernetes/pkg/apis/core` 与标准库，两者均不互相引用，由上层 kubelet/admission 组合调用。
