模块按职责分为三层：
- 顶层 `pkg/printers` 暴露稳定接口：`ResourcePrinter`/`ResourcePrinterFunc`（`PrintObj`）用于任意 runtime.Object 到 io.Writer 的文本输出；`TableGenerator`/`PrintHandler` 配合 `GenerateOptions` 通过反射将对象转为 `metav1.Table`，由 `HumanReadableGenerator` 维护 `map[reflect.Type]*handlerEntry` 完成类型→函数分发，并通过 `ValidateRowPrintHandlerFunc` 强制 handler 签名 `func(obj, options) ([]TableRow, error)`。
- `internalversion` 子包集中注册所有内置 API 类型的表格列定义与 `printXxx`/`printXxxList` 处理器，统一通过 `AddHandlers(printers.PrintHandler)` 注入；`import_known_versions.go` 以空白导入触发各 API group 的 install 初始化，使 `get` 等命令能反序列化内部版本对象。
- `storage` 子包仅包含 `TableConvertor`，把 `metav1.TableOptions` 映射为 `GenerateOptions` 并委托给 `TableGenerator`，作为 REST storage 层与打印器的适配桥。
依赖方向严格单向：`storage` → `printers`；`internalversion` → `printers`；顶层接口不依赖任何具体 API 类型，`internalversion` 是唯一引入 `k8s.io/kubernetes/pkg/apis/*` 的地方，且受 `.import-restrictions` 约束禁止其他文件直接 import `pkg/api|apis`。
