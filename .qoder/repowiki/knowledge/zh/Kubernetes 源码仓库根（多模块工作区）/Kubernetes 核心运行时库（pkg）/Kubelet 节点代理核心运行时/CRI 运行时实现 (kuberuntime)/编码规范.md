- 平台相关代码通过文件名后缀 `_linux.go`、`_windows.go`、`_unsupported.go` 配合 Go build tag 在编译期分离，同一功能在不同平台提供同名函数/类型。
- 对外错误采用具名 sentinel error（如 `ErrCreateContainerConfig`、`ErrCreateContainer`、`ErrPreStartHook` 等），并在事件记录前将 containerID 替换为稳定名称以避免事件爆炸。
- 所有对底层 CRI 服务的调用均先经过 `instrumentedServices` 包装层，统一注入 Prometheus metrics 与 OpenTelemetry tracing span。
- kubecontainer 与 runtimeapi 之间的双向转换集中在独立函数（`ToKubeContainerImageSpec`、`ToRuntimeAPIImageSpec`、`toKubeContainer`、`sandboxToKubeContainer` 等），避免在业务逻辑中散落类型转换。
- FeatureGate 开关（如 `features.RuntimeClassInImageCriAPI`）在关键路径上显式检查后再填充对应字段，保持向后兼容。
