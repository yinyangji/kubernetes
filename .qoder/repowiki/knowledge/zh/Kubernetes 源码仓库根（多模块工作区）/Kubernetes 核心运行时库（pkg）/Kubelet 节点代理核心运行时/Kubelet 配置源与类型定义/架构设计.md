模块按职责拆分为四个子包：
- `config`：核心层。`mux.go` 通过 `merger` 接口把多个 source 的 channel 聚合到单一 goroutine；`config.go` 中的 `PodConfig` + `podStorage` 负责去重、语义比较（忽略本地 annotation）、生成 ADD/UPDATE/DELETE/REMOVE/RECONCILE 等 `kubetypes.PodUpdate` 并顺序投递；`sources.go` 暴露 `SourcesReady` 接口用于判断所有 source 是否已发送过 SET。
- 具体 Source 实现均通过 `NewSource*` 函数启动后台 goroutine，将解析后的 `[]*v1.Pod` 封装成 `sourceUpdate` 写入 mux 提供的 channel：`file.go`（文件/目录轮询 + inotify）/`http.go`（HTTP GET 单 Pod 或 PodList）/`apiserver.go`（基于 `cache.Reflector` 监听 nodeName 过滤的 Pod）。三者共享 `common.go` 中的 `tryDecodeSinglePod` / `tryDecodePodList` / `applyDefaults`（补 UID、命名空间、NodeName、Tolerations、Status=Pending），并通过 `warning_cache.go` 的 LRU 缓存去重静态 Pod 优先级告警。
- `kubeletconfig/types.go` 与 `defaults.go` 仅声明 kubelet 自身运行期配置片段及默认路径常量，由上层 `pkg/kubelet/config` 加载；`configfiles/` 与 `util/codec` 提供配置文件编解码辅助。
- `types/` 提供跨 kubelet 子系统的共享类型：`PodUpdate` 操作码、`Timestamp`、`SortedContainerStatuses`、资源预留 `Reservation`、镜像标注键等。
- `envvars/` 纯函数式工具，将 Service 列表转换为容器环境变量（兼容 Docker link 风格）。
依赖方向：`config` → `types` + `kubeletconfig` + `envvars`；其余子包互不依赖。
