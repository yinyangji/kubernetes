- 对外暴露的 Manager 类型均以接口形式定义（`devicemanager/types.go` 的 `Manager`、`dra/types.go` 的 `ContainerInfo`），具体实现放在同包内并以 `Impl` 后缀命名，便于测试时替换。
- 插件生命周期采用 `cache.PluginHandler` 回调模式：`RegisterPlugin`/`ValidatePlugin`/`DeRegisterPlugin` 由外部 plugin manager 驱动，内部仅维护 store 并触发同步逻辑（如 DRA 的 `sync` 与 `wipeResourceSlices`）。
- 关键状态变更后统一通过 `writeCheckpoint`/`syncToCheckpoint` 落盘，并在 `Start` 中先 `readCheckpoint` 恢复，保证 kubelet 重启后的幂等性。
- 错误处理遵循“不包装无意义上下文”约定：对外返回的错误直接包含 driver/resource 名称，避免重复的 `failed to ...: failed to ...` 嵌套。
- 并发安全通过 `sync.Mutex`/`RWMutex` 保护核心 map（`endpointStore`、`store`、`claimInfoCache`），并在释放锁后再执行可能阻塞的外部 I/O（如 gRPC 调用、文件写入）。
