- 策略通过 `policyName` 字符串常量（如 `PolicyStatic`）注册，`NewManager` 中以 switch 分支构造具体策略实例，新增策略需同时添加名称常量与分支。
- 所有对外暴露的 `Manager`/`Policy`/`State` 方法均以 `klog.Logger` 作为首参，日志上下文通过 `klog.FromContext(ctx)` 注入后转成 logger 传递。
- 对共享状态的读写统一先加 `m.Lock()`，并在 defer 中释放，避免并发 Allocate/Reconcile 竞争。
- 状态变更遵循幂等语义：`Allocate`/`RemoveContainer` 文档注释明确标注 idempotent，内部通过查询现有分配决定是否实际修改。
- Checkpoint 版本演进通过嵌入旧结构体（V2/V3）+ 新 `Data`/`DataChecksum` 字段实现，Marshal/Unmarshal 时同时生成兼容校验和以保证滚动升级。
