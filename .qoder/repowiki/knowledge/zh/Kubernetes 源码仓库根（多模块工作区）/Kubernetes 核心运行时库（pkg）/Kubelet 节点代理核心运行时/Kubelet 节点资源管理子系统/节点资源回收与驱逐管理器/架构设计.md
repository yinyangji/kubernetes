模块围绕 `Manager` 接口（`eviction_manager.go`）组织：
- `api/types.go` 定义跨包共享的 `Signal`、`ThresholdOperator`、`ThresholdValue`、`Threshold` 等类型，作为本模块对外的公共 API。
- `types.go` 声明 `Config`、`Manager`、`ImageGC`/`ContainerGC`、`CgroupNotifier`/`NotifierFactory`、`ThresholdNotifier` 等内部接口，以及 `managerImpl` 结构体；`NewManager` 同时返回 `Manager` 和 `lifecycle.PodAdmitHandler`，使驱逐状态能反向影响 Admission。
- `helpers.go` 集中实现信号到 NodeCondition 的映射（`signalToNodeCondition`）、Pod 排序/选择策略（`rankFunc`）、阈值解析（`ParseThresholdConfig`、`UpdateContainerFsThresholds`）与事件消息模板；平台差异通过 `_linux.go` / `_windows.go` / `_others.go` 文件后缀拆分。
- `memory_threshold_notifier.go` + `threshold_notifier_linux.go`/`_unsupported.go` 提供基于内核 memcg 通知的 `ThresholdNotifier` 实现，由 `CgroupNotifierFactory` 构造，用于在内存阈值跨越时立即触发 `synchronize`，避免轮询延迟。
- 控制流：`Start` 启动周期性同步循环，并在启用 `KernelMemcgNotification` 或 Windows 时为每个内存类阈值注册 `MemoryThresholdNotifier`；`Admit` 根据当前 `nodeConditions` 拒绝新 Pod，仅允许 Critical 或非 BestEffort 且容忍对应 Taint 的 Pod 进入。
- 依赖方向：本模块依赖 `stats.SummaryProvider`、`lifecycle`、`metrics`、`server/stats`、`component-helpers/resource` 等上层 kubelet 组件，但不被其反向依赖，边界清晰。
