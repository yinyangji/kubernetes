- 对外暴露 `Manager` 接口，内部 `manager` 结构体实现，并通过 `NewManager` / `NewInMemoryManager` 两种构造函数分别绑定 checkpoint 与内存 state 实现。
- 所有涉及 `InPlacePodVerticalScaling` 或 `InPlacePodLevelResourcesVerticalScaling` 的行为均以 `utilfeature.DefaultFeatureGate.Enabled(...)` 包裹，并在 `features_linux.go` / `features_windows.go` / `features_unsupported.go` 中以构建标签拆分平台差异。
- 写入 `state.State` 前统一先 `UpdatePodFromAllocation` 对比 desired 与 allocated，仅在存在差异时才触发 Set/Admit 流程，避免无变更的重复持久化。
- 错误路径通过 `metrics.PodInfeasibleResizes.WithLabelValues(reason).Inc()` 记录拒绝原因，再返回 `PodReasonInfeasible` 的 `PodAdmitResult`。
- 需要并发安全的共享状态（如 `podsWithPendingResizes`）一律通过 `allocationMutex` 保护，并在方法入口加锁、defer 解锁。
