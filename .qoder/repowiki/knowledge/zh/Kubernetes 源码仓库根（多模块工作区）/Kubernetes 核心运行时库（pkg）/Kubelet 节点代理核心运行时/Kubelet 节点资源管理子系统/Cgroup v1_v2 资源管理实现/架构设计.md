模块以 `CgroupManager` 接口为边界，通过两个无状态结构体 `cgroupV1impl` / `cgroupV2impl` 分别实现 v1 与 v2 语义，二者均嵌入共享的 `cgroupCommon`（由 `newCgroupCommon` 构造）复用通用逻辑；对外通过 `NewCgroupV1Manager` / `NewCgroupV2Manager` 工厂函数返回具体实例。

- 文件分层：`cgroup_v1_manager_linux.go` 与 `cgroup_v2_manager_linux.go` 各自实现 `Version()`、`Validate()`、`Exists()`、`MemoryUsage()`、`GetCgroupConfig()`，并通过 `getCgroupCPUConfig` / `getCgroupMemoryConfig` 调用共享的 `readCgroupMemoryConfig` 完成内存配置解析。
- v2 特有逻辑集中在本文件内：`getSupportedUnifiedControllers` 使用 `availableRootControllersOnce` 单例缓存根 cgroup 的控制器集合，`buildCgroupUnifiedPath` 基于 `cmutil.CgroupRoot` 拼接统一层级路径，`cpuWeightToCPUShares` 将 v2 `cpu.weight` 映射到 v1 `cpu.shares`。
- 依赖方向单向向下：manager 层仅依赖 `github.com/opencontainers/cgroups` 与 `fscommon` 做 cgroupfs 读写，并委托 `pkg/kubelet/cm/util` 下的平台相关工具（`GetPids`、v1 相对路径解析）；`util` 包再 fork 自 `third_party/forked/libcontainer/utils` 处理 v1 子系统挂载点查找。
- 平台隔离：`util/cgroups_linux.go` 与 `util/cgroups_unsupported.go` 通过 Go build tag 区分 Linux 与非 Linux 编译产物，使上层在跨平台时仍能编译通过。
