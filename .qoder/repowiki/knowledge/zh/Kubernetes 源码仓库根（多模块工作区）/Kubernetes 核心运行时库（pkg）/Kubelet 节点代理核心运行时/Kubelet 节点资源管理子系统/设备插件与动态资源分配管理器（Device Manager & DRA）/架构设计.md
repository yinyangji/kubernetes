本模块由两个并列的子包组成，分别实现两种扩展资源机制：
- `devicemanager`：传统 Device Plugin 的管理器。`manager.go` 中的 `ManagerImpl` 通过 `plugin/v1beta1/server.go` 启动 gRPC 服务监听 `/var/lib/kubelet/device-plugins`，接收插件的 `ListAndWatch` 推送并维护 `allDevices`/`healthyDevices`/`unhealthyDevices` 集合；`endpoint.go` 封装对单个插件的 gRPC 调用（Allocate/PreStartContainer/GetPreferredAllocation），并通过 `checkpoint/checkpoint.go` + `checkpointmanager` 持久化 pod→device 映射，支持 kubelet 重启恢复；`topology_hints.go` 与 `pod_devices.go` 配合 TopologyManager 做 NUMA 亲和过滤与预分配；`types.go` 暴露 `Manager` 接口供上层 lifecycle 调用。
- `dra`：Dynamic Resource Allocation 管理器。`manager.go` 中的 `Manager` 以 60s 周期运行 `reconcileLoop`，协调 `PrepareResources`/`UnprepareResources` 调用；`plugin/dra_plugin_manager.go` 的 `DRAPluginManager` 实现 `cache.PluginHandler`，负责按 driverName 发现、连接、监控 DRA 插件 gRPC 端点，并在驱动不可用时延迟删除 Node 上的 `ResourceSlice`；`state/` 子包提供基于文件的 claimInfo 与 healthInfo 检查点（`checkpointer.go` + `zz_generated.deepcopy.go`）；`claiminfo.go`/`healthinfo.go` 维护内存缓存并通过 `resourceupdates.Update` channel 向上层推送健康变更。
依赖方向：两者均依赖 `kubelet/pluginmanager/cache` 作为插件发现入口，向 `lifecycle`/`schedulerframework` 暴露 Allocate/UpdatePluginResources 等回调，并共享 `cm/resourceupdates` 通道用于健康事件传播。
