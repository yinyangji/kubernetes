模块按调用方拆分为两个同名包，职责清晰且单向依赖：
- `pkg/kubelet/qos`：面向 kubelet 主流程的公共 QoS 工具集，入口函数为 `GetContainerOOMScoreAdjust`，依据 `v1qos.GetPodQOS(pod)` 返回 Guaranteed/Burstable/BestEffort 三类容器的 OOM 调整值；Burstable 分支通过 `helpers.go` 中的 `remainingPodMemReqPerContainer` 在开启 `features.PodLevelResources` 时把未分配至容器的 Pod 级内存请求均摊到每个容器再参与公式 `1000 - 1000*(memReq)/capacity` 计算；同时针对 sidecar（由 `isSidecarContainer` 基于 `RestartPolicy==Always` 且名称匹配 initContainer 推断）做额外保护，确保其 OOM 分数不高于同 Pod 常规容器的最小值。
- `pkg/kubelet/cm/qos`：专供 Container Manager 使用的轻量子包，仅暴露 `ResourceIsolationLevel` 枚举及 `IsContainerEquivalentQOSGuaranteed`，用于判断容器是否满足“requests=limits”的等价 Guaranteed，从而决定 host/pod/container 三级资源隔离粒度。
依赖方向：cm/qos → core/v1；kubelet/qos → core/v1 + component-helpers/resource + apiserver feature gate + kubelet/types，二者互不引用，避免 kubelet 与 cm 之间的循环依赖。
