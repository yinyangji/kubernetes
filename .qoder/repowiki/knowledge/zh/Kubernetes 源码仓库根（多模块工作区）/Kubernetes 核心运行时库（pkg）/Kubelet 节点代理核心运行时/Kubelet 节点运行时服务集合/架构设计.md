本模块由一组彼此松耦合的 leaf 子包组成，每个子包围绕一个“节点级能力”提供独立的 Manager/Setter/Handler 实现，并通过接口与 kubelet 主流程解耦。

- 状态层：`status` 通过 `Manager` 接口维护 PodStatus 缓存（按 UID 映射到带版本号的 `versionedPodStatus`），以 10s ticker + 缓冲 channel 触发 `syncBatch`，仅当 kubelet 拥有的条件发生变化时才写 API；`nodestatus/setters.go` 用函数式 `Setter = func(ctx, *v1.Node) error` 组合出 Node 各字段（地址、容量、条件等）的增量更新逻辑。
- 可观测性：`metrics` 集中注册所有 kubelet 指标（`k8s.io/component-base/metrics`），`collectors/` 下按领域拆分 CRI、日志、资源、卷统计、PodCertificate 等 Collector。
- 配置注入：`token`、`secret`、`configmap`、`clustertrustbundle`、`podcertificate`、`runtimeclass` 各自暴露 `*Manager` 类型，负责从 API server 拉取并落盘对应对象供容器挂载。
- 安全与准入：`sysctl` 实现 `lifecycle.PodAdmitHandler`，基于内核已知 namespaced 前缀构建 allowlist 并在 admission 阶段拒绝不合规 sysctl；`events` 封装节点级事件常量与 resize 事件构造。
- 系统健康：`watchdog` 定义 `HealthChecker` 接口，`oom` 监听 Linux OOM 事件，`nodeshutdown` 通过 systemd inhibit 与本地 JSON 文件持久化优雅关机状态，均按 `*_linux.go` / `*_windows.go` / `*_unsupported.go` 做 build tag 隔离。
- 用户命名空间：`userns` 管理 hostUsers=false 的 pod 生命周期，同样使用平台 build tag 分离 Windows 与 Linux 路径。

依赖方向：这些子包被 kubelet 上层（pod worker、node status sync loop、admission chain）消费，自身只依赖 client-go、core/v1、component-helpers 与标准库，不反向依赖 kubelet 核心调度逻辑。
