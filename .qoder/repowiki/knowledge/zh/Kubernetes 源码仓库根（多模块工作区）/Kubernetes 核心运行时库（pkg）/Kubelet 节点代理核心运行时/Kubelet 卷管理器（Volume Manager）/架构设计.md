模块以 `volumeManager` 为顶层门面，暴露 `Run`、`WaitForAttachAndMount`、`WaitForUnmount` 等接口，内部组合四个子包形成分层：
- `cache`：线程安全的内存缓存，定义 `DesiredStateOfWorld` 与 `ActualStateOfWorld` 两个接口，分别维护“应该被挂载的卷→Pod”和“已实际挂载的卷→Pod”，并实现 `operationexecutor.ActualStateOfWorldMounterUpdater/AttacherUpdater` 回调以便操作执行器回写结果。
- `populator`：`DesiredStateOfWorldPopulator` 周期性遍历 `PodManager`，将新增/删除的 Pod 同步到 DSW，并在首次全量扫描后设置 `hasAddedPods` 标志供其他组件使用。
- `reconciler`：`Reconciler` 以固定周期运行主循环，先 `reconstructVolumes` 从磁盘重建 ASW（标记为 uncertain），再按 unmount → mountOrAttach → unmountDetachDevices → cleanOrphanVolumes 的顺序收敛；`readyToUnmount()` 依赖 populator 的 `HasAddedPods` 与 node.status 修复完成才允许卸载，防止误删仍在使用的卷。
- `metrics`：注册基于 DSW/ASW 与 VolumePluginMgr 的指标。

依赖方向：`volume_manager.go` 聚合各子包并通过 `operationexecutor.OperationExecutor` 下发异步 attach/mount/unmount/detach 任务；`populator` 只读访问 API Server 获取 PV/PVC 并写入 DSW；`reconciler` 读写 DSW/ASW 并调用 operation executor；`cache` 不反向依赖上层。CSI in-tree 迁移由 `csimigration.PluginManager` + `intreeToCSITranslator` 注入到 populator 与 manager 中处理。
