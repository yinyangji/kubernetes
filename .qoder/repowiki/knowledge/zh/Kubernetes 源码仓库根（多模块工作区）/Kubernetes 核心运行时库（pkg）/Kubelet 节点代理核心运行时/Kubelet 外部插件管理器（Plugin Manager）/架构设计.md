模块采用经典的 Desired State / Actual State 双缓存 + Reconciler 架构，按职责拆分为四个子包：
- `cache`：维护 `DesiredStateOfWorld`（由 pluginwatcher 填充）和 `ActualStateOfWorld`（由 operationexecutor 更新），并通过 `PluginHandler` 接口暴露 Validate/Register/DeRegister 状态机给上层消费者。
- `pluginwatcher`：监听 socket 目录变化，将发现的插件信息写入 desired state；包含 v1beta1/v1beta2 示例 API 与 example handler/plugin 用于演示。
- `reconciler`：周期性循环对比两个 cache，先触发 Unregister 再 Register，保证幂等与重注册场景下 UUID 变更时的正确顺序。
- `operationexecutor`：基于 `goroutinemap.GoRoutineMap` 对同一 socketPath 的操作串行化，并通过 `OperationGenerator` 生成实际 gRPC 调用函数，使用 `k8s.io/kubelet/pkg/apis/pluginregistration/v1` 协议经 Unix Domain Socket 与插件通信，并在成功/失败时回调 `notifyPlugin` 通知插件自身。
入口 `plugin_manager.go` 仅负责组装各组件、启动 watcher 与 reconciler 协程，并通过 `AddHandler` 注入具体插件类型处理器。依赖方向单向：plugin_manager → reconciler → operationexecutor → (cache, pluginregistration/gRPC)，pluginwatcher 只写 desired state，不反向依赖执行层。
