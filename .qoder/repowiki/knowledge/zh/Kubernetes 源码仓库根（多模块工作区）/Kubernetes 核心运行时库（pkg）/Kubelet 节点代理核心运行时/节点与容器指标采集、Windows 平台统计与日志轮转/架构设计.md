本模块由三个相互独立的子包组成，分别面向 Linux/通用指标、Windows 指标以及容器日志管理：

- `pkg/kubelet/stats`：以 `provider.go` 中的 `Provider` 为对外入口，内部通过 `containerStatsProvider` 接口解耦两种后端——`cri_stats_provider.go`（优先走 CRI `ListContainerStats/ListPodSandboxStats`，失败时回退到 cAdvisor）和 `cadvisor_stats_provider.go`（纯 cAdvisor 路径）。`host_stats_provider.go` 负责宿主文件系统指标，`pidlimit/` 子包读取 `/proc/sys/kernel/pid_max` 与 `/proc/loadavg` 暴露进程数限制。`helper.go` 集中了 cadvisor→statsapi 的转换逻辑。`*_linux.go` / `*_windows.go` / `*_others.go` 按构建标签拆分平台差异。
- `pkg/kubelet/winstats`：仅 `//go:build windows` 编译，提供 Windows 专属的 `Client` 接口与 `StatsClient` 实现，底层通过 `perfcounters.go` 调用 `github.com/JeffAshton/win_pdh` 访问 PDH 计数器（CPU、内存、网络），并通过 `syscall.GetDiskFreeSpaceExW` 获取目录容量；输出格式对齐 cAdvisor 的 `cadvisorapi.ContainerInfo/MachineInfo/VersionInfo`，以便被上层复用。
- `pkg/kubelet/logs`：`container_log_manager.go` 基于 `workqueue.TypedRateLimitingInterface[string]` + 固定 worker 协程循环，周期性扫描运行中容器的日志文件，按 `LogRotatePolicy.MaxSize/MaxFiles` 执行 `.gz` 压缩与时间戳后缀重命名，并通过 `runtimeService.ReopenContainerLog` 通知运行时重新打开句柄；`container_log_manager_stub.go` 在配置禁用轮转时返回空实现。

依赖方向：`stats.Provider` → `containerStatsProvider`（CRI/cAdvisor）+ `HostStatsProvider` + `pidlimit.Stats()`；`winstats` 独立于 stats，仅被 Windows 路径的 CRI provider 间接使用；`logs` 仅依赖 CRI RuntimeService 与 OS 抽象，三者互不耦合。
