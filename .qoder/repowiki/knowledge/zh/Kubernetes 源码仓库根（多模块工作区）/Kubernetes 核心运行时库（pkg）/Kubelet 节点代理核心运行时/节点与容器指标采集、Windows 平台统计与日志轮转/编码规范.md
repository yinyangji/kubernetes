- 跨平台差异通过 Go build tag 文件对拆分（如 `*_linux.go` / `*_windows.go` / `*_unsupported.go`），而非运行时 if-else。
- 可插拔的数据源通过小接口抽象（`containerStatsProvider`、`HostStatsProvider`、`winNodeStatsClient`），再由构造函数注入具体实现，便于测试替换与平台切换。
- 错误统一使用 `fmt.Errorf("...: %w", err)` 包裹底层错误，并在关键路径用 `errors.Is` 判断特定 sentinel（如 `cadvisormemory.ErrDataNotFound`）。
- 日志输出一律从上下文提取 logger（`klog.FromContext(ctx)`），避免全局 logger。
- Windows 平台代码顶部显式声明 `//go:build windows`，非 Windows 文件则保持无标签或带 `!windows` 标签。
