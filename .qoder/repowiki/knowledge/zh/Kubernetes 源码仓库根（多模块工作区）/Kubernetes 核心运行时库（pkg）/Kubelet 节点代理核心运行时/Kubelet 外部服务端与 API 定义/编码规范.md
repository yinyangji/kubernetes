- gRPC 服务采用 UnimplementedXxxServer 嵌入模式（如 `podsv1alpha1.UnimplementedPodsServer`、`podresources` 的 v1/v1alpha1 服务器），以便在不破坏 ABI 的前提下扩展方法。
- API 版本化遵循 v1/v1alpha1/v1beta1 三件套：每个版本目录包含 `doc.go`、`register.go`、`conversion.go`、`defaults.go` 及 `zz_generated.conversion.go`/`zz_generated.defaults.go`，并在顶层 `register.go` 中统一注册 scheme。
- gRPC 请求限流通过 `grpc.WithRateLimiter(ctx, serviceName, qps, burst)` 返回的 `UnaryInterceptor` 选项挂载，而非在业务 handler 内手动判断。
- Unix Socket 名称与默认 QPS/Burst 等连接参数集中在各自包的 `constants.go` 中定义（如 `Socket = "kubelet"`、`DefaultQPS = 100`），避免硬编码散落各处。
- 验证逻辑按平台拆分为 `_linux.go`/`_windows.go`/`_others.go` 文件后缀，同一组函数在不同 OS 下编译出不同实现。
