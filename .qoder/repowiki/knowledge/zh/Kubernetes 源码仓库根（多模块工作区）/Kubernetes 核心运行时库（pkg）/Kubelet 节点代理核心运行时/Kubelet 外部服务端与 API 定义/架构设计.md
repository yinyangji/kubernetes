本模块由两大子域组成，共同构成 Kubelet 的外部接口层：

- `pkg/kubelet/server`（HTTP REST 入口）：以 `server.go` 中的 `Server` 结构体为核心，基于 `go-restful/v3` 构建根容器，并通过 `ListenAndServeKubeletServer` 启动 `http.Server`；内置 `/metrics`、`/stats/`、`/logs/`、`/debug/pprof/`、`/pods`、`/runningpods/` 等路径，使用 `filteringContainer` 包装确保所有 handler 都经过统一的 filter 链（认证、鉴权、日志、OpenTelemetry 追踪）。`auth.go` 通过 `AuthInterface` 抽象认证/授权，`HostInterface` 抽象主机能力，使 server 与 kubelet 核心解耦。
- `pkg/kubelet/server/stats`：独立子包实现 `/stats/` 端点的数据采集与汇总，通过 `ResourceAnalyzer` 接口从 cadvisor/container runtime 拉取指标，按节点/系统容器/卷维度生成 summary，并支持 Windows/Linux 平台差异。
- `pkg/kubelet/server/metrics`：集中注册 kubelet server 维度的 Prometheus 指标。
- `pkg/kubelet/apis/config`：Kubelet 自身配置的 API 对象（`KubeletConfiguration` 等），位于 `kubelet.config.k8s.io` group，按 v1/v1alpha1/v1beta1 多版本组织，每个版本含 `register.go`、`conversion.go`、`defaults.go`、`zz_generated.*.go`，并由 `validation/` 目录按平台拆分 Linux/Windows 校验逻辑；`scheme/` 下存放 roundtrip 测试用例数据，用于保证跨版本序列化一致性。
- `pkg/kubelet/apis/podresources`：PodResources gRPC 服务（v1/v1alpha1）及其 client，默认 Unix Socket 名 `kubelet`，带独立的 QPS/Burst 常量。
- `pkg/kubelet/apis/pods`：Pods gRPC 服务（v1alpha1），内部维护一个 `broadcaster` 将 pod 变更事件广播给 watch 客户端，并提供 List/Get/Watch 三个 RPC。
- `pkg/kubelet/apis/grpc/ratelimit.go`：通用的 gRPC unary 限流拦截器，基于 `golang.org/x/time/rate`，供上述 gRPC 服务复用。

依赖方向：`server` → `stats` + `apis/*`；`apis/*` 之间相互独立，仅共享 `grpc` 限流工具；config 类型被上层 kubelet 启动流程消费，不反向依赖 server。
