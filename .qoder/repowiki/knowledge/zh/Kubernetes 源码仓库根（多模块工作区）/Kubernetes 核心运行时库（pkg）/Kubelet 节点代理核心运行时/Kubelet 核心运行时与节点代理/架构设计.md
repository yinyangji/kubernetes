本模块是 kubelet 的 `pkg/kubelet` 包，围绕一个全局 `Kubelet` 结构体组织，按职责拆分为多文件：
- 入口与装配：`kubelet.go` 定义 `NewMainKubelet` / `Run` 等构造与启动流程，聚合所有子子系统（lifecycle、pleg、eviction、server、pluginmanager、volumemanager 等）。
- 运行时状态：`runtime.go` 通过 `runtimeState` + `healthCheck` 机制集中维护 runtime/network/storage 三类错误与健康探针，供 Node 状态上报消费。
- Pod 调度执行：`pod_workers.go` 暴露 `PodWorkers` 接口，为每个 Pod UID 分配独立 goroutine，按 Sync/Terminating/Terminated 三态推进；`pod_container_deletor.go` 以有界 channel 异步删除历史 exited 容器；`reason_cache.go` 用 LRU 缓存最近一次容器启动失败原因以便回填到 status。
- 资源与卷：`volume_host.go` 实现 `volume.VolumeHost`/`KubeletVolumeHost`，将 kubelet 内部 secret/configmap/token/podCertificate/informerFactory 暴露给 volume plugin 层；`kubelet_volumes.go` 负责 VolumePluginMgr 初始化与挂载协调。
- 节点特性与状态：`kubelet_node_status*.go`、`kubelet_nodecache.go`、`kubelet_node_declared_features.go` 处理 Node 对象同步、NodeStatus 上报与 Declared Features 广播。
- 辅助策略：`active_deadline.go` 基于 `status.PodStatusProvider` 和 `record.EventRecorder` 判断并驱逐超过 `ActiveDeadlineSeconds` 的 Pod；`preemption/preemption.go` 实现 `CriticalPodAdmissionHandler`，在 Admission 因资源不足失败时按 QoS 优先级选择被抢占 Pod 集合。
- 持久化框架：`checkpointmanager/` 提供跨子系统的 Checkpointing Manager 抽象（checksum/errors/testing），被 devicemanager、cpumanager、memorymanager、allocation 等复用，README 明确社区倾向避免新增 checkpoint。
- 安全与网络：`certificate/` 封装 bootstrap 证书签发与 TLS transport；`client/` 提供带 mTLS 的 kubelet 客户端；`kubelet_network*.go` 隔离 Linux 与非 Linux 的网络差异。
- 工具库：`util/` 下按功能分 subpackage（cache/env/format/ioutils/manager/queue/sliceutils/store/swap），并通过 build tag 拆分 `boottime_util_*.go`、`util_linux.go`、`util_windows.go` 等平台相关实现。
依赖方向：本模块作为 kubelet 根包，向上被 `cmd/kubelet` 使用，向下依赖 `kubelet/*` 子包及 `k8s.io/kubernetes/pkg/volume`、`k8s.io/cri-client`、`k8s.io/client-go` 等外部库，自身不反向依赖上层 server 或 cmd。
