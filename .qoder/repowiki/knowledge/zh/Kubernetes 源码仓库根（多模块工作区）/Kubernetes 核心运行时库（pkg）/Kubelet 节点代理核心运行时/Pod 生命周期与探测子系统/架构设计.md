本模块由四个相互协作的子包构成，围绕 Pod 从发现到就绪的全生命周期：
- `pod/`：通过 `Manager` 接口维护静态 Pod 与其在 API Server 上对应的 Mirror Pod 的双向映射（UID/FullName 索引），提供 `GetPodByMirrorPod`、`TranslatePodUID` 等 API 屏蔽镜像语义差异；`mirror_client.go` 负责与 API Server 同步镜像 Pod。
- `lifecycle/`：定义 `PodAdmitHandler` 准入链与 `handlerRunner`，统一执行 Exec/HTTP/Sleep 三种 Lifecycle Hook，并通过 `features_linux.go`/`features_unsupported.go` 按平台裁剪能力；`predicate.go` 提供准入谓词组合。
- `prober/`：`Manager` 为每个容器的 Liveness/Readiness/Startup 探针创建独立 `worker`，将探测动作委托给 `prober`（调用 `kubecontainer.CommandRunner`），并将结果写入 `results.Manager`（三个独立实例分别对应三类探针）；`UpdatePodStatus` 根据缓存结果计算容器 Ready 状态。
- `pleg/`：定义 `PodLifecycleEventGenerator` 抽象，`evented.go` 基于 Watch 流实现增量事件，`generic.go` 基于周期性 Relist 实现兜底；上层仅消费 `Watch()` 通道中的 `PodLifecycleEvent`。
依赖方向：`lifecycle` 和 `prober` 均依赖 `kubecontainer` 运行期接口；`prober/results` 被 `prober` 独占；`pod/mirror_client` 依赖 API Server 客户端。各子包通过接口暴露边界，避免 kubelet 主流程直接耦合内部实现。
