本模块由三个相对独立的子包组成，分别实现不同层面的鉴权能力：
- `pkg/auth/authorizer/abac`：基于属性访问控制（ABAC）的策略引擎，从 JSONL 文件加载 `abac.Policy` 列表并实现 `authorizer.Authorizer` / `authorizer.RuleResolver` 接口，按 subject→verb→resource/non-resource 顺序匹配，返回 `DecisionAllow` 或 `NoOpinion`。
- `pkg/auth/nodeidentifier`：定义 `NodeIdentifier` 接口及默认实现，用于从 `user.Info` 中识别节点身份，供上层根据用户来源判断是否代表 Node。
- `pkg/certauthorization`：为 `certificates.k8s.io/signers` 资源提供便捷授权辅助函数，先以精确 `signerName` 再回退到 `domain/*` 通配符进行二次授权检查。
- `pkg/serviceaccount`：围绕 JWT 的完整生命周期——`claims.go` 构造标准 + 私域 claims；`jwt.go` 通过 `TokenGenerator` 接口支持 RSA/ECDSA/OpaqueSigner 三种签名后端，并通过泛型 `Validator[PrivateClaims]` 注入业务校验逻辑（内部使用 `NewValidator(getter)` 绑定 SA/Pod/Secret/Node 对象存在性校验），同时暴露 `PublicKeysGetter` 抽象与 `StaticPublicKeysGetter` 静态实现；`externaljwt/plugin/` 通过 gRPC Unix Socket 调用外部 `ExternalJWTSigner` 进程完成实际签名，并维护 `keyCache` 周期性同步公钥；`openidmetadata.go` 对外发布 JWKS/OIDC Discovery 元数据。
依赖方向：serviceaccount 与 certauthorization 仅依赖 apiserver 的 `authentication/user` 与 `authorization/authorizer` 接口，不反向依赖 auth 包；externaljwt plugin 依赖 `k8s.io/externaljwt/apis/v1` gRPC 协议，与外部签名器解耦。
