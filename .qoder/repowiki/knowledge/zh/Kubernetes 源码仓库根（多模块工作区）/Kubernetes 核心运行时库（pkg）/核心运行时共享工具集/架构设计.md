本模块是 `k8s.io/kubernetes` 内部的共享工具库，按能力域拆分为多个独立子包，彼此之间无强依赖，被上层组件按需引入：
- 平台抽象层：`filesystem` 通过 `Filesystem`/`File` 接口 + `defaultfs.go` 默认实现 + `util_unix.go`/`util_windows.go` 构建标签文件隔离 OS 差异；`oom`（`oom_linux.go`/`oom_unsupported.go`）、`procfs`（`procfs_linux.go`/`procfs_unsupported.go`）、`rlimit`、`flock`、`kernel`、`env` 均遵循同一模式。
- 系统子系统封装：`iptables` 在 Linux 下通过 `exec("iptables"|"ip6tables")` 暴露 `Interface`，并提供 `testing/fake.go` 供测试注入；`windows/service` 通过 `golang.org/x/sys/windows/svc` 将 Windows SCM 事件桥接到 apiserver 的优雅关闭流程。
- 并发与生命周期：`async.Runner` 管理一组以 `stop chan struct{}` 退出的 goroutine；`goroutinemap` 基于名称去重并可选指数退避，内部复用 `exponentialbackoff` 子包。
- Kubernetes 领域工具：`flag` 实现 `pflag.Value` 用于 IP/IPPort/PortRange/ReservedMemory/Taints 等自定义标志；`taints`、`tolerations`、`node`、`pod`、`labels` 提供对 v1.Taint/Toleration/Node/Pod 的解析、合并、Patch 等操作；`hash.DeepHashObject` 基于 `k8s.io/utils/dump` 做深度哈希；`cluster/ports` 集中声明各组件默认端口常量。
- 测试友好性：大量包通过 `_fake.go`/`_unsupported.go`/`_test_deps.go` 或 `testing/` 子目录提供 fake 实现，使单元测试无需真实内核/iptables/Windows 环境。
