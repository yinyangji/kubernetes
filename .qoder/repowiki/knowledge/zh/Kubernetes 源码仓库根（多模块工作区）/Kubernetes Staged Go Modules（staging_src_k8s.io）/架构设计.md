本目录是 Kubernetes monorepo 的“staged”发布层：每个顶级子目录（api、apimachinery、client-go、apiserver、cli-runtime、cloud-provider、component-base、code-generator、controller-manager、cri-api/cri-client/cri-streaming、csi-translation-lib、dynamic-resource-allocation、endpointslice、externaljwt、kms、kube-aggregator、kube-controller-manager、kube-proxy、kube-scheduler、kubectl、kubelet、metrics、mount-utils、pod-security-admission、sample-apiserver、sample-cli-plugin、sample-controller、streaming）都是一个独立的 Go module，拥有自己的 go.mod/go.sum 和 OWNERS 文件；它们之间通过 go.mod 中的 `replace` 指令互相引用 staging 内的其他模块（例如 api 将 k8s.io/apimachinery 替换为 ../apimachinery），从而在仓库内形成“多模块工作区”，对外则以版本号发布。

依赖方向自底向上：apimachinery → streaming → api → client-go/apiserver/cli-runtime/cloud-provider/component-base/... 等上层组件；code-generator 位于最底层，提供 deepcopy-gen、conversion-gen、defaulter-gen、validation-gen、applyconfiguration-gen、client-gen、informer-gen、lister-gen、register-gen、openapi-gen 等二进制，由 kube_codegen.sh 统一编排，扫描源码中 `+k8s:deepcopy-gen=`、`+genclient`、`+groupName` 等注释标签生成 zz_generated.* 文件。

每个模块遵循标准 Go 包布局：pkg/ 放库代码，cmd/ 或 main.go 放可执行程序入口，hack/ 放构建脚本，testdata/ 放测试数据，.github/PULL_REQUEST_TEMPLATE.md 提供 PR 模板。
