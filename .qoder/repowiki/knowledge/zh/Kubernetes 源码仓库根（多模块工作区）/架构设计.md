本仓库是 Kubernetes 的单一 Git 仓库，通过 go.work 将顶层 k8s.io/kubernetes 与 staging/src/k8s.io/* 下的数十个独立 Go module 组合为单个工作区；staging 目录内的每个子模块拥有自己的 go.mod，供外部项目通过 replace 指向源码本地开发，而顶层模块本身不被当作库使用（README 明确禁止）。

- 分层关系：core_runtime（pkg/...）提供所有组件共享的内部运行时；staged_libraries（staging/src/k8s.io/*）把 API、client-go、apiserver、controller-manager、kubelet 等拆成可独立发布的库；cli_and_build_tools（cmd/...、hack/...）消费上述库产出 kube-apiserver、kubectl 等二进制；tests 与 cluster_scripts 依赖这些产物运行集成/e2e 测试与集群启动。
- 构建编排：build_tooling（hack/）是所有 make target、代码生成、校验、CI 的统一入口，由根 Makefile 调用；go.work 由工具自动生成（首行注释 Do not edit directly），use 列表即当前版本产出的全部模块集合。
- 边界契约：.import-restrictions 约束 import 路径，确保 pkg/ 内部包不被外部直接引用，强制通过 staging 模块暴露公共 API；api_specs 与 .generated_files 作为只读规格与生成产物被各层共同消费。
