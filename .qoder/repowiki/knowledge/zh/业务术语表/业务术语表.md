### CVM (Confidential VM)
- 定义：机密虚拟机，基于 TEE（可信执行环境）硬件支持的隔离 VM 技术，如 Intel TDX、AMD SEV-SNP。在 Kubernetes 中通过 Device Plugin 上报扩展资源，结合 NodeSelector/Taint 机制调度到具备 TEE 能力的节点上运行敏感工作负载。
- 别名：机密虚拟机、confidential vm

### TEE (Trusted Execution Environment)
- 定义：可信执行环境，CPU 硬件提供的安全隔离区域，用于保护代码和数据免受恶意软件攻击。在 Kubernetes 场景中，Intel TDX 和 AMD SEV-SNP 是主要的 TEE 技术实现，通过 Device Plugin 机制向集群暴露为可调度的扩展资源。
- 别名：可信执行环境、trusted execution environment

### Device Plugin
- 定义：设备插件机制，Kubernetes 中节点上报硬件特性的标准框架。通过独立的进程注册到 Kubelet，动态上报 GPU、FPGA、TEE 设备等扩展资源，使 Scheduler 能够在调度时考虑这些特殊硬件能力。
- 别名：设备插件、device plugin manager

### Node Feature Discovery (NFD)
- 定义：节点特性发现工具，自动探测节点硬件能力（如 CPU 特性、GPU、TEE 支持等）并设置相应的 Labels，无需手动配置 --node-labels。常用于云环境中自动化节点能力标注。
- 别名：节点特性发现、NFD

### Lease 心跳
- 定义：Kubelet 定期更新的轻量级心跳机制，默认每 10 秒更新一次 NodeLease 对象。Node Controller 通过监控 Lease 的最后更新时间来判断节点健康状态，超时则标记节点为 NotReady。
- 别名：lease heartbeat、node lease

### PodWorkers
- 定义：Kubelet 中为每个 Pod 维护的独立 Goroutine，负责处理单个 Pod 的生命周期同步。每个 Pod 有独立的 channel 和状态机，确保 Pod 操作的并发安全性和隔离性。
- 别名：pod worker goroutine、pod synchronization worker

### syncLoop
- 定义：Kubelet 的主事件循环，从多个 channel 读取配置变更、容器状态事件、定时任务等，分发给相应的处理器执行。是 Kubelet 协调所有节点本地状态的核心调度器。
- 别名：kubelet sync loop、主同步循环

### Assume 机制
- 定义：Scheduler 的性能优化策略，在内存中假设绑定操作成功，立即开始下一个 Pod 的调度，而不等待实际的 API Server 写入完成。通过假设状态保证调度决策的一致性。
- 别名：scheduler assume、binding assumption

### Filter + Score 调度模型
- 定义：Kubernetes 调度器的两阶段算法模型：Filter 阶段并行过滤不可用节点，Score 阶段对可行节点打分排序。这是 Scheduler 选择最优节点的核心决策流程。
- 别名：filter score algorithm、two-phase scheduling

### Informer/Watch 机制
- 定义：Kubernetes 组件间通信的基础模式，通过 List-Watch 长连接获取 API Server 的数据变更推送，配合本地缓存实现高效的状态同步。是所有控制器和组件的核心通信范式。
- 别名：informer watch pattern、list-watch mechanism
